## Проблеми са инсталацијом

### Инсталација Питона

#### Проблем: Верзија Питона је прениска
**Грешка:** `Потребан је Python 3.6 или новија верзија`

**Решење:**
1. Преузмите најновији Python 3 са [python.org](https://www.python.org/downloads/)
2. Током инсталације на Windows-у, обавезно означите „Add Python to PATH“
3. Проверите инсталацију:
   ```bash
   python3 --version
   ```

#### Проблем: Вишеструке верзије Питона изазивају конфликте
**Симптоми:** Руши се покретање погрешне верзије Python-а, пакети се инсталирају на погрешну локацију

**Решење:**
- **Windows:** Користите `py -3` уместо `python` за изрично покретање Python 3
- **macOS/Linux:** Користите `python3` уместо `python`
- Увек правите и користите виртуелна окружења за пројекте

#### Проблем: команда pip није пронађена
**Грешка:** `'pip' није препознат као интерна или екстерна команда`

**Решење:**
1. Покушајте `pip3` уместо `pip`
2. Или користите `python -m pip` или `python3 -m pip`
3. Проверите да ли је Python додат PATH-у (поново инсталирајте Python и означите ту опцију)

### VS Code и екстензије

#### Проблем: Pylance екстензија не ради
**Симптоми:** Нема Python IntelliSense-а, аутоматског довршавања кода или провере типова

**Решење:**
1. Отворите командну палету у VS Code-у (`Ctrl+Shift+P` или `Cmd+Shift+P`)
2. Покрените „Python: Select Interpreter“
3. Изаберите одговарајући Python интерпретер (виртуелно окружење ако користите)
4. Поново учитајте VS Code прозор

#### Проблем: VS Code не детектује виртуелно окружење
**Симптоми:** Изабран је погрешан Python интерпретер

**Решење:**
1. Проверите да ли сте активирали виртуелно окружење у терминалу
2. Отворите командну палету и покрените "Python: Select Interpreter"
3. Изаберите интерпретер из `.venv` фасцикле
4. Проверите да ли статусна трака (лево доле) показује исправну верзију Python-а

### PlatformIO (Wio Terminal)

#### Проблем: Неуспешна инсталација PlatformIO
**Грешка:** Разне грешке током инсталације PlatformIO

**Решење:**
1. Обавезно имате најсвежију верзију VS Code-а
2. Прво инсталирајте C/C++ екстензију
3. Поново покрените VS Code након инсталације PlatformIO-а
4. Проверите интернет конекцију (PlatformIO преузима велике датотеке)

#### Проблем: Плата није детектована у PlatformIO-у
**Симптоми:** Не можете отпремити код на Wio Terminal

**Решење:**
1. Покушајте други USB кабл (неке каблове користити само за пуњење)
2. Проверите Device Manager (Windows) или `ls /dev/tty*` (macOS/Linux)
3. Инсталирајте или ажурирајте USB драјвере
4. Покушајте други USB порт
5. Брзо повуците прекидач напајања на Wio Terminal два пута да бисте ушли у bootloader режим

#### Проблем: Грешке приликом компилације у PlatformIO-у
**Грешка:** `fatal error: Arduino.h: No such file or directory`

**Решење:**
1. Обришите `.pio` фасциклу у вашем пројекту
2. Покрените "PlatformIO: Rebuild" из командне палете
3. Уверите се да `platformio.ini` садржи исправну конфигурацију плоче:
   ```ini
   [env:seeed_wio_terminal]
   platform = atmelsam
   board = seeed_wio_terminal
   framework = arduino
   ```

### Grove библиотеке

#### Проблем: Неуспешан увоз Grove библиотеке на Raspberry Pi
**Грешка:** `ModuleNotFoundError: No module named 'grove'`

**Решење:**
1. Поново инсталирајте Grove библиотеке:
   ```bash
   cd ~
   git clone https://github.com/Seeed-Studio/grove.py
   cd grove.py
   sudo pip3 install .
   ```
2. Ако користите виртуелно окружење, можда ћете морати инсталирати глобално или копирати библиотеке
3. Проверите да ли је I2C омогућен: `sudo raspi-config nonint do_i2c 0`

#### Проблем: Grove сензор није детектован
**Грешка:** `IOError: [Errno 121] Remote I/O error`

**Решење:**
1. Проверите физичка повезивања (проверите да ли је Grove кабл у потпуности уметнут)
2. Проверите да ли је сензор повезан на исправан порт (аналогни, дигитални, I2C, UART)
3. Покрените `i2cdetect -y 1` да видите да ли уређај појављује на I2C сабирници
4. Покушајте други Grove кабл
5. Уверите се да је Grove Base Hat правилно постављен на Raspberry Pi GPIO пинове

## Проблеми са хардвером

### Raspberry Pi

#### Проблем: Raspberry Pi се не покреће
**Симптоми:** Нема слике, нема активности LED лампица, или појављује се шарена/дугинина екран

**Решење:**
1. **Проверите напајање:** Користите званични 5В 3А USB-C адаптер за Pi 4
2. **Проблеми са SD картицом:** 
   - Форматирајте SD картицу и поново инсталирајте Raspberry Pi OS
   - Покушајте другу SD картицу (користите препоручене брендове)
   - Уверите се да је SD картица правилно уметнута
3. **Проверите HDMI конекцију:** Испробајте оба HDMI порта на Pi 4, користите HDMI порт ближи напајању

#### Проблем: Не могу да се прикључим на Raspberry Pi преко SSH-а
**Симптоми:** Веза одбијена или истек времена покушаја

**Решење:**
1. Омогућите SSH:
   - При флешовању SD картице користите Raspberry Pi Imager и унапред конфигуришите SSH у напредним опцијама
   - Или направите празну датотеку звaну `ssh` (без екстензије) у boot партицији
2. Пронађите IP адресу Pi-а:
   - Прегледајте повезане уређаје на вашем рутеру
   - Користите `ping raspberrypi.local` (ако mDNS ради)
   - Користите алате за скенирање мреже као `nmap` или Angry IP Scanner
3. Проверите мрежу:
   - Уверите се да је Pi на истој мрежи као ваш рачунар
   - Покушајте Ethernet везу уместо WiFi
4. Потврдите корисничко име/лозинку (подразумевано: корисник `pi`, лозинка `raspberry`)

#### Проблем: Grove Base Hat није препознат
**Симптоми:** Сензори не раде, појављују се I2C грешке

**Решење:**
1. Уверите се да је Base Hat правилно постављен на све GPIO пинове
2. Проверите да ли има савијених пинова на Pi-у или Base Hat-у
3. Омогућите I2C интерфејс:
   ```bash
   sudo raspi-config nonint do_i2c 0
   sudo reboot
   ```
4. Потврдите да I2C ради: `i2cdetect -y 1`

#### Проблем: Raspberry Pi ради споро
**Симптоми:** Заостајање интерфејса, споро реаговање

**Решење:**
1. Проверите брзину SD картице (користите Class 10 или бољу, или SSD преко USB)
2. Ослободите простор на диску: `df -h` за проверу, обришите непотребне фајлове
3. Смањите GPU меморију у `raspi-config` ако не користите камеру/екран интензивно
4. Затворите непотребне апликације
5. Размислите о надоградњи на Pi 4 са више RAM-а ако користите Pi 3 или старији модел

### Wio Terminal

#### Проблем: Екран на Wio Terminal остаје празан
**Симптоми:** Нема слике након отпремања кода

**Решење:**
1. Проверите да ли код иницијализује екран (користећи TFT_eSPI библиотеку)
2. Ажурирајте Wio Terminal firmware са [Seeed Wiki](https://wiki.seeedstudio.com/Wio-Terminal-Getting-Started/)
3. Додајте код за иницијализацију екрана:
   ```cpp
   #include <TFT_eSPI.h>
   TFT_eSPI tft;
   tft.begin();
   tft.fillScreen(TFT_BLACK);
   ```
4. Покушајте да отпремите пример из PlatformIO-а да тестирате хардвер

#### Проблем: WiFi не ради на Wio Terminal-у
**Симптоми:** Не можете да се повежете на WiFi, јављају се мрежне грешке

**Решење:**
1. **Ажурирајте WiFi firmware:** Пратите [Wio Terminal WiFi firmware update guide](https://wiki.seeedstudio.com/Wio-Terminal-Network-Overview/)
2. **Проверите WiFi акредитиве:** Уверите се да су SSID и лозинка исправни
3. **WiFi опсег:** Wio Terminal подржава само 2.4GHz WiFi (не 5GHz)
4. **Јачина сигнала:** Приближите се рутеру
5. **Подешавања рутера:** Неки enterprise/WPA-Enterprise рутери можда неће радити

#### Проблем: Wio Terminal није препознат од стране рачунара
**Симптоми:** Уређај преко USB-а није детектован

**Решење:**
1. **Пробајте други USB кабл:** Користите USB кабл за пренос података, не само за пуњење
2. **Уђите у bootloader режим:** Брзо повуците прекидач напајања надоле два пута
   - Плави LED би требало да пулсира, уређај се појављује као "Arduino" у Device Manager-у
3. **Инсталирајте драјвере (Windows):**
   - Преузмите и инсталирајте [Seeed USB driver](https://wiki.seeedstudio.com/Driver_for_Seeeduino/)
4. **Пробајте други USB порт:** Избегавајте USB хабове, користите директну везу
5. **Ажурирајте системске USB драјвере**

#### Проблем: Сензори не раде на Wio Terminal-у
**Симптоми:** Grove сензори не читају податке

**Решење:**
1. Проверите везе Grove каблова
2. Уверите се да користите исправан Grove порт (леви или десни)
3. Укључите исправне библиотеке за сензор
4. Проверите напајање сензора
5. Тестирајте сензор примером кода из библиотеке

### Виртуелни уређај (CounterFit)

#### Проблем: CounterFit апликација се не покреће
**Грешка:** Разне Python грешке приликом покретања CounterFit-а

**Решење:**
1. Уверите се да је виртуелно окружење активирано
2. Инсталирајте/поново инсталирајте CounterFit:
   ```bash
   pip install CounterFit
   ```
3. Проверите да ли порт 5000 није већ у употреби:
   - Windows: `netstat -ano | findstr :5000`
   - macOS/Linux: `lsof -i :5000`
4. Убијте процес који користи порт 5000 или користите други порт:
   ```bash
   counterfit --port 5001
   ```

#### Проблем: Не може се повезати на CounterFit из кода
**Грешка:** Веза одбијена или истек времена

**Решење:**
1. Потврдите да CounterFit ради: Отворите претраживач на `http://127.0.0.1:5000`
2. Проверите да ли URL конекције у коду одговара адреси CounterFit-а
3. Уверите се да firewall не блокира везу
4. Покушајте поново да покренете и CounterFit апликацију и ваш код

#### Проблем: Сензори се не појављују у CounterFit-у
**Симптоми:** Креирани сензори се не приказују у CounterFit корисничком интерфејсу

**Решење:**
1. Креирајте сензоре у CounterFit интерфејсу пре покретања кода
2. Освежите страницу претраживача
3. Проверите да се тип сензора поклапа са очекивањима кода
4. Очистите кеш претраживача

## Проблеми са повезивањем

### WiFi конекција

#### Проблем: Уређај се не повезује на WiFi
**Симптоми:** Веза истиче, аутентификација није успела

**Решење:**
1. **Проверите SSID и лозинку:** Потврдите да су акредитиви тачни
2. **WiFi опсег:** Већина IoT уређаја подржава само 2.4GHz (не 5GHz)
3. **Подешавања рутера:**
   - Искључите AP изолацију ако је укључена
   - Користите WPA2-PSK безбедност (избегавајте WPA3, WEP или отворене мреже)
   - Уверите се да је DHCP укључен
4. **Скривене мреже:** Ако је SSID скривен, морате га јасно конфигурисати
5. **Јачина сигнала:** Приближите уређај рутеру
6. **Интерференција:** Остали уређаји, микроталасне пећнице или зидови могу ометати сигнал

#### Проблем: WiFi конекција се често губи
**Симптоми:** Повремене прекиде везе

**Решење:**
1. Проверите стабилност рутера и размотрите рестарт
2. Ажурирајте firmware уређаја
3. Користите статичку IP адресу уместо DHCP-а
4. Смањите раздаљину до рутера или додајте WiFi појачивач
5. Проверите интерференцију са другим уређајима
6. Потврдите да је напајање адекватно (посебно за Raspberry Pi)

### Cloud сервисе

#### Проблем: Не може се повезати на Azure IoT Hub
**Грешка:** Аутентификација није успела, веза одбијена

**Решење:**
1. **Проверите акредитиве:**
   - Потврдите да је конекциони стринг тачан
   - Уверите се да нема додатних размаза или прекида у стрингу
2. **Проверите регистрацију уређаја:** Уређај мора бити регистрован у IoT Hub-у
3. **Firewall/proxy:** Уверите се да је дозвољен излазни MQTT (порт 8883) или HTTPS (порт 443)
4. **Регион IoT Hub-а:** Проверите да је IoT Hub активан и није у другом региону који изазива кашњење
5. **Квоте:** Проверите да ли су лимити бесплатног нивоа премашени
6. **Тестирајте везу:**
   ```bash
   az iot hub device-identity show-connection-string --hub-name YourIoTHub --device-id YourDevice
   ```

#### Проблем: Azure Functions се не покрећу
**Симптоми:** Поруке се шаљу, али функција се не извршава

**Решење:**
1. Проверите да је Function App у покрету (није заустављен)
2. Потврдите конекциони стринг у подешавањима Function App-а
3. Прегледајте логове функције у Azure порталу
4. Уверите се да је Event Hub компатибилан сајт тачно подешен
5. Проверите да формат поруке одговара очекивањима функције
6. Проверите план услуга Function App-а (consumption vs dedicated)

### MQTT
#### Проблем: MQTT веза не успева
**Грешка:** Веза одбијена, аутентификација није успела

**Решење:**
1. **Адреса брокера:** Проверите да ли је URL/IP брокера тачан
2. **Порт:** Проверите број порта (1883 за нешифровану, 8883 за TLS)
3. **Аутентификација:** Проверите корисничко име/лозинку ако је потребно
4. **TLS/SSL:** Уверите се да су сертификати важећи и поуздани
5. **Фајервол:** Проверите да порт није блокиран
6. **Тестирајте са MQTT клијентом:** Користите MQTT Explorer или mosquitto_pub/sub за тест

#### Проблем: MQTT поруке нису примљене
**Симптоми:** Поруке су објављене али нису примљене од стране претплатника

**Решење:**
1. **Називи тема:** Проверите да тема претплатника тачно одговара теми издавача
2. **Ниво QoS:** Покушајте са QoS 1 или 2 уместо 0
3. **Вајлдкардови:** Проверите правилну употребу вајлдкардова теме (`+` за један ниво, `#` за више нивоа)
4. **Сачуване поруке:** Издавач може подесити retain заставицу да сачува последњу поруку
5. **Време повезивања:** Уверите се да се претплатник повезује пре него што поруке буду објављене

## Проблеми са сензорима и извршиоцима

### Grove сензори

#### Проблем: Сензор враћа нетачне вредности
**Симптоми:** Очитавања су 0, -1 или бесмислене вредности

**Решење:**
1. **Проверите везе:** Уверите се да је сензор исправно повезан
2. **Правилан порт:** Проверите да је сензор у исправном типу порта:
   - Аналогни сензори → Аналогни портови (A0, A2, A4)
   - Дигитални сензори → Дигитални портови (D5, D16, D18 итд.)
   - I2C сензори → I2C портови
3. **Калибрација:** Неки сензори захтевају калибрацију (влага земљишта, светлост)
4. **Поново прикључите:** Искључите и поново прикључите сензор
5. **Подаци о сензору:** Проверите спецификације и захтеве сензора

#### Проблем: Капацитивни сензор влажности земљишта увек очитава влажност
**Симптоми:** Сензор очитава високу влажност чак и када је суво

**Решење:**
1. **Потребна калибрација:** Сензори влажности земљишта захтевају калибрацију:
   - Очитајте вредност у ваздуху (суви почетак)
   - Очитајте вредност у води (мокри почетак)
   - Мапирајте очитавања између тих вредности
2. **Проверите премаз сензора:** Сензори влажности могу бити оштећени ако је премаз нарушен
3. **Положај:** Уверите се да је сензор у потпуности убацен у тло

#### Проблем: Нетачна очитавања температуре/влажности
**Симптоми:** DHT11/DHT22 показује погрешну температуру или влажност

**Решење:**
1. **Положај сензора:** Избегавајте директно сунчево светло, изворе топлоте или проток ваздуха
2. **Време загревања:** Дозволите да сензор ради 2 секунде након укључивања пре очитавања
3. **Учесталост очитавања:** DHT сензори захтевају време између очитавања (најмање 2 секунде)
4. **Проверите кондензацију:** Може утицати на очитавања
5. **Квалитет сензора:** DHT11 је мање прецизан од DHT22

### Камера

#### Проблем: Камера није детектована на Raspberry Pi
**Грешка:** `mmal: mmal_vc_component_create: failed to create component 'vc.ril.camera'`

**Решење:**
1. **Омогућите интерфејс камере:**
   ```bash
   sudo raspi-config
   ```
   Идите у Interface Options → Camera → Enable
2. **Проверите лентовски кабл:** Уверите се да је кабл исправно уметнут
   - Плава страна окренута ка USB портовима на Pi Zero
   - Плава страна окренута од USB портова на Pi 4
3. **Ажурирајте фирмвер:**
   ```bash
   sudo apt update
   sudo apt full-upgrade
   sudo reboot
   ```
4. **Пробајте камеру:**
   ```bash
   raspistill -o test.jpg
   ```

#### Проблем: Слике са камере су лошег квалитета
**Симптоми:** Замењене, тамне или испране слике

**Решење:**
1. **Фокус:** Уклоните заштитни фолиј и по потреби подесите фокус ако је могуће
2. **Осветљење:** Обезбедите адекватно осветљење
3. **Поставке камере:** Подесите експозицију, ISO, баланс белог у коду
4. **Стабилност:** Држите камеру мирно, користите трипод ако је потребно
5. **Резолуција:** Немојте прелазити максималну резолуцију камере

### Микрофон и звучник

#### Проблем: Нема улаза/излаза звука
**Симптоми:** Микрофон не снима, звучник не репродукује звук

**Решење:**
1. **Проверите везе:** Уверите се да су аудио уређаји исправно повезани
2. **Тестирајте опрему:**
   - Звучник: `speaker-test -t wav -c 2`
   - Микрофон: `arecord -l` за листање, `arecord test.wav` за снимање
3. **Подешавања јачине звука:** Проверите и подесите јачину звука:
   ```bash
   alsamixer
   ```
4. **Одаберите аудио уређај:** Наведите исправан аудио уређај у коду
5. **Проблеми са драјвером:** Ажурирајте ALSA или поново инсталирајте аудио драјвере

#### Проблем: ReSpeaker хет не ради
**Симптоми:** Аудио уређај није детектован

**Решење:**
1. **Инсталирајте драјвере:**
   ```bash
   git clone https://github.com/HinTak/seeed-voicecard
   cd seeed-voicecard
   sudo ./install.sh
   sudo reboot
   ```
2. **Проверите инсталацију:** `arecord -l` треба да листа ReSpeaker
3. **Ажурирајте фирмвер:** Неке верзије Pi OS захтевају ажурирање драјвера
4. **Проверите седење:** Уверите се да је хет исправно причвршћен на GPIO пинове

## Проблеми у развојном окружењу

### VS Code

#### Проблем: Терминал аутоматски не активира виртуелно окружење
**Симптоми:** Терминал се отвара али venv није активиран

**Решење:**
1. **Подесите Python интерпретер:** Command Palette → "Python: Select Interpreter" → Изаберите venv
2. **Рестартујте VS Code** након одабира интерпретера
3. **Проверите подешавања:** У `settings.json` додajte:
   ```json
   "python.terminal.activateEnvironment": true
   ```

#### Проблем: Код се не покреће на уређају
**Симптоми:** Код се извршава али ништа се не дешава на уређају

**Решење:**
1. **Проверите да ли је код сачуван** (проверите тачку на картици фајла)
2. **Проверите који Python се користи:** `which python` или `where python`
3. **За Wio Terminal:** Осигурајте да је код отпремљен преко PlatformIO (кликните дугме за отпремање)
4. **За Raspberry Pi:** SSH-ујте се у Pi и тамо покрените код
5. **Проверите прозор са излазом** за грешке

#### Проблем: IntelliSense не приказује функције библиотеке
**Симптоми:** Нема аутокомплита за увезене модуле

**Решење:**
1. Уверите се да је библиотека инсталирана у текућем окружењу
2. Поново учитајте VS Code прозор
3. Проверите да је Python интерпретер исправан
4. Инсталирајте type stubs ако су доступни: `pip install types-<library-name>`

### Python виртуелна окружења

#### Проблем: Не може се направити виртуелно окружење
**Грешка:** `The virtual environment was not created successfully`

**Решење:**
1. **Инсталирајте venv модул:**
   - Ubuntu/Debian: `sudo apt install python3-venv`
   - macOS: Треба да буде укључен уз Python
   - Windows: Поново инсталирајте Python са свим компонентама
2. **Проверите Python инсталацију:** Уверите се да је Python исправно инсталиран
3. **Користите путању:** Покушајте `python3 -m venv .venv` са експлицитним позивом python3

#### Проблем: Пакети инсталирани на погрешно место
**Симптоми:** Грешка при увозу након инсталације пакета

**Решење:**
1. **Уверите се да је venv активиран:** У командном позиву треба да стоји `(.venv)`
2. **Проверите локацију pip:** `which pip` треба да показује на `.venv/bin/pip`
3. **Поново инсталирајте у venv:** Активирајте venv, па `pip install <package>`
4. **Не користите sudo са pip у вртелном окружењу**

#### Проблем: Виртуелно окружење није преносиво
**Симптоми:** Venv не ради након премештања или на другом рачунару

**Решење:**
1. **Не премештајте venv:** Обришите и направите ново на новој локацији
2. **Користите requirements.txt:**
   ```bash
   pip freeze > requirements.txt
   pip install -r requirements.txt
   ```
3. **Поново направите venv:**
   ```bash
   python3 -m venv .venv
   source .venv/bin/activate  # или activate.bat на Виндовс
   pip install -r requirements.txt
   ```

### Зависности

#### Проблем: Инсталација пакета не успева
**Грешка:** Различите pip грешке током инсталације

**Решење:**
1. **Ажурирајте pip:**
   ```bash
   pip install --upgrade pip
   ```
2. **Инсталирајте алате за компајлирање:**
   - Ubuntu/Debian: `sudo apt install build-essential python3-dev`
   - macOS: `xcode-select --install`
   - Windows: Инсталирајте Visual Studio Build Tools
3. **Проверите интернет конекцију**
4. **Пробајте други пакет индекс:** `pip install --index-url https://pypi.org/simple/ <package>`
5. **Инсталирајте одређену верзију:** `pip install <package>==<version>`

#### Проблем: Конфликти зависности
**Грешка:** `ERROR: pip's dependency resolver does not currently take into account all the packages that are installed`

**Решење:**
1. **Користите свеже виртуелно окружење** за сваки пројекат
2. **Ажурирајте пакете:** `pip install --upgrade <package>`
3. **Проверите захтеве:** Користите `pip check` за проналажење конфликата
4. **Инсталирајте компатибилне верзије:** Наведите опсеге верзија у requirements.txt

## Проблеми са перформансама

### Проблем: Код ради споро
**Симптоми:** Задржавања, истек времена, непокретно понашање

**Решење:**
1. **Смањите учесталост очитавања сензора:** Не очитавајте сензоре превише често
2. **Оптимизујте петље:** Избегавајте активно чекaње, користите sleep() или кашњења
3. **Проблеми са меморијом:**
   - Затворите непотребне апликације
   - Ослободите простор на складишту
   - Пратите са `top` или `htop` на Pi
4. **Брзина SD картице:** Користите бржу SD картицу или SSD за Raspberry Pi
5. **Мрежна кашњења:** Користите асинхроне операције за мрежне позиве

### Проблем: Грешке недостатка меморије
**Грешка:** `MemoryError` или замрзавање система

**Решење:**
1. **За Raspberry Pi:**
   - Затворите непотребне апликације
   - Повећајте swap простор
   - Користите лакши ОС (Lite верзије)
   - Надоградите RAM (Pi 4 има опције 2/4/8GB)
2. **За Wio Terminal:**
   - Смањите величине буфера
   - Коришћење мањих слика
   - Оптимизујте коришћење стрингова
   - Провери цурење меморије (неослобођена меморија)

### Проблем: Губитак или оштећење података
**Симптоми:** Недостају поруке, оштећени фајлови

**Решење:**
1. **Проблеми са SD картицом:**
   - Користите квалитетне SD картице (избегавајте јефтине/лажне)
   - Редовне резерне копије
   - Чисто искључивање (не искључујте струју нагло)
2. **Препуњен буфер:** Повећајте величине буфера у коду
3. **Поузданост мреже:** Имплементирајте поновна покушаја и обраду грешака
4. **Квалитет услуге:** Користите MQTT QoS 1 или 2 за важне поруке

## Честа порука о грешци

### `ModuleNotFoundError: No module named 'X'`
**Узрок:** Пакет није инсталиран или виртуелно окружење није активирано

**Решење:**
```bash
pip install X
```
Прво осигурајте да је виртуелно окружење активирано.

### `Permission denied` на Linux/macOS
**Узрок:** Потребне су повишене дозволе или проблем са дозволама фајла

**Решење:**
- За системске операције: Користите `sudo`
- За pip: НЕ користите sudo у виртуелном окружењу, прво активирајте venv
- За серијски порт: Додајте корисника у dialout групу: `sudo usermod -a -G dialout $USER`, па се излогујте и улогујте

### `OSError: [Errno 98] Address already in use`
**Узрок:** Порт већ користи други процес

**Решење:**
1. Пронађите процес који користи порт: `lsof -i :<port>` или `netstat -ano | findstr :<port>`
2. Убијте процес или користите други порт у вашем коду

### `SSL: CERTIFICATE_VERIFY_FAILED`
**Узрок:** Неуспешна провера SSL сертификата

**Решење:**
1. Ажурирајте сертификате: `pip install --upgrade certifi`
2. Проверите да ли је време на систему тачно: `date`
3. Само за развој (не продукцију): Онемогућите валидацију у коду

### `IndentationError: unexpected indent`
**Узрок:** Проблеми са увлачењем у Python-у (мешање табова/размака)

**Решење:**
1. Користите доследно увлачење (Python стандард је 4 размака)
2. Конфигуришите едитор да користи размаке уместо табова
3. VS Code: Подесите `"editor.insertSpaces": true` и `"editor.tabSize": 4`

### `UnicodeDecodeError` or `UnicodeEncodeError`
**Узрок:** Проблеми са кодирањем карактера

**Решење:**
```python
# При читању фајлова
with open('file.txt', 'r', encoding='utf-8') as f:
    content = f.read()

# При писању фајлова
with open('file.txt', 'w', encoding='utf-8') as f:
    f.write(content)
```
