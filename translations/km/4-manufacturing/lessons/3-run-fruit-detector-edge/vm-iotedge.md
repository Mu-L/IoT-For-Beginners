# បង្កើតម៉ាស៊ីនមេមួយដំណើរការជា IoT Edge

នៅក្នុង Azure អ្នកអាចបង្កើតម៉ាស៊ីនមេមួយ - គឺជា কম্পিউទ័រមួយនៅលើមេឃ ដែលអ្នកអាចកំណត់រចនាបថតាមដែលអ្នកចង់បាន ហើយរត់កម្មវិធីផ្ទាល់ខ្លួនរបស់អ្នកលើវា។

> 💁 អ្នកអាចអានបន្ថែមអំពីម៉ាស៊ីនមេនៅលើ [ទំព័រ Virtual Machine នៅលើវិគីភីឌា](https://wikipedia.org/wiki/Virtual_machine)។

## កិច្ចការជំហាន - កំណត់ម៉ាស៊ីនមេ IoT Edge មួយ

1. រត់ពាក្យបញ្ជាដូចខាងក្រោមដើម្បីបង្កើត VM ដែលមាន Azure IoT Edge បានដំឡើងរួចហើយ៖

    ```sh
    az deployment group create \
                --resource-group fruit-quality-detector \
                --template-uri https://raw.githubusercontent.com/Azure/iotedge-vm-deploy/1.2.0/edgeDeploy.json \
                --parameters dnsLabelPrefix=<vm_name> \
                --parameters adminUsername=<username> \
                --parameters deviceConnectionString="<connection_string>" \
                --parameters authenticationType=password \
                --parameters adminPasswordOrKey="<password>"
    ```

    ជំនួស `<vm_name>` ជាមួយឈ្មោះសម្រាប់ម៉ាស៊ីនមេនេះ។ វាត្រូវតែមានតែមួយនៅលើពិភពលោក, ដូច្នេះប្រើអ្វីមួយដូចជា `fruit-quality-detector-vm-` ជាមួយឈ្មោះរបស់អ្នក ឬ តម្លៃផ្សេងនៅចុង។

    ជំនួស `<username>` និង `<password>` ជាមួយឈ្មោះអ្នកប្រើ និង ពាក្យសម្ងាត់ដែលអ្នកចង់ប្រើសម្រាប់ចូលទៅ VM។ អ្នកត្រូវការសុវត្ថិភាពគ្រប់គ្រាន់ ដូច្នេះកុំប្រើ admin/password។

    ជំនួស `<connection_string>` ជាមួយខ្សែភ្ជាប់របស់ឧបករណ៍ IoT Edge `fruit-quality-detector-edge` របស់អ្នក។

    វានឹងបង្កើត VM ដែលបានកំណត់ជាម៉ាស៊ីនមេប្រភេទ `DS1 v2`។ ប្រភេទទាំងនេះបង្ហាញពីថាមពលរបស់ម៉ាស៊ីន ហើយបើយោងដល់វា តម្លៃរបស់វា។ VM នេះមាន CPU 1 និង RAM 3.5GB។

    > 💰 អ្នកអាចមើលតម្លៃបច្ចុប្បន្ននៃ VM ទាំងនេះនៅលើ [មគ្គុទេសក៍តម្លៃម៉ាស៊ីនមេនៅ Azure](https://azure.microsoft.com/pricing/details/virtual-machines/linux/?WT.mc_id=academic-17441-jabenn)

    ពេលដែល VM ត្រូវបានបង្កើតរួចហើយ កម្មវិធី IoT Edge runtime នឹងត្រូវបានដំឡើងដោយស្វ័យប្រវត្តិ ហើយបានកំណត់ឲ្យអ្នកភ្ជាប់ទៅកាន់ IoT Hub របស់អ្នកជាឧបករណ៍ `fruit-quality-detector-edge`។

1. អ្នកនឹងត្រូវការអាសយដ្ឋាន IP ឬឈ្មោះ DNS នៃ VM ដើម្បីហៅកម្មវិធីត្រួតពិនិត្យរូបភាពពីវា។ រត់ពាក្យបញ្ជាដូចខាងក្រោមដើម្បីទទួលបានវា៖

    ```sh
    az vm list --resource-group fruit-quality-detector \
               --output table \
               --show-details
    ```

    ចំលងតម្លៃពីកន្លែង `PublicIps` ឬ `Fqdns` ។

1. VM គឺមានការចំណាយ។ នៅពេលសរសេរនេះ, VM DS1 មានតម្លៃប្រហែល $0.06 ក្នុងមួយម៉ោង។ ដើម្បីកាត់បន្ថយការចំណាយ អ្នកគួរត្រូវបិទ VM នៅពេលមិនប្រើ ហើយលុបវាចេញនៅពេលបញ្ចប់គម្រោងនេះ។

    អ្នកអាចកំណត់ VM របស់អ្នកឲ្យបិទដោយស្វ័យប្រវត្តិនៅពេលម៉ោងជាក់លាក់រៀងរាល់ថ្ងៃ។ នេះមានន័យថាបើអ្នកភ្លេចបិទវា អ្នកនឹងមិនត្រូវបង់ប្រាក់ទាន់ពេលរហូតដល់ម៉ោងបិទដោយស្វ័យប្រវត្តិនោះឡើយ។ ប្រើពាក្យបញ្ជាខាងក្រោមដើម្បីកំណត់វា៖

    ```sh
    az vm auto-shutdown --resource-group fruit-quality-detector \
                        --name <vm_name> \
                        --time <shutdown_time_utc>
    ```

    ជំនួស `<vm_name>` ជាមួយឈ្មោះម៉ាស៊ីនមេរបស់អ្នក។

    ជំនួស `<shutdown_time_utc>` ជាមួយម៉ោង UTC ដែលអ្នកចង់ឲ្យ VM បិទ ដោយប្រើលេខ 4 ខ្ទង់ជាទ្រង់ទ្រាយ HHMM។ ឧទាហរណ៏ ប្រសិនបើអ្នកចង់បិទម៉ាស៊ីនវេលាចក្រិចម៉ោងនៃថ្ងៃ (midnight UTC) អ្នកត្រូវដាក់ `0000`។ សម្រាប់ម៉ោង 7:30 ល្ងាច នៅឆ្នេរផ្សេងគ្នា នៅអាមេរិក, ប្រើ 0230 (7:30 ល្ងាច នៅឆ្នេរពិភពប៉ាស៊ីហ្វិកប្រទេសអាមេរិក គឺម៉ោង 2:30 ព្រឹក UTC)។

1. កម្មវិធីត្រួតពិនិត្យរូបភាពរបស់អ្នកនឹងរត់លើឧបករណ៍ edge នេះ ដើម្បីស្តាប់នៅព្រទ្បទ្វា 80 (ព្រទ្បទ្វ HTTP ស្តង់ដា)។ នៅលំនាំដើម, ម៉ាស៊ីនមេមានការបិទព្រទ្បទ្វចេញមុខ, ដូចនេះ អ្នកត្រូវតែបើកព្រទ្បទ្វ 80។ ការបើកព្រទ្បទ្វត្រូវធ្វើនៅក្នុងក្រុមសន្តិសុខបណ្តាញ (network security group), ដូច្នេះអ្នកត្រូវដឹងឈ្មោះក្រុមសន្តិសុខបណ្តាញសម្រាប់ VM របស់អ្នកជាមុន សម្រាប់រកឃើញវា សូមប្រើពាក្យបញ្ជាខាងក្រោម៖

    ```sh
    az network nsg list --resource-group fruit-quality-detector \
                        --output table
    ```

    ចម្លងតម្លៃក្នុងកន្លែង `Name`។

1. រត់ពាក្យបញ្ជាដូចខាងក្រោមដើម្បីបន្ថែមច្បាប់បើកព្រទ្បទ្វ 80 សម្រាប់ក្រុមសន្តិសុខបណ្តាញ៖

    ```sh
    az network nsg rule create \
                        --resource-group fruit-quality-detector \
                        --name Port_80 \
                        --protocol tcp \
                        --priority 1010 \
                        --destination-port-range 80 \
                        --nsg-name <nsg name>
    ```

    ជំនួស `<nsg name>` ជាមួយឈ្មោះក្រុមសន្តិសុខបណ្តាញពីជំហានមុន។

### កិច្ចការជំហាន - គ្រប់គ្រង VM របស់អ្នកដើម្បីកាត់បន្ថយថ្លៃដើម

1. នៅពេលអ្នកមិនប្រើ VM អ្នកគួរតែបិទវា។ ដើម្បីបិទ VM សូមប្រើពាក្យបញ្ជាខាងក្រោម៖

    ```sh
    az vm deallocate --resource-group fruit-quality-detector \
                     --name <vm_name>
    ```

    ជំនួស `<vm_name>` ជាមួយឈ្មោះម៉ាស៊ីនមេរបស់អ្នក។

    > 💁 មានពាក្យបញ្ជា `az vm stop` ដែលនឹងបិទ VM ប៉ុន្តែវាក្នុងស្ថានភាពរក្សាទុក ជាហេតុដែលអ្នកបន្តបង់ប្រាក់ លួចបិទវាដោយពុំដកពីការចំណាយ។

1. ដើម្បីបើក VM ម្តងទៀត សូមប្រើពាក្យបញ្ជាខាងក្រោម៖

    ```sh
    az vm start --resource-group fruit-quality-detector \
                --name <vm_name>
    ```

    ជំនួស `<vm_name>` ជាមួយឈ្មោះម៉ាស៊ីនមេរបស់អ្នក។
