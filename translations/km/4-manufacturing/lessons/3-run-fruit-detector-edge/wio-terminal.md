# ចាត់ថ្នាក់រូបភាពដោយប្រើកម្មវិធីចាត់ថ្នាក់រូបភាពផ្អែកលើ IoT Edge - Wio Terminal

នៅក្នុងផ្នែកនេះនៃមេរៀន អ្នកនឹងប្រើកម្មវិធីចាត់ថ្នាក់រូបភាពដែលដំណើរការនៅលើឧបករណ៍ IoT Edge។

## ប្រើកម្មវិធីចាត់ថ្នាក់ IoT Edge

ឧបករណ៍ IoT អាចត្រូវបានបញ្ជូនទៅប្រើកម្មវិធីចាត់ថ្នាក់រូបភាព IoT Edge។ URL សម្រាប់កម្មវិធីចាត់ថ្នាក់រូបភាពគឺ `http://<IP address or name>/image` ដោយជំនួស `<IP address or name>` ជាមួយអាសយដ្ឋាន IP ឬឈ្មោះម៉ាស៊ីនគណនេយ្យដែលដំណើរការលើ IoT Edge។

### ភារកិច្ច - ប្រើកម្មវិធីចាត់ថ្នាក់ IoT Edge

1. បើកគម្រោងកម្មវិធី `fruit-quality-detector` ប្រសិនបើវានៅមិនបានបើក។

1. កម្មវិធីចាត់ថ្នាក់រូបភាពដំណើរការជា REST API ដោយប្រើ HTTP មិនមែន HTTPS ដូច្នេះការហៅត្រូវតែប្រើឧបករណ៍ WiFi client ដែលដំណើរការជាមួយការហៅ HTTP តែប៉ុណ្ណោះ។ នេះមានន័យថាគ្មានការទាមទារសញ្ញាបត្រ។ លុប `CERTIFICATE` ចេញពីឯកសារ `config.h`។

1. URL ទស្សន៍ទាយនៅក្នុងឯកសារ `config.h` ត្រូវតែបានធ្វើបច្ចុប្បន្នភាពទៅ URL ថ្មី។ អ្នកអាចលុប `PREDICTION_KEY` ដែរ ព្រោះវាមិនត្រូវការទេ។

    ```cpp
    const char *PREDICTION_URL = "<URL>";
    ```

    ជំនួស `<URL>` ជាមួយ URL សម្រាប់កម្មវិធីចាត់ថ្នាក់រូបភាពរបស់អ្នក។

1. នៅក្នុង `main.cpp` ប្តូរបទបញ្ជានាំចូលសម្រាប់ WiFi Client Secure ទៅនាំចូលសម្រាប់ HTTP ជាទម្រង់ស្តង់ដារ៖

    ```cpp
    #include <WiFiClient.h>
    ```

1. ប្តូរពាក្យប្រកាស `WiFiClient` ទៅជាទម្រង់ HTTP ៖

    ```cpp
    WiFiClient client;
    ```

1. ជ្រើសខ្សែបណ្តា​ដែលកំណត់សញ្ញាបត្រលើ WiFi client ហើយលុបខ្សែបណ្តា `client.setCACert(CERTIFICATE);` ចេញពីមុខងារ `connectWiFi`។

1. នៅក្នុងមុខងារ `classifyImage` លុបខ្សែបណ្តា `httpClient.addHeader("Prediction-Key", PREDICTION_KEY);` ដែលកំណត់ key ទស្សន៍ទាយនៅក្នុង header។

1. បញ្ចូល និងរត់កម្មវិធីរបស់អ្នក។ បង្ហាញកាមេរ៉ាទៅលើផ្លែប៉ះមួយចំនួន ហើយចុចប៊ូតុង C។ អ្នកនឹងឃើញលទ្ធផលនៅក្នុងម៉ូនីទ័រច្រកសរសេរ៖

    ```output
    Connecting to WiFi..
    Connected!
    Image captured
    Image read to buffer with length 8200
    ripe:   56.84%
    unripe: 43.16%
    ```
