# ಪ್ರಶ್ನೋತ್ತರಗಳು

ಈ ಪ್ರಶ್ನೋತ್ತರಗಳು https://aka.ms/iot-beginners ನಲ್ಲಿ IoT for Beginners ಪಾಠಕ್ರಮದ ಮೊದಲು ಮತ್ತು ನಂತರದ ಪ್ರಶ್ನೋತ್ತರಗಳಾಗಿವೆ

## ಯೋಜನೆ ಸജ್ಜುಗೊಳಿಕೆ

```
npm install
```

### ಡೆವಲಪ್‌ಮೆಂಟ್‌ಗಾಗಿ ಸಂಪಾದಿಸಿ ಮತ್ತು ಹಾಟ್-ರೀಲಾಗಿಸಿ

```
npm run serve
```

### ಉತ್ಪಾದನೆಗಾಗಿ ಕಾಂಪೈಲ್ ಮಾಡಿ ಮತ್ತು ಕನಿಷ್ಠಗೊಳಿಸಿ

```
npm run build
```

### ಫೈಲ್‌ಗಳನ್ನು ಲಿಂಟ್ ಮಾಡಿ ಮತ್ತು ಸರಿಪಡಿಸಿ

```
npm run lint
```

### ಸಂರಚನೆಯನ್ನು ಕಸ್ಟಮೈಸ್ ಮಾಡಿ

[Configuration Reference](https://cli.vuejs.org/config/) ನೋಡಿ.

ಕ್ರೆಡಿಟ್ಸ್: ಈ ಪ್ರಶ್ನೋತ್ತರ ಅಪ್ಲಿಕೇಶನ್ನಿನ ಮೂಲ ಆವೃತ್ತಿಗೆ ಧನ್ಯವಾದಗಳು: https://github.com/arpan45/simple-quiz-vue


## ಅಜೂರ್‌ಗೆ ನಿಯೋಜನೆ

ಈಗಾಗಲೇ ಪ್ರಾರಂಭಿಸಲು ನಿಮಗೆ ಸಹಾಯ ಮಾಡುವ ಹಂತ-ಹಂತ ಮಾರ್ಗದರ್ಶಿ ಇಲ್ಲಿ ಇದೆ:

1. GitHub ರೆಪೊಸಿಟೋರಿಯನ್ನು ಫೋರ್ಕ್ ಮಾಡಿ
ನಿಮ್ಮ ಸ್ಥಿರ ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್ ಕೋಡ್ ನಿಮ್ಮ GitHub ರೆಪೊಸಿಟೋರಿಯಲ್ಲಿದೆ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ. ಈ ರೆಪೊಸಿಟೋರಿಯನ್ನು ಫೋರ್ಕ್ ಮಾಡಿ.

2. ಅಜೂರ್ ಸ್ಥಿರ ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್ ರಚಿಸಿ
- [ಅಜೂರ್ ಖಾತೆ](http://azure.microsoft.com) ರಚಿಸಿ
- [ಅಜೂರ್ ಪೋರ್ಟಲ್](https://portal.azure.com)ಗೆ ಹೋಗಿ
- “Create a resource” ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ ಮತ್ತು “Static Web App” ಅನ್ನು ಹುಡುಕಿ.
- “Create” ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ.

3. ಸ್ಥಿರ ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ಸಂರಚಿಸಿ
- ಮೂಲಭೂತ: ಸಬ್ಸ್ಕ್ರಿಪ್ಷನ್: ನಿಮ್ಮ ಅಜೂರ್ ಸಬ್ಸ್ಕ್ರಿಪ್ಷನ್ ಆಯ್ಕೆಮಾಡಿ.
- ರಿಸೋರ್ಸ್ ಗ್ರೂಪ್: ಹೊಸ ರಿಸೋರ್ಸ್ ಗ್ರೂಪ್ ರಚಿಸಿ ಅಥವಾ ಈಗಿರುವದೊಂದನ್ನು ಬಳಸಿ.
- ಹೆಸರು: ನಿಮ್ಮ ಸ್ಥಿರ ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್ಗೆ ಹೆಸರು ನೀಡಿ.
- ಪ್ರದೇಶ: ನಿಮ್ಮ ಬಳಕೆದಾರರಿಗೆ ಅಧಿಕ ಸಂಖ್ಯೆಯ ಪ್ರದೇಶವನ್ನು ಆಯ್ಕೆಮಾಡಿ.

- #### ನಿಯೋಜನೆ ವಿವರಗಳು:
- ಮೂಲ: “GitHub” ಆಯ್ಕೆಮಾಡಿ.
- GitHub ಖಾತೆ: ಅಜೂರ್‌ಗೆ ನಿಮ್ಮ GitHub ಖಾತೆಗೆ ಪ್ರವೇಶಕ್ಕೆ ಅನುಮತಿಸಿ.
- ಸಂಸ್ಥೆ: ನಿಮ್ಮ GitHub ಸಂಸ್ಥೆಯನ್ನು ಆಯ್ಕೆಮಾಡಿ.
- ರೆಪೊಸಿಟರಿ: ನಿಮ್ಮ ಸ್ಥಿರ ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್ ಇರುವ ರೆಪೊಸಿಟೋರಿಯನ್ನು ಆಯ್ಕೆಮಾಡಿ.
- ಶಾಖೆ: ನೀವು ನಿಯೋಜಿಸಲು ಬಯಸುವ ಶಾಖೆಯನ್ನು ಆಯ್ಕೆಮಾಡಿ.

- #### ನಿರ್ಮಾಣ ವಿವರಗಳು:
- ನಿರ್ಮಾಣ ಪೂರ್ವನಿಯಮಗಳು: ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ನಿರ್ಮಾಣವಾಗಿರುವ ಫ್ರೇಮ್ವರ್ಕ್ ಆಯ್ಕೆಮಾಡಿ (ಉದಾ., React, Angular, Vue ಇತ್ಯಾದಿ).
- ಅಪ್ಲಿಕೇಶನ್ ಸ್ಥಳ: ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಕೋಡ್ ಇರುವ ಫೊಲ್ಡರ್ ನಮೂದಿಸಿ (ಉದಾ., ರುಟ್‌ಗೆ ಇದ್ದರೆ /).
- API ಸ್ಥಳ: ನೀವು API ಹೊಂದಿದ್ದರೆ ಅದರ ಸ್ಥಳವನ್ನು ನಮೂದಿಸಿ (ಐಚ್ಛಿಕ).
- ಔಟ್‌ಪುಟ್ ಸ್ಥಳ: ನಿರ್ಮಾಣ ಔಟ್‌ಪುಟ್ ಜನರೇಟ್ ಆಗುವ ಫೊಲ್ಡರ್ ಅನ್ನು ನಮೂದಿಸಿ (ಉದಾ., build ಅಥವಾ dist).

4. ಪರಿಶೀಲಿಸಿ ಮತ್ತು ರಚಿಸಿ
ನಿಮ್ಮ ಸೆಟ್ಟಿಂಗ್ಸ್ ಪರಿಶೀಲಿಸಿ ಮತ್ತು “Create” ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ. ಅಜೂರ್ ಅಗತ್ಯವಿರುವ ಸಂಪನ್ಮೂಲಗಳನ್ನು ರಚಿಸಿ ಮತ್ತು ನಿಮ್ಮ ರೆಪೊಸಿಟೋರಿಯಲ್ಲಿ GitHub Actions ವರ್ಕ್‌ಫ್ಲೋವನ್ನು ನಿರ್ಮಿಸುತ್ತದೆ.

5. GitHub Actions ವರ್ಕ್‌ಫ್ಲೋ
ಅಜೂರ್ ನಿಮ್ಮ ರೆಪೊಸಿಟೋರಿಯಲ್ಲಿ ಸ್ವಯಂಚಾಲಿತವಾಗಿ GitHub Actions ವರ್ಕ್‌ಫ್ಲೋ ಫೈಲ್ ರಚಿಸುತ್ತದೆ (.github/workflows/azure-static-web-apps-<name>.yml). ಈ ವರ್ಕ್‌ಫ್ಲೋ ನಿರ್ಮಾಣ ಮತ್ತು ನಿಯೋಜನೆ ಪ್ರಕ್ರಿಯೆಯನ್ನು ನಿರ್ವಹಿಸುತ್ತದೆ.

6. ನಿಯೋಜನೆ ಮೇಲ್ವಿಚಾರಣೆ ಮಾಡಿ
ನಿಮ್ಮ GitHub ರೆಪೊಸಿಟೋರಿಯಲ್ಲಿ “Actions” ತ್ಯಾಬ್‌ಗೆ ಹೋಗಿ.
ನೀವು ಒಂದು ವರ್ಕ್‌ಫ್ಲೋ ಚಲಿಸುತ್ತಿರುವುದನ್ನು ನೋಡಬಹುದು. ಈ ವರ್ಕ್‌ಫ್ಲೋ ನಿಮ್ಮ ಸ್ಥಿರ ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ಅಜೂರ್‌ಗೆ ನಿರ್ಮಿಸಿ ನಿಯೋಜಿಸುತ್ತದೆ.
ವರ್ಕ್‌ಫ್ಲೋ ಪೂರ್ಣಗೊಂಡ ನಂತರ, ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಒದಗಿಸಲಾದ ಅಜೂರ್ URL ನಲ್ಲಿ ಲೈವ್ ಆಗಿರುತ್ತದೆ.

### ಉದಾಹರಣೆಯ ವರ್ಕ್‌ಫ್ಲೋ ಫೈಲ್

GitHub Actions ವರ್ಕ್‌ಫ್ಲೋ ಫೈಲ್ ಹೇಗಿರಬಹುದು ಎಂಬ ಉದಾಹರಣೆ ಇಲ್ಲಿದೆ:
name: Azure Static Web Apps CI/CD
```
on:
  push:
    branches:
      - main
  pull_request:
    types: [opened, synchronize, reopened, closed]
    branches:
      - main

jobs:
  build_and_deploy_job:
    runs-on: ubuntu-latest
    name: Build and Deploy Job
    steps:
      - uses: actions/checkout@v2
      - name: Build And Deploy
        id: builddeploy
        uses: Azure/static-web-apps-deploy@v1
        with:
          azure_static_web_apps_api_token: ${{ secrets.AZURE_STATIC_WEB_APPS_API_TOKEN }}
          repo_token: ${{ secrets.GITHUB_TOKEN }}
          action: "upload"
          app_location: "quiz-app" #App source code path
          api_location: ""API source code path optional
          output_location: "dist" #Built app content directory - optional
```
