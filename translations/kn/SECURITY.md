# ಭದ್ರತೆ

Microsoft ನಮ್ಮ ಸಾಫ್ಟ್ವೇರ್ ಉತ್ಪನ್ನಗಳು ಮತ್ತು ಸೇವೆಗಳ ಭದ್ರತೆಯನ್ನು ಗಂಭೀರವಾಗಿ ತೆಗೆದುಕೊಳ್ಳುತ್ತದೆ, ಇದರಲ್ಲಿ ನಮ್ಮ GitHub ಸಂಸ್ಥೆಗಳ ಮೂಲಕ ನಿರ್ವಹಿಸಲಾದ ಎಲ್ಲಾ ಮೂಲಕೋಡ್ ರೆಪೊಸಿಟರಿಗಳು ಸೇರಿವೆ, ಅವುಗಳಲ್ಲಿ [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet), [Xamarin](https://github.com/xamarin), ಮತ್ತು [ನಮ್ಮ GitHub ಸಂಸ್ಥೆಗಳು](https://opensource.microsoft.com/) ಸೇರಿವೆ.

ನೀವು ಯಾವುದೇ Microsoft-ಜವಾಬ್ದಾರಿಯಲ್ಲಿರುವ ರೆಪೊಸಿಟರಿಯಲ್ಲಿ [Microsoft ನ ಭದ್ರತಾ ದುರ್ಬಲತೆ ವಿವರಣೆ](https://docs.microsoft.com/en-us/previous-versions/tn-archive/cc751383(v=technet.10))ಗೆ ತಕ್ಕ ಭದ್ರತಾ ದುರ್ಬಲತೆಯನ್ನು ಕಂಡುಹಿಡಿದಿದ್ದೀರಾ ಎಂದು ಭಾವಿಸಿದರೆ, ದಯವಿಟ್ಟು ಕೆಳಗಿನಂತೆ ವರದಿ ಮಾಡಿ.

## ಭದ್ರತಾ ಸಮಸ್ಯೆಗಳನ್ನು ವರದಿ ಮಾಡುವುದು

**ದಯವಿಟ್ಟು ಭದ್ರತಾ ದುರ್ಬಲತೆಗಳನ್ನು ಸಾರ್ವಜನಿಕ GitHub ಸಮಸ್ಯೆಗಳ ಮೂಲಕ ವರದಿ ಮಾಡಬೇಡಿ.**

ಅದನ್ನು ಬದಲಿಗೆ Microsoft Security Response Center (MSRC) ಗೆ [https://msrc.microsoft.com/create-report](https://msrc.microsoft.com/create-report) ನಲ್ಲಿ ವರದಿ ಮಾಡಿ.

ನೀವು ಲಾಗಿನ್ ಮಾಡಲು ಇಚ್ಛೆಯಾಗದಿದ್ದರೆ, ದಯವಿಟ್ಟು [secure@microsoft.com](mailto:secure@microsoft.com) ಗೆ ಇಮೇಲ್ ಕಳುಹಿಸಿ. ಸಾಧ್ಯವಾದರೆ, ನಮ್ಮ PGP ಕೀಲಿಯನ್ನು ಬಳಸಿ ನಿಮ್ಮ ಸಂದೇಶವನ್ನು ಕ್ಲಿಪ್ಟೋ ಮಾಡಿ; ದಯವಿಟ್ಟು ಅದನ್ನು [Microsoft Security Response Center PGP ಕೀ ಪೇಜ್](https://www.microsoft.com/en-us/msrc/pgp-key-msrc) ನಿಂದ ಡೌನ್‌ಲೋಡ್ ಮಾಡಿ.

ನೀವು 24 ಗಂಟೆಗಳಲ್ಲಿ ಪ್ರತಿಕ್ರಿಯೆ ಪಡೆಯುವಿರಿ. ಯಾವುದೋ ಕಾರಣಕ್ಕಾಗಿ ನೀವು ಪ್ರತ್ಯುತ್ತರ ಪಡೆಯದಿದ್ದರೆ, ದಯವಿಟ್ಟು ನಿಮ್ಮ ಮೂಲ ಸಂದೇಶ ಸ್ವೀಕಾರವನ್ನು ಖಚಿತಪಡಿಸಲು ಇಮೇಲ್ ಮೂಲಕ ಅನುಸರಿಸಿ. ಹೆಚ್ಚುವರಿ ಮಾಹಿತಿಯನ್ನು [microsoft.com/msrc](https://www.microsoft.com/msrc) ನಲ್ಲಿ ನೋಡಬಹುದು.

ದಯವಿಟ್ಟು ಸಾಧ್ಯವಾದಷ್ಟು ಕೆಳಗಿನ ವಿನಂತಿ ಮಾಹಿತಿಯನ್ನು ಸೇರಿಸಿ, ಇದರಿಂದ ನಾವು ಸಮಸ್ಯೆಯ ಸ್ವರೂಪ ಮತ್ತು ವ್ಯಾಪ್ತಿಯನ್ನು ಉತ್ತಮವಾಗಿ ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು ಸಹಾಯವಾಗುತ್ತದೆ:

  * ಸಮಸ್ಯೆಯ ಪ್ರಕಾರ (ಉದಾಹರಣೆ: ಬಫರ್ ಓವರ್‌ಫ್ಲೋ, SQL ಇಂಜೆಕ್ಷನ್, ಕ್ರಾಸ್-ಸೈಟ್ ಸ್ಕ್ರಿಪ್ಟಿಂಗ್, ಇತ್ಯಾದಿ)
  * ಸಮಸ್ಯೆಯ ಆಗಮನಕ್ಕೆ ಸಂಬಂಧಿಸಿದ ಮೂಲ ಕಡತದ ಸಂಪೂರ್ಣ ದಾರಿಗಳು
  * ಪ್ರಭಾವಿತ ಮೂಲಕೋಡ್ ಸ್ಥಳ (ಟ್ಯಾಗ್/ಬ್ರಾಂಚ್/ಕಮಿಟ್ ಅಥವಾ ನೇರ URL)
  * ಸಮಸ್ಯೆ ಪುನರಾವರ್ತಿಸಲು ಯಾವುದೇ ವಿಶೇಷ ಸಂರಚನೆ ಅಗತ್ಯವಿದ್ದರೆ
  * ಸಮಸ್ಯೆ ಪುನರಾವರ್ತಿಸಲು ಹಂತ ಹಂತದ ಸೂಚನೆಗಳು
  * ಸಾದ್ಯವಾದರೆ ಪ್ರೂಫ್-ಆಫ್-ಕಾನ್ಸೆಪ್ಟ್ ಅಥವಾ ಎಕ್ಸ್‌ಪ್ಲಾಯಿಟ್ ಕೋಡ್
  * ಸಮಸ್ಯೆಯ ಪರಿಣಾಮ, ಹಳಿ ಸರಿಸುಮುರುಕಾಗಿರುವ ವ್ಯಕ್ತಿ ಹೇಗೆ ಅದನ್ನು ದುರುಪಯೋಗ ಮಾಡಬಹುದು ಎಂಬುದು

ಈ ಮಾಹಿತಿ ನಮ್ಮ ವರದಿಯನ್ನು ವೇಗವಾಗಿ ತಪಾಸಿಸಲು ಸಹಾಯ ಮಾಡಲಿದೆ.

ನೀವು ಬಗ್ ಬೌಂಟಿಗಾಗಿ ವರದಿ ಮಾಡುತ್ತಿದ್ದರೆ, ಪೂರ್ಣ ವರದಿಗಳು ಹೆಚ್ಚಿನ ಬೌಂಟಿ ಬಹುಮಾನಕ್ಕೆ ಸಹಾಯ ಮಾಡಬಹುದು. ದಯವಿಟ್ಟು ನಮ್ಮ [Microsoft Bug Bounty Program](https://microsoft.com/msrc/bounty) ಪುಟವನ್ನು ನೋಡಿರಿ, ನಮ್ಮ ಸಕ್ರಿಯ ಕಾರ್ಯಕ್ರಮಗಳ ಕುರಿತು ಹೆಚ್ಚು ವಿವರಗಳಿಗೆ.

## ಆದ್ಯತೆಯ ಭಾಷೆಗಳು

ನಾವು ಎಲ್ಲಾ ಸಂವಹನಗಳನ್ನು ಇಂಗ್ಲಿಷ್ ಭಾಷೆಯಲ್ಲಿ ಇಚ್ಛಿಸುತ್ತೇವೆ.

## ನೀತಿ

Microsoft [ಸಂಯೋಜಿತ ದುರ್ಬಲತೆ ಬಹಿರ್ಗಮನ](https://www.microsoft.com/en-us/msrc/cvd) ತತ್ತ್ವವನ್ನು ಅನುಸರಿಸುತ್ತದೆ.
