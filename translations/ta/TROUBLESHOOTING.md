## நிறுவல் சிக்கல்கள்

### Python நிறுவல்

#### சிக்கல்: Python பதிப்பு பழையது
**பிழை:** `Python 3.6 அல்லது அதற்கு மேற்பட்ட பதிப்பு தேவையானது`

**தீர்வு:**
1. [python.org](https://www.python.org/downloads/) எனும் தளத்திலிருந்து லேட்டஸ்ட் Python 3 ஐ பதிவிறக்கு.
2. விண்டோஸ் நிறுவலின் போது "Add Python to PATH" என்பதைக் குறிக்கவும்.
3. நிறுவலை சரிபார்க்கவும்:
   ```bash
   python3 --version
   ```

#### சிக்கல்: பல Python பதிப்புகள் மோதல் ஏற்படுத்துகின்றன
**அலெர்ட்:** தவறான Python பதிப்பு இயங்குகிறது, பாக்கேஜ்கள் தவறான இடத்துக்கு நிறுவப்படுகின்றன

**தீர்வு:**
- **விண்டோஸ்:** Python 3 ஐ அழைக்க `python` பதிலாக `py -3` பயன்படுத்தவும்
- **macOS/Linux:** `python` பதிலாக `python3` பயன்படுத்தவும்
- எப்போதும் திட்டங்களுக்கான மெய்நிகர் சூழலை உருவாக்கி பயன்படுத்தவும்

#### சிக்கல்: pip கட்டளை காணவில்லை
**பிழை:** `'pip' என்பது உள்ளக அல்லது புற அளவீட்டுக் கட்டளையாக கருதப்படவில்லை`

**தீர்வு:**
1. `pip`ன பதிலாக `pip3` முயற்சிக்கவும்
2. அல்லது `python -m pip` அல்லது `python3 -m pip` பயன்படுத்தவும்
3. Python PATH இல் சேர்க்கப்பட்டுள்ளதா என சரிபார்க்கவும் (Python ஐ மீண்டும் நிறுவி அந்த விருப்பத்தைத் தேர்வு செய்யவும்)

### VS குறியீடு மற்றும் நீட்டிப்புகள்

#### சிக்கல்: Pylance நீட்டிப்பு செயல்படவில்லை
**அலெர்ட்:** Python IntelliSense, code completion அல்லது type checking இல்லை

**தீர்வு:**
1. VS Code கட்டளை பட்டியலில் செல்லவும் (`Ctrl+Shift+P` அல்லது `Cmd+Shift+P`)
2. "Python: Select Interpreter" செயலியை இயக்கவும்
3. சரியான Python interpreter (மெய்நிகர் சூழல் இருந்தால் அதனை) தேர்ந்தெடுக்கவும்
4. VS Code விண்டோவை மீண்டு ஏற்றவும்

#### சிக்கல்: VS Code மெய்நிகர் சூழலை கண்டறியவில்லை
**அலெர்ட்:** தவறான Python interpreter தேர்ந்தெடுக்கப்பட்டுள்ளது

**தீர்வு:**
1. டெர்மினலில் மெய்நிகர் சூழலை இயக்கியுள்ளீர்களா என உறுதி செய்யவும்
2. கட்டளை பட்டியலில் "Python: Select Interpreter" செயலியை இயக்கவும்
3. `.venv` ஃபோல்டரில் இருந்த interpreter ஐ தேர்ந்தெடுக்கவும்
4. நிலை பட்டியில் (கீழே இடது பக்கம்) சரியான Python பதிப்பு காட்டப்படுகிறதா எனப் பார்

### PlatformIO (Wio Terminal)

#### சிக்கல்: PlatformIO நிறுவல் தோல்வி
**பிழை:** PlatformIO நிறுவலின் போது பல்வேறு பிழைகள்

**தீர்வு:**
1. VS Code புதுப்பிக்கப்பட்டதா என உறுதி செய்யவும்
2. முதலில் C/C++ நீட்டிப்பை நிறுவவும்
3. PlatformIO நிறுவியபின் VS Code ஐ மீண்டும் துவக்கவும்
4. உங்களுடைய இணைய இணைப்பைச் சரிபார்க்கவும் (PlatformIO பெரிய கோப்புகளை பதிவிறக்கும்)

#### சிக்கல்: PlatformIO மூலம் போர்ட்டு கண்டறியப்படவில்லை
**அலெர்ட்:** Wio Terminalக்கு குறியீடு பதிவேற்ற முடியவில்லை

**தீர்வு:**
1. வேறு USB கேபிள் ஒன்று முயற்சிக்கவும் (சில கேபிள்கள் மட்டும் சார்ஜ் செய்கின்றன)
2. சாதன மேலாளர் (விண்டோஸ்) அல்லது `ls /dev/tty*` (macOS/Linux) பயன்படுத்தி சரிபார்க்கவும்
3. USB டிரைவர்களை நிறுவ அல்லது புதுப்பிக்கவும்
4. வேறு USB போர்டு முயற்சிக்கவும்
5. Wio Terminal இல் பவர் ஸ்விட்சை விரைவில் இரண்டு முறை ஸ்லைடு செய்து பூட்ட்லோடர் முறையில் நுழையவும்

#### சிக்கல்: PlatformIO இல் கொடுப்பனவு பிழைகள்
**பிழை:** `fatal error: Arduino.h: No such file or directory`

**தீர்வு:**
1. உங்கள் திட்டத்தில் உள்ள `.pio` கோப்புறையை நீக்கு
2. கட்டளை பட்டியலில் இருந்து "PlatformIO: Rebuild" ஐ இயக்கவும்
3. `platformio.ini` இல் சரியான போர்ட் அமைப்பை உறுதி செய்யவும்:
   ```ini
   [env:seeed_wio_terminal]
   platform = atmelsam
   board = seeed_wio_terminal
   framework = arduino
   ```

### Grove நூலகங்கள்

#### சிக்கல்: ராஸ்பெரி பையில் Grove நூலகம் இறக்குமதி தோல்வி
**பிழை:** `ModuleNotFoundError: No module named 'grove'`

**தீர்வு:**
1. Grove நூலகங்களை மீண்டும் நிறுவவும்:
   ```bash
   cd ~
   git clone https://github.com/Seeed-Studio/grove.py
   cd grove.py
   sudo pip3 install .
   ```
2. மெய்நிகர் சூழல் பயன்படுத்தினால், பொதுவாக நிறுவ அல்லது நூலகங்களை நகலெடுக்க வேண்டியிருப்பது
3. I2C இயக்கப்பட்டுள்ளது என உறுதி செய்யவும்: `sudo raspi-config nonint do_i2c 0`

#### சிக்கல்: Grove சென்சார் கண்டறியப்படவில்லை
**பிழை:** `IOError: [Errno 121] Remote I/O error`

**தீர்வு:**
1. உடல் இணைப்புகளைச் சரிபார்க்கவும் (Grove கேபிள் முழுமையாக இணைக்கப்பட்டுள்ளதா)
2. சென்சார் சரியான போர்ட்டில் (அனலைக், டிஜிட்டல், I2C, UART) இணைக்கப்பட்டுள்ளதா என பாருங்க
3. `i2cdetect -y 1` இயக்கி சாதனம் I2C பேருந்தில் தோன்றுகிறதா பாருங்கள்
4. வேறு Grove கேபிள் முயற்சி செய்யவும்
5. Grove Base Hat ராஸ்பெரி பை GPIO பின்களில் சரியாக அமர்த்தப்பட்டுள்ளதா என உறுதி செய்யவும்

### ராஸ்பெரி பை

#### சிக்கல்: ராஸ்பெரி பை பூட் ஆகவில்லை
**அலெர்ட்:** திரை இல்லை, LED செயல்பாடு இல்லை அல்லது வானவில் திரை

**தீர்வு:**
1. **பவர் சப்ளை சரிபார்க்கவும்:** Pi 4க்கு அதிகாரப்பூர்வமான 5V 3A USB-C பவர் சப்ளை பயன்படுத்தவும்
2. **SD கார்ட் சிக்கல்கள்:** 
   - SD கார்ட்டை மறுகட்டமைத்து Raspberry Pi OS ஐ மீண்டும் நிறுவவும்
   - வேறு SD கார்ட்டை முயற்சி செய்யவும் ( பரிந்துரைக்கப்படும் பிராண்டுகளைப் பயன்படுத்தவும் )
   - SD கார்ட் சரியாக நுழையட்டுள்ளது என உறுதி செய்யவும்
3. **HDMI இணைப்பைச் சரிபார்க்கவும்:** Pi 4இன் இரு HDMI போர்ட்டையும் முயற்சி செய்து, பவருக்கு அருகிலுள்ள போர்ட்டைப் பயன்படுத்தவும்

#### சிக்கல்: Raspberry Piக்கு SSH இணைக்க முடியவில்லை
**அலெர்ட்:** இணைப்பு மறுக்கப்பட்டது அல்லது நேரம் முடிந்து விட்டது

**தீர்வு:**
1. SSHஐ இயக்கவும்:
   - Raspberry Pi Imager பயன்படுத்தி SD கார்டை பிளாஷ் செய்யும்போது SSHஐ மேம்பட்ட விருப்பங்களில் அசைவு செய்யவும்
   - அல்லது பூட் பகுதியிலேயே `ssh` என்ற பெயரிடப்பட்ட காலி கோப்பை உருவாக்கவும் (எxtension இல்லாமல்)
2. Piயின் IP முகவரியை கண்டறியவும்:
   - உங்கள் ரௌட்டரின் இணைக்கப்பட்ட சாதனங்களை சரிபார்க்கவும்
   - `ping raspberrypi.local` (mDNS இயங்குமானால்) ஓட்டவும்
   - `nmap` அல்லது Angry IP Scanner போன்ற நெட்வொர்க் ஸ்கேனிங் கருவிகளைப் பயன்படுத்தவும்
3. நெட்வொர்க்கை சரிபார்க்கவும்:
   - Pi மற்றும் உங்கள் கணினி ஒரே நெட்வொர்க்கில் இருக்கிறதா என பார்த்து
   - WiFiக்கு பதிலாக எதர்நெட் இணைப்பை முயற்சி செய்யவும்
4. பயனாளர் பெயர்/கடவுச்சொல் சரிபார்க்கவும் (இயல்புநிலை: பயனாளர் பெயர் `pi`, கடவுச்சொல் `raspberry`)

#### சிக்கல்: Grove Base Hat அங்கீகரிக்கப்படவில்லை
**அலெர்ட்:** சென்சார்கள் இயங்கவில்லை, I2C பிழைகள்

**தீர்வு:**
1. Base Hat அனைத்துப் GPIO பின்களிலும் சரியாக அமர்த்தப்பட்டிருக்க வேண்டும்
2. Pi அல்லது Base Hat இல் முறுக்கு பின்கள் உள்ளதா என பார்க்கவும்
3. I2C இடைமுகத்தை இயக்கு:
   ```bash
   sudo raspi-config nonint do_i2c 0
   sudo reboot
   ```
4. I2C சரியாக இயங்குகிறதா என உறுதி செய்ய: `i2cdetect -y 1`

#### சிக்கல்: ராஸ்பெரி பை மெதுவாக இயங்குகிறது
**அலெர்ட்:** UI இல் தாமதம், மெதுவான பதில்

**தீர்வு:**
1. SD கார்ட் வேகத்தை பாருங்கள் (பட்ஜெட் கூடப்பட்ட Class 10 அல்லது மேல், அல்லது USB SSD)
2. இடத்தைவிடுதலை செய்ய: `df -h` கொண்டு நிர்வகிக்கவும், தேவையற்ற கோப்புகளை நீக்கவும்
3. கோணக் கேமரா/திரை அதிகமாக பயன்படுத்தவில்லை என்றால் `raspi-config` இல் GPU நினைவகத்தை குறைக்கவும்
4. தேவையற்ற செயலிகளைக் கடைக்கோவையாக மூடவும்
5. Pi 3 அல்லது பழைய Pi உபயோகிக்கப்பட்டால் RAM அதிகம் உள்ள Pi 4 இற்கு மேம்படுத்தல் பரிந்துரைக்கப்படும்

### Wio Terminal

#### சிக்கல்: Wio Terminal திரை வெற்றிடமாக இருக்கும்
**அலெர்ட்:** குறியீடு பதிவேற்றத்துக்குப் பிறகு எந்த திரை வெளியீடும் இல்லை

**தீர்வு:**
1. குறியீடு திரையை ஆரம்பிப்பதா என்று பாருங்கள் (TFT_eSPI நூலகம்)
2. [Seeed Wiki](https://wiki.seeedstudio.com/Wio-Terminal-Getting-Started/) இல் இருந்து Wio Terminal விருப்பமுறை புதுப்பிப்பு மேற்கொள்ளவும்
3. திரை ஆரம்பிக்கும் குறியீட்டை சேர்க்கவும்:
   ```cpp
   #include <TFT_eSPI.h>
   TFT_eSPI tft;
   tft.begin();
   tft.fillScreen(TFT_BLACK);
   ```
4. PlatformIO இல் இருந்து உதாரண ஸ்கெட்ச் பதிவேற்ற முயற்சி செய்யவும்

#### சிக்கல்: Wio Terminal இல் WiFi வேலை செய்யவில்லை
**அலெர்ட்:** WiFiக்கு இணைக்க முடியவில்லை, நெட்வொர்க் பிழைகள்

**தீர்வு:**
1. **WiFi விருப்பமுறை புதுப்பிக்க:** [Wio Terminal WiFi firmware update guide](https://wiki.seeedstudio.com/Wio-Terminal-Network-Overview/) பின்பற்றவும்
2. **WiFi கல்வெட்டு சாக்குகள் சரிபார்:** SSID மற்றும் கடவுச்சொல் தவறுக்கு இடமில்லை என உறுதி செய்யவும்
3. **WiFi பட்டை:** Wio Terminal 2.4GHz WiFi மட்டுமே ஆதரிக்கிறது (5GHz கிடையாது)
4. **சிக்னல் வலிமை:** ரௌட்டருக்கு நெருக்கமாக நகரவும்
5. **ரௌட்டர் அமைப்புகள்:** சில தொழில்நுட்ப/WPA-Enterprise பிணையங்கள் வேலை செய்யாது

#### சிக்கல்: Wio Terminal கணினி மூலம் அங்கீகரிக்கப்படவில்லை
**அலெர்ட்:** USB சாதனம் கண்டறியப்படவில்லை

**தீர்வு:**
1. வேறு USB கேபிள் முயற்சி செய்யவும்: தரவு கேபிள், சார்ஜ் மட்டும் கேபிள் அல்ல
2. பூட்ட்லோடர் முறையில் நுழையவும்: பவர் ஸ்விட்சை விரைவில் இரண்டு முறை கீழே ஸ்லைடு செய்யவும்
   - நீலம் LED மின்னலிட்டுக் கொள்வது வேண்டும், சாதனம் சாதன மேலாளரில “Arduino” என தோன்றும்
3. டிரைவர்களை நிறுவவும் (விண்டோஸ்):
   - [Seeed USB driver](https://wiki.seeedstudio.com/Driver_for_Seeeduino/) பதிவிறகு மற்றும் நிறுவவும்
4. வேறு USB போர்டு (ஹப் இல்லாமல் நேரடி இணைப்பு) முயற்சி செய்யவும்
5. அமைப்பு USB டிரைவர்களை புதுப்பிக்கவும்

#### சிக்கல்: Wio Terminal சென்சார்கள் வேலை செய்யவில்லை
**அலெர்ட்:** Grove சென்சார்கள் தரவு படிக்கவில்லை

**தீர்வு:**
1. Grove கேபிள் இணைப்புகளைச் சரிபார்க்கவும்
2. சரியான Grove போர்ட்டைப் பயன்படுத்துகிறீர்களா என உறுதி செய்யவும் (இடது அல்லது வலது)
3. சென்சாருக்கு தேவையான நூலகங்கள் உள்ளனவா பார்ங்கள்கு
4. சென்சாரின் மின்சாரம் தேவைகளைச் சரிபார்க்கவும்
5. நூலகத்தில் இருந்து உதாரணக்குறியீடு மூலம் சென்சாரை சோதனை செய்யவும்

### மெய்நிகர் சாதனம் (CounterFit)

#### சிக்கல்: CounterFit செயலி துவங்கவில்லை
**பிழை:** CounterFit துவங்கும் போது பல்வேறு Python பிழைகள்

**தீர்வு:**
1. மெய்நிகர் சூழல் இயக்கப்பட்டுள்ளதா உறுதி செய்யவும்
2. CounterFit ஐ நிறுவவும்/மீண்டும் நிறுவவும்:
   ```bash
   pip install CounterFit
   ```
3. 5000 போர்ட் ஏற்கனவே பயன்படுத்தப்படுகிறதா பாருங்கள்:
   - விண்டோஸ்: `netstat -ano | findstr :5000`
   - macOS/Linux: `lsof -i :5000`
4. போர்ட்டை பயன்படுத்தும் செயலியை நிறுத்தவும் அல்லது வேறு போர்ட் பயன்படுத்தவும்:
   ```bash
   counterfit --port 5001
   ```

#### சிக்கல்: குறியீட்டிலிருந்து CounterFit இணைக்க முடியவில்லை
**பிழை:** இணைப்பு மறுக்கப்பட்டது அல்லது நேரம் முடிந்தது

**தீர்வு:**
1. CounterFit இயங்குகிறதா என சரிபார்க்கவும்: `http://127.0.0.1:5000` என உலாவியில் திறக்கவும்
2. குறியீட்டிலுள்ள இணைப்பு URL CounterFit முகவரியுடன் பொருந்துகிறதா பாருங்கள்
3. அகவலைச் சுவரம் (firewall) இணைப்பை தடை செய்யவில்லை என உறுதி செய்யவும்
4. CounterFit செயலியும் உங்கள் குறியீடும் இரண்டும் மீண்டும் துவக்கி முயற்சி செய்யவும்

#### சிக்கல்: CounterFit இல் சென்சார்கள் தோன்றவில்லை
**அலெர்ட்:** உருவாக்கிய சென்சார்கள் CounterFit UI இல் காணவில்லை

**தீர்வு:**
1. குறியீடு இயக்கும் முன் CounterFit UI இல் சென்சார்கள் உருவாக்கவும்
2. உலாவி பக்கத்தை புதுப்பிக்கவும்
3. சென்சார் வகை குறியீட்டு எதிர்பார்ப்புடன் பொருந்துகிறதா சரிபார்க்கவும்
4. உலாவி கேச் டேட்டாவை அழிக்கவும்

## இணையச் சிக்கல்கள்

### WiFi இணைப்பு

#### சிக்கல்: சாதனம் WiFiக்கு இணைக்க முடியவில்லை
**அலெர்ட்:** இணைப்பு நேரம் முடி, அங்கீகாரம் தவறானது

**தீர்வு:**
1. **SSID மற்றும் கடவுச்சொல் சரிபார்க்கவும்:** கல்வெட்டுகள் சரியாக உள்ளனவா என உறுதி செய்யவும்
2. **WiFi பட்டை:** பல IoT சாதனங்கள் 2.4GHz மட்டுமே ஆதரிக்கின்றன (5GHz இல்லை)
3. **ரௌட்டர் அமைப்புகள்:**
   - AP தனிமைப்படுத்தலை அணைத்து விடவும்
   - WPA2-PSK பாதுகாப்பை பயன்படுத்தவும் (WPA3, WEP அல்லது திறந்த நெட்வொர்க்குகளைத் தவிர்க்கவும்)
   - DHCP இயங்குகிறதா என உறுதி செய்யவும்
4. **மறைவு நெட்வொர்க்குகள்:** SSID மறைவு இருந்தால், அதை தெளிவாக அமைக்க வேண்டியிருக்கும்
5. **சிக்னல் வலிமை:** சாதனத்தை ரௌட்டருக்கு அருகிலே வைத்து காணவும்
6. **சொருக்கல்:** மற்ற சாதனங்கள், மைக்ரோவேவ் அல்லது சுவற்கள் தடை செய்யக்கூடும்

#### சிக்கல்: WiFi இணைப்பு அடிக்கடி துண்டிக்கிறது
**அலெர்ட்:** இடைவிடா இணைப்பு பிரச்சனை

**தீர்வு:**
1. ரௌட்டர் நிலைத்தன்மையைப் பார்த்து மீண்டும் துவக்கவும்
2. சாதன விருப்பமுறையை புதுப்பிக்கவும்
3. DHCP விட ஸ்டாட்டிக் IP பயன்படுத்தவும்
4. ரௌட்டருக்கு தூரத்தை குறைத்து அல்லது WiFi விரிவாக்கியைச் சேர்க்கவும்
5. பிற சாதனங்கள் ஏற்படுத்தும் சுடுகாட்டைக் காண்க
6. சக்தி வழங்கல் போதுமானதா என பரிசீலிக்கவும் (ராஸ்பெரி பை வசேஷம்)

### மேகம் சேவைகள்

#### சிக்கல்: Azure IoT ஹப்பில் இணைக்க முடியவில்லை
**பிழை:** அங்கீகாரம் தோல்வி, இணைப்பு மறுக்கப்பட்டது

**தீர்வு:**
1. **கல்வெட்டுகளை சரிபார்க்கவும்:**
   - இணைப்பு நிலை சரியாக உள்ளது என்பதை உறுதி செய்யவும்
   - இணைப்பு நிலையில் கூடுதல் இடைவெளி அல்லது வரிசை இடைவெளி இருக்காதீர்கள்
2. **சாதன பதிவு சரிபார்க்கவும்:** சாதனம் IoT ஹப்பில் பதிவு செய்யப்பட்டிருக்க வேண்டும்
3. **அகவலைச் சுவரம்/ப்ராக்ஸி:** MQTT (போர்ட் 8883) அல்லது HTTPS (போர்ட் 443) வெளியே செல்ல அனுமதியுள்ளதா பாருங்கள்
4. **IoT ஹப் பிரதேசம்:** ஹப் இயக்கத்தில் உள்ளது மற்றும் வேறு பிரதேசத்தில் இருந்து நேரம் தாமதம் இல்லாததா என்பதையும் உறுதி செய்யவும்
5. **அளவுகோள் வரம்புகள்:** இலவச தர வரம்புகள் மீறப்படலா கணக்கிடவும்
6. **இணைப்பு சோதனை செய்யவும்:**
   ```bash
   az iot hub device-identity show-connection-string --hub-name YourIoTHub --device-id YourDevice
   ```

#### சிக்கல்: Azure செயலிகள் துவங்கவில்லை
**அலெர்ட்:** செய்தி அனுப்பப்பட்டது ஆனால் செயலி இயங்கவில்லை

**தீர்வு:**
1. செயலி செயல்படுகிறதா (நிறுத்தப்படவில்லை என்பதை) பார்த்து உறுதி செய்யவும்
2. செயலி அமைப்பில் இணைப்பு நிலையை சரிபார்க்கவும்
3. Azure போர்டல் இல் செயலி பதிவு கொள்கைகளை சோதிக்கவும்
4. Event Hub இணக்கமான இறுதிப் புள்ளி சரியான முறையில் அமைக்கப்பட்டுள்ளது என உறுதி செய்யவும்
5. செய்தி வடிவமைப்பு செயலியின் எதிர்பார்ப்புடன் பொருந்துகிறதா பாருங்கள்
6. செயலி சேவை திட்டத்தை (கையெழுத்து vs. தனிப்பயன்) சரிபார்க்கவும்

### MQTT
#### பிரச்சனை: MQTT இணைப்பு தோல்வி
**பிழை:** இணைப்பு நிராகரிக்கப்பட்டது, அங்கீகாரம் தோல்வி

**விளக்கம்:**
1. **தொடர்பு முகவரி:** பிரோக்கர் URL/IP சரியென உறுதி செய்க
2. **போர்ட்:** போர்ட் எண் சரிபார் (1883 குறியாக்கமில்லாதது, 8883 TLS க்கானது)
3. **அங்கீகாரம்:** தேவைப்பட்டால் பயனர் பெயர்/கடவுச்சொல்லை சரிபார்
4. **TLS/SSL:** சான்றிதழ்கள் செல்லுபடியாகவும் நம்பகத்துவம் உறுதி செய்க
5. **நெருப்பு வேலி:** போர்ட் தடையிலிருந்து விடுபட்டவா என சரிபார்
6. **MQTT கிளையண்டுடன் சோதனை:** MQTT Explorer அல்லது mosquitto_pub/sub பயன்படுத்தி சோதனை செய்க

#### பிரச்சனை: MQTT செய்திகள் பெறப்படவில்லை
**அளவுருக்கள்:** செய்திகள் வெளியிடப்பட்டாலும் சந்தாகியாளர்களால் பெறப்படவில்லை

**விளக்கம்:**
1. **தலைப்பு பெயர்கள்:** சந்தாகியாளர் தலைப்பு வெளியீட்டாளரின் தலைப்புடன் சரியாக பொருந்துமா என்பதை உறுதி செய்க
2. **QoS நிலை:** 0 க்கு பதிலாக QoS 1 அல்லது 2 முயற்சிக்கவும்
3. **காடைக்களம்:** தலைப்பு காடைக்களம் சரியாகப் பயன்படுத்தப்பட்டுள்ளதா எனப் பாருங்கள் (`+` ஒரு நிலைக்கான, `#` பல நிலைக்கான)
4. **பிடிக்கப்பட்ட செய்திகள்:** வெளியீட்டாளர் retain கொடி வைத்து கடைசி செய்தியை வைத்திருக்கலாம்
5. **இணைப்பு நேரம்:** சந்தாகியாளர் செய்திகள் வெளியிடுவதற்கு முன் இணைக்கப்பட்டிருக்க வேண்டும்

## சென்சார் மற்றும் செயல்படுத்திகள் பிரச்சனைகள்

### கிரோவ் சென்சார்கள்

#### பிரச்சனை: சென்சார் தவறான மதிப்புகளைத் தருகிறது
**அளவுருக்கள்:** வாசிப்புகள் 0, -1 அல்லது பொருத்தமற்ற மதிப்புகள்

**விளக்கம்:**
1. **இணைப்புகளைச் சரிபார்:** சென்சார் நன்கு இணைக்கப்பட்டுள்ளதாக உறுதி செய்க
2. **சரியான போர்ட்:** சென்சார் சரியான போர்ட் வகையில் உள்ளதா என உறுதி செய்யவும்:
   - அநலைக் சென்சார்கள் → அநலைக் போர்ட்கள் (A0, A2, A4)
   - டிஜிட்டல் சென்சார்கள் → டிஜிட்டல் போர்ட்கள் (D5, D16, D18, மற்றும் பிற)
   - I2C சென்சார்கள் → I2C போர்ட்கள்
3. **காலிபிரேஷன்:** சில சென்சார்களுக்கு காலிபிரேஷன் தேவை ( நிலத்தடி ஈரப்பதம், ஒளி)
4. **பவர் சைக்கிள்:** சென்சாரை துண்டித்து மீண்டும் இணைத்து பார்க்கவும்
5. **சென்சார் தகவல் தாள்:** சென்சார் குறிப்புகள் மற்றும் தேவைகளைச் சரிபார்

#### பிரச்சனை: கெபாசிட்டிவ் நிலத்தடி ஈரப்பதச் சென்சார் எப்போதும் ஈரமானதை வாசிக்கிறது
**அளவுருக்கள்:** நிலத்தைத் துவத்திற்கு விட்டு இருந்தாலும் சென்சார் அதிக ஈரப்பதத்தை வாசிக்கிறது

**விளக்கம்:**
1. **காலிபிரேஷன் தேவை:** நிலத்தடி சென்சார்களுக்கு காலிபிரேஷன் தேவை:
   - காற்றில் (இருண்ட நிலை) மதிப்பு வாசிக்கவும்
   - நீரில் (ஈரமான நிலை) மதிப்பு வாசிக்கவும்
   - இந்த மதிப்புகளுக்கு இடையில் வாசிப்புகளை ஒப்பிடுக
2. **சென்சார் பூச்சு சரிபார்:** ஈரப்பத சென்சார்களின் பூச்சுகள் சேதமடைந்திருந்தால் خرابம் ஏற்படலாம்
3. **துண்டு இடம்:** சென்சார் முழுமையாக நிலத்தில் செருகப்பட்டிருப்பதை உறுதி செய்க

#### பிரச்சனை: வெப்பநிலை/ஈரப்பதச் சென்சார் வாசிப்பு தவறானது
**அளவுருக்கள்:** DHT11/DHT22 தவறான வெப்பநிலை அல்லது ஈரப்பதத்தை காட்டுகிறது

**விளக்கம்:**
1. **சென்சார் இடம்:** நேரடி சூரியஒளி, வெப்ப மூலங்கள் அல்லது காற்றின் ஓட்டத்திலிருந்து தவிர்க்கவும்
2. **பாதுகாப்பு நேரம்:** சென்சார் தொடரலை பவர்-ஆனுக்கு பின் 2 விநாடிகள் அனுமதிக்கவும்
3. **வாசிப்பு அடிக்கடி:** DHT சென்சார்கள் வாசிப்புக்கு இடையில் (குறைந்தது 2 விநாடிகள்) நேரம் தேவை
4. **கன்சன்டன்ஸின் சோதனை:** வாசிப்பை பாதிக்கும் ஆபாயம் உள்ளது
5. **சென்சார் தரம்:** DHT11 குறைவான துல்லியத்துடன் உள்ளது DHT22 இற்கு ஒப்பாக

## மேம்பாட்டு சூழல் பிரச்சனைகள்

### VS Code

#### பிரச்சனை: டெர்மினல் தானாக சர்வதேச சூழலை இயங்கவிடவில்லை
**அளவுருக்கள்:** டெர்மினல் திறக்கும் ஆனால் venv இயங்கவில்லை

**விளக்கம்:**
1. **Python உள்ளீடுநிலைத் தேர்வு செய்க:** Command Palette → "Python: Select Interpreter" → venv தேர்வு செய்க
2. தேர்ந்தெடுத்த பிறகு VS Code மீண்டும் துவக்கவும்
3. **அமைப்புகளைச் சரிபார்:** `settings.json` இல் சேர்க்கவும்:
   ```json
   "python.terminal.activateEnvironment": true
   ```

#### பிரச்சனை: கருவியில் குறியீடு இயக்கப்படவில்லை
**அளவுருக்கள்:** குறியீடு இயங்குகிறது ஆனால் கருவியில் எதுவும் நிகழவில்லை

**விளக்கம்:**
1. **குறியீடு சேமிக்கப்பட்டுள்ளதா என உறுதி செய்க** (கோப்பு தாவலில் புள்ளியைச் சரிபார்)
2. **வேறு Python இயக்கம் உறுதி செய்க:** `which python` அல்லது `where python`
3. **Wio Terminal க்கு:** PlatformIO மூலம் குறியீடு பதிவேற்றப்பட்டுள்ளது என்பதை உறுதி செய்க (பதிவேற்ற பொத்தானை கிளிக் செய்க)
4. **Raspberry Pi க்கானது:** Pi உள்நுழைந்து குறியீடு இயக்கவும்
5. பிழைகள் வரையறுக்க வெளியீட்டு சாளரம் சரிபார்

#### பிரச்சனை: IntelliSense நூலக செயல்பாடுகள் காட்டவில்லை
**அளவுருக்கள்:** இறக்குமதி செய்யப்பட்ட கூறுகளுக்கு தானாக பூர்த்தி இல்லை

**விளக்கம்:**
1. இன்றைய சூழலில் நூலகம் நிறுவப்பட்டுள்ளதா என உறுதி செய்க
2. VS Code சாளரத்தை மீள ஏற்றுக
3. Python உள்ளீடுநிலை சரி பார்க்கவும்
4. கிடைத்தால் வகை ஸ்டப்புகளை நிறுவுக: `pip install types-<library-name>`

### Python மெய்நிகர் சூழல்கள்

#### பிரச்சனை: மெய்நிகர் சூழலை உருவாக்க முடியவில்லை
**பிழை:** `The virtual environment was not created successfully`

**விளக்கம்:**
1. **venv தொகுதியை நிறுவுக:**
   - Ubuntu/Debian: `sudo apt install python3-venv`
   - macOS: Python உடன் சேர்ந்து வரும்
   - Windows: Python ஐ அனைத்து கூறுகளுடன் மீண்டும் நிறுவுக
2. **Python நிறுவலை சரிபார்:** Python முறையாக நிறுவப்பட்டுள்ளது என்பதை உறுதி செய்க
3. **முழு பாதையைப் பயன்படுத்துக:** `python3 -m venv .venv` என்ற தெளிவான python3 அழைப்புடன் முயற்சி செய்க

#### பிரச்சனை: தொகுதிகள் தவறான இடத்தில் நிறுவப்பட்டுள்ளன
**அளவுருக்கள்:** தொகுதி நிறுவிய பிறகு இறக்குமதி பிழை

**விளக்கம்:**
1. **venv இயங்குகிறது என உறுதி செய்யவும்:** கட்டளை வரி `(.venv)` காட்ட வேண்டும்
2. **pip இடம் சரிபார்:** `which pip` `.venv/bin/pip` ஐ குறிக்க வேண்டும்
3. **venv இல் மீண்டும் நிறுவுக:** venv இயக்கி, பிறகு `pip install <package>` செய்க
4. மெய்நிகர் சூழலில் pip உடன் sudo பயன்படுத்த வேண்டாமே

#### பிரச்சனை: மெய்நிகர் சூழல் நகர்த்தத்தக்கதாக இல்லை
**அளவுருக்கள்:** நகர்த்திய பிறகு அல்லது வேறு கணினியில் venv வேலை செய்யவில்லை

**விளக்கம்:**
1. **venvs நகர்த்த வேண்டாம்:** விலக்கு, புதிய இடத்தில் மறுபடி உருவாக்குக
2. **requirements.txt பயன்படுத்துக:**
   ```bash
   pip freeze > requirements.txt
   pip install -r requirements.txt
   ```
3. **venv மறுபடியும் உருவாக்குக:**
   ```bash
   python3 -m venv .venv
   source .venv/bin/activate  # அல்லது விண்டோஸில் activate.bat
   pip install -r requirements.txt
   ```

### சார்புள்ளவை

#### பிரச்சனை: தொகுதி நிறுவல் தோல்வி
**பிழை:** நிறுவல் போது பல்வேறு pip பிழைகள்

**விளக்கம்:**
1. **pip ஐ மேம்படுத்துக:**
   ```bash
   pip install --upgrade pip
   ```
2. **கட்டுமான கருவிகளை நிறுவுக:**
   - Ubuntu/Debian: `sudo apt install build-essential python3-dev`
   - macOS: `xcode-select --install`
   - Windows: Visual Studio Build Tools ஐ நிறுவுக
3. **இணைய இணைப்பை சரிபார்**
4. **வேறு தொகுதி குறியீட்டை முயற்சிக:** `pip install --index-url https://pypi.org/simple/ <package>`
5. **குறிப்பிட்ட பதிப்பு நிறுவுக:** `pip install <package>==<version>`

#### பிரச்சனை: சார்பு முரண்பாடுகள்
**பிழை:** `ERROR: pip's dependency resolver does not currently take into account all the packages that are installed`

**விளக்கம்:**
1. **ஒவ்வொரு திட்டத்திற்கும் புதிய மெய்நிகர் சூழலை பயன்படுத்தவும்**
2. **தொகுதிகளைக் மேம்படுத்துக:** `pip install --upgrade <package>`
3. **தேவைகளைச் சரிபார்:** `pip check` மூலம் முரண்பாடுகளை கண்டறியவும்
4. **போகத்துடன் பொருந்தும் பதிப்புகளை நிறுவுக:** requirements.txt இல் பதிப்பு வரம்புகளை குறிப்பிடுக

4. **SD கார்டு வேகம்:** ராஸ்பெர்ரி பிக்கு வேகமான SD கார்டோ SSD யோ பயன்படுத்துக

   - நினைவுக் கசிவுகள் இருப்பதைச் சரிபார்

## பொதுவான பிழை செய்திகள்

### `ModuleNotFoundError: No module named 'X'`
**காரணம்:** தொகுதி நிறுவப்படாதது அல்லது மெய்நிகர் சூழல் இயங்காதது

**விளக்கம்:**
```bash
pip install X
```
முதலில் மெய்நிகர் சூழல் இயங்கிச் சொப்பி ஆக வேண்டும்.

### `Permission denied` Linux/macOS இல்
**காரணம்:** உயர்ந்த அனுமதிகள் தேவை அல்லது கோப்பு அனுமதி பிரச்சனை

**விளக்கம்:**
- கணினி இயக்கங்களுக்காக: `sudo` பயன்படுத்தவும்
- pip க்காக: venv இயங்கும்படி செய்து sudo தவிர்க்கவும்
- தொடர் போர்ட்: பயனரை dialout குழுவுக்கு சேர்: `sudo usermod -a -G dialout $USER`, பிறகு வெளியேறவும்/உள்நுழையவும்

### `OSError: [Errno 98] Address already in use`
**காரணம்:** போர்ட் ஏற்கனவே வேறு செயலியில் பயன்படுத்தப்படுகிறது

**விளக்கம்:**
1. போர்ட் பயன்படுத்தும் செயலியை கண்டறிய: `lsof -i :<port>` அல்லது `netstat -ano | findstr :<port>`
2. செயலியை நிறுத்துக அல்லது குறியீட்டில் வேறு போர்ட் பயன்படுத்துக

### `SSL: CERTIFICATE_VERIFY_FAILED`
**காரணம்:** SSL சான்றிதழ் சரிபார்ப்பு தோல்வி

**விளக்கம்:**
1. சான்றிதழ்களை புதுப்பிக்க: `pip install --upgrade certifi`
2. கணினியின் நேரம் சரி என பார்த்துக் கொள்ளவும்: `date`
3. மேம்பாட்டுக்காக மட்டும் (உற்பத்திக்கு அல்ல): குறியீட்டில் சரிபார்ப்பை முடக்கு

### `IndentationError: unexpected indent`
**காரணம்:** Python அடுக்கு பிழைகள் (Tabs/Spaces கலக்கல்)

**விளக்கம்:**
1. சீரான உள்ளமைப்பைப் பயன்படுத்துக (4 இடைகளான இடைவெளிகள் Python நியமம்)
2. தொகுப்பி டேப்களை_SPACEகளாக மாற்ற அமைக்கவும்
3. VS Code: `"editor.insertSpaces": true` மற்றும் `"editor.tabSize": 4` ஐ அமைக்கவும்

### `UnicodeDecodeError` அல்லது `UnicodeEncodeError`
**காரணம்:** எழுத்துரு குறியாக்க பிழைகள்

**விளக்கம்:**
```python
# கோப்புகளை வாசிக்கும் போது
with open('file.txt', 'r', encoding='utf-8') as f:
    content = f.read()

# கோப்புகளை எழுதும் போது
with open('file.txt', 'w', encoding='utf-8') as f:
    f.write(content)
```

6. **பதிவுகளை புதுப்பிக்கவும்:** மென்பொருள்/பராமரிப்புகளை சமீபத்தியதாக வைத்திருங்கள்
7. **தரமான கூறுகளை பயன்படுத்தவும்:** மலிவான கேபிள்கள்/மின்சாரம் வழங்கிகளை தவிர்க்கவும்
8. **நிலையான மின்சாரம்:** பொருத்தமான மின்சார வழங்கியைப் பயன்படுத்தவும் (முகாந்திரமாக பை)

### மேம்பாட்டு வேலைதளம்
1. **எளிமையாக தொடங்கவும்:** வேலை செய்கின்ற எடுத்துக்காட்டு கோடுடன் துவங்கவும்
2. **ஒரே நேரத்தில் ஒரு மாற்றம்:** எது உடைந்தது என்பதை எளிதாக கண்டுபிடிக்க
3. **அடிக்கடி சோதிக்கவும்:** பிரச்சினைகளை துவக்கத்திலேயே பிடிக்க
4. **துப்புரவு வைத்திருங்கள்:** கோப்புகளையும் கோடியையும் முறைபடுத்து ஒழுங்குபடுத்தவும்
5. **கோடுகளை கருத்திடவும்:** எதிர்காலம் நீக்கு இதன் மதிப்பை அறியும்
