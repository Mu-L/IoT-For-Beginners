## انسٹالیشن کے مسائل

### پائتھون کی انسٹالیشن

#### مسئلہ: پائتھون کا ورژن بہت پرانا ہے
**خرابی:** `Python 3.6 or higher is required`

**حل:**
1. تازہ ترین پائتھون 3 [python.org](https://www.python.org/downloads/) سے ڈاؤن لوڈ کریں
2. ونڈوز پر انسٹال کرتے وقت "Add Python to PATH" کو منتخب کریں
3. انسٹالیشن کی تصدیق کریں:
   ```bash
   python3 --version
   ```

#### مسئلہ: متعدد پائتھون ورژنز کی وجہ سے تصادم
**علامات:** غلط پائتھون ورژن چل رہا ہے، پیکجز غلط جگہ انسٹال ہو رہے ہیں

**حل:**
- **ونڈوز:** Python 3 چلانے کے لیے `python` کی بجائے `py -3` استعمال کریں
- **macOS/Linux:** `python` کی جگہ `python3` استعمال کریں
- ہمیشہ پروجیکٹس کے لیے ورچوئل ماحول بنائیں اور استعمال کریں

#### مسئلہ: pip کمانڈ نہیں مل رہی
**خرابی:** `'pip' is not recognized as an internal or external command`

**حل:**
1. `pip` کی جگہ `pip3` آزما کر دیکھیں
2. یا `python -m pip` یا `python3 -m pip` استعمال کریں
3. یقینی بنائیں پائتھون PATH میں شامل ہے (پائتھون دوبارہ انسٹال کریں اور یہ آپشن چیک کریں)

### VS کوڈ اور ایکسٹینشنز

#### مسئلہ: Pylance ایکسٹینشن کام نہیں کر رہا
**علامات:** پائتھون انٹیلی سنس، کوڈ کمپلیشن، یا ٹائپ چیکنگ نہیں ہو رہی

**حل:**
1. VS کوڈ کمانڈ پیلیٹ کھولیں (`Ctrl+Shift+P` یا `Cmd+Shift+P`)
2. "Python: Select Interpreter" چلائیں
3. صحیح پائتھون انٹرپریٹر منتخب کریں (اگر ورچوئل ماحول ہے تو وہ)
4. VS کوڈ ونڈو کو ری لوڈ کریں

#### مسئلہ: VS کوڈ ورچوئل ماحول کی شناخت نہیں کر رہا
**علامات:** غلط پائتھون انٹرپریٹر منتخب ہو گیا ہے

**حل:**
1. ٹرمینل میں ورچوئل ماحول کو فعال کریں
2. کمانڈ پیلیٹ کھولیں اور "Python: Select Interpreter" چلائیں
3. `.venv` فولڈر سے انٹرپریٹر منتخب کریں
4. اسٹیٹس بار (نیچے بائیں) پر درست پائتھون ورژن ظاہر ہو رہا ہے، چیک کریں

### پلیٹ فارم آئی او (Wio ٹرمینل)

#### مسئلہ: پلیٹ فارم آئی او انسٹالیشن ناکام
**خرابی:** پلیٹ فارم آئی او انسٹالیشن کے دوران مختلف غلطیاں آ رہی ہیں

**حل:**
1. یقینی بنائیں VS کوڈ تازہ ترین ہے
2. پہلے C/C++ ایکسٹینشن انسٹال کریں
3. پلیٹ فارم آئی او انسٹال کرنے کے بعد VS کوڈ کو ری اسٹارٹ کریں
4. انٹرنیٹ کنکشن کو چیک کریں (پلیٹ فارم آئی او بڑی فائلیں ڈاؤن لوڈ کرتا ہے)

#### مسئلہ: پلیٹ فارم آئی او بورڈ کو شناخت نہیں کر رہا
**علامات:** Wio ٹرمینل پر کوڈ اپلوڈ نہیں ہو رہا

**حل:**
1. مختلف USB کیبل آزمائیں (کچھ کیبلز صرف چارجنگ کے لیے ہوتی ہیں)
2. ڈیوائس منیجر (ونڈوز) یا `ls /dev/tty*` (macOS/Linux) چیک کریں
3. USB ڈرائیور انسٹال یا اپ ڈیٹ کریں
4. مختلف USB پورٹ آزمائیں
5. Wio ٹرمینل کے پاور سوئچ کو دو بار تیز رفتار سے سلیڈ کریں تاکہ بوٹ لوڈر موڈ میں جائے

#### مسئلہ: پلیٹ فارم آئی او میں کمپائل کی غلطیاں
**خرابی:** `fatal error: Arduino.h: No such file or directory`

**حل:**
1. اپنے پروجیکٹ میں `.pio` فولڈر حذف کریں
2. کمانڈ پیلیٹ سے "PlatformIO: Rebuild" چلائیں
3. `platformio.ini` میں بورڈ کی درست ترتیب کو یقینی بنائیں:
   ```ini
   [env:seeed_wio_terminal]
   platform = atmelsam
   board = seeed_wio_terminal
   framework = arduino
   ```

### گروو لائبریریاں

#### مسئلہ: ریسپبرری پائی پر گروو لائبریری امپورٹ ناکام
**خرابی:** `ModuleNotFoundError: No module named 'grove'`

**حل:**
1. گروو لائبریریاں دوبارہ انسٹال کریں:
   ```bash
   cd ~
   git clone https://github.com/Seeed-Studio/grove.py
   cd grove.py
   sudo pip3 install .
   ```
2. اگر ورچوئل ماحول استعمال کر رہے ہیں تو ممکن ہے لائبریریاں عالمی طور پر انسٹال کرنی ہوں یا نقل کرنی ہوں
3. I2C کو فعال کریں: `sudo raspi-config nonint do_i2c 0`

#### مسئلہ: گروو سینسر کی شناخت نہیں ہو رہی
**خرابی:** `IOError: [Errno 121] Remote I/O error`

**حل:**
1. فزیکل کنکشن چیک کریں (گروو کیبل مکمل طور پر داخل ہے)
2. سینسر صحیح پورٹ پر جڑا ہوا ہے (ایناکلاگ، ڈیجیٹل، I2C، UART)
3. `i2cdetect -y 1` چلائیں تاکہ معلوم ہو کہ ڈیوائس I2C بس پر نظر آ رہا ہے یا نہیں
4. مختلف گروو کیبل آزما کر دیکھیں
5. گروو بیس ہیٹ کو ریسپبرری پائی GPIO پنز پر درست طریقے سے لگائیں

## ہارڈویئر کے مسائل

### ریسپبرری پائی

#### مسئلہ: ریسپبرری پائی بوٹ نہیں ہو رہا
**علامات:** کوئی ڈسپلے نہیں، LED سرگرمی نہیں، یا رینبو سکرین

**حل:**
1. **پاور سپلائی چیک کریں:** Pi 4 کے لیے رسمی 5V 3A USB-C پاور سپلائی استعمال کریں
2. **SD کارڈ کے مسائل:** 
   - SD کارڈ کو فارمیٹ کریں اور Raspberry Pi OS دوبارہ انسٹال کریں
   - مختلف SD کارڈ آزمائیں (مناسب برانڈز استعمال کریں)
   - SD کارڈ صحیح طور پر داخل کریں
3. **HDMI کنکشن چیک کریں:** Pi 4 کے دونوں HDMI پورٹس آزمائیں، پاور کے قریب والا پورٹ استعمال کریں

#### مسئلہ: ریسپبرری پائی پر SSH کنکٹ نہیں ہو رہا
**علامات:** کنکشن مسترد یا ٹائم آؤٹ

**حل:**
1. SSH فعال کریں:
   - SD کارڈ فلش کرتے وقت Raspberry Pi Imager کی ایڈوانسڈ آپشنز میں SSH چالو کریں
   - یا بوٹ پارٹیشن میں `ssh` نامی خالی فائل بنائیں (کوئی توسیع نہ ہو)
2. Pi کا IP ایڈریس معلوم کریں:
   - اپنے روٹر کے کنیکٹڈ ڈیوائسز چیک کریں
   - `ping raspberrypi.local` استعمال کریں (اگر mDNS کام کرتا ہے)
   - نیٹ ورک اسکیننگ ٹولز جیسے `nmap` یا Angry IP Scanner استعمال کریں
3. نیٹ ورک چیک کریں:
   - یقینی بنائیں Pi آپ کے کمپیوٹر کے نیٹ ورک پر ہے
   - وائی فائی کی بجائے ایتھرنیٹ کنکشن آزما کر دیکھیں
4. یوزر نیم/پاس ورڈ کی تصدیق کریں (ڈیفالٹ: یوزر نیم `pi`, پاس ورڈ `raspberry`)

#### مسئلہ: گروو بیس ہیٹ کی شناخت نہیں ہو رہی
**علامات:** سینسر کام نہیں کر رہے، I2C کی غلطیاں

**حل:**
1. بیس ہیٹ کو تمام GPIO پنز پر درست طریقے سے لگائیں
2. Pi یا بیس ہیٹ پر ٹیڑے ہوئے پنز چیک کریں
3. I2C انٹرفیس کو فعال کریں:
   ```bash
   sudo raspi-config nonint do_i2c 0
   sudo reboot
   ```
4. I2C کی درستگی کی تصدیق کریں: `i2cdetect -y 1`

#### مسئلہ: ریسپبرری پائی سست چل رہا ہے
**علامات:** یوزر انٹرفیس میں تاخیر، سست ریسپانس

**حل:**
1. SD کارڈ کی رفتار چیک کریں (کلاس 10 یا بہتر، یا USB کے ذریعے SSD)
2. ڈسک کی جگہ خالی کریں: `df -h` چیک کریں، غیر ضروری فائلز حذف کریں
3. `raspi-config` میں GPU میموری کم کریں اگر کیمرہ/ڈسپلے بھاری استعمال میں نہیں ہیں
4. غیر ضروری ایپلیکیشنز بند کریں
5. اگر Pi 3 یا پرانا استعمال کر رہے ہیں تو Pi 4 پر اپ گریڈ کرنے پر غور کریں جس میں زیادہ RAM ہو

### Wio ٹرمینل

#### مسئلہ: Wio ٹرمینل کی سکرین خالی رہتی ہے
**علامات:** کوڈ اپلوڈ ہونے کے بعد کوئی ڈسپلے نہیں آ رہا

**حل:**
1. چیک کریں کہ کوڈ ڈسپلے کو انیشیئلائز کر رہا ہے (TFT_eSPI لائبریری)
2. Wio ٹرمینل فرم ویئر کو [Seeed Wiki](https://wiki.seeedstudio.com/Wio-Terminal-Getting-Started/) سے اپ ڈیٹ کریں
3. ڈسپلے انیشیئلائزیشن کوڈ شامل کریں:
   ```cpp
   #include <TFT_eSPI.h>
   TFT_eSPI tft;
   tft.begin();
   tft.fillScreen(TFT_BLACK);
   ```
4. ہارڈویئر ٹیسٹ کے لیے پلیٹ فارم آئی او سے مثال کوڈ اپلوڈ کریں

#### مسئلہ: Wio ٹرمینل پر وائی فائی کام نہیں کر رہا
**علامات:** وائی فائی سے کنیکٹ نہیں ہو پا رہا، نیٹ ورک کی غلطیاں

**حل:**
1. **وائی فائی فرم ویئر اپ ڈیٹ کریں:** [Wio Terminal WiFi firmware update guide](https://wiki.seeedstudio.com/Wio-Terminal-Network-Overview/) دیکھیں
2. **وائی فائی کی تفصیلات چیک کریں:** SSID اور پاس ورڈ بالکل درست ہوں
3. **وائی فائی بینڈ:** Wio ٹرمینل صرف 2.4GHz وائی فائی سپورٹ کرتا ہے (5GHz نہیں)
4. **سگنل کی طاقت:** روٹر کے قریب لے جائیں
5. **روٹر کی سیٹنگز:** بعض انٹرپرائز/WPA-Enterprise نیٹ ورکس کام نہیں کر سکتے

#### مسئلہ: کمپیوٹر Wio ٹرمینل کو شناخت نہیں کر رہا
**علامات:** USB ڈیوائس شناخت نہیں ہو رہا

**حل:**
1. مختلف USB کیبل آزمائیں: ڈیٹا کیبل استعمال کریں، صرف چارجنگ کیبل نہیں
2. بوٹ لوڈر موڈ میں جائیں: پاور سوئچ کو دو بار تیز نیچے کی طرف سلائیڈ کریں
   - نیلا LED پلس ہونا چاہیے، ڈیوائس منیجر میں "Arduino" کے طور پر ظاہر ہوگا
3. (ونڈوز) ڈرائیور انسٹال کریں:
   - [Seeed USB driver](https://wiki.seeedstudio.com/Driver_for_Seeeduino/) ڈاؤن لوڈ کریں اور انسٹال کریں
4. مختلف USB پورٹ آزمائیں: USB ہب سے بچیں، ڈائریکٹ کنکشن کریں
5. سسٹم USB ڈرائیور کو اپ ڈیٹ کریں

#### مسئلہ: Wio ٹرمینل پر سینسر کام نہیں کر رہے
**علامات:** گروو سینسر ڈیٹا نہیں پڑھ رہے

**حل:**
1. گروو کیبل کنکشن چیک کریں
2. یقینی بنائیں صحیح گروو پورٹ استعمال کر رہے ہیں (بائیں یا دائیں)
3. سینسر کے لیے صحیح لائبریریاں شامل کریں
4. سینسر کی پاور ضروریات چیک کریں
5. لائبریری سے ملنے والا مثال کوڈ استعمال کر کے سینسر کو ٹیسٹ کریں

### ورچوئل ڈیوائس (کاؤنٹر فِٹ)

#### مسئلہ: کاؤنٹر فِٹ ایپ اسٹارٹ نہیں ہو رہی
**خرابی:** کاؤنٹر فِٹ شروع کرتے وقت مختلف پائتھون کی غلطیاں

**حل:**
1. یقینی بنائیں ورچوئل ماحول فعال ہے
2. کاؤنٹر فِٹ انسٹال یا دوبارہ انسٹال کریں:
   ```bash
   pip install CounterFit
   ```
3. پورٹ 5000 پہلے سے استعمال میں نہیں ہے چیک کریں:
   - ونڈوز: `netstat -ano | findstr :5000`
   - macOS/Linux: `lsof -i :5000`
4. پورٹ 5000 استعمال کرنے والے عمل کو ختم کریں یا مختلف پورٹ استعمال کریں:
   ```bash
   counterfit --port 5001
   ```

#### مسئلہ: کوڈ سے کاؤنٹر فِٹ کنیکٹ نہیں ہو رہا
**خرابی:** کنکشن مسترد یا ٹائم آؤٹ

**حل:**
1. تصدیق کریں کہ کاؤنٹر فِٹ چل رہا ہے: براؤزر میں `http://127.0.0.1:5000` کھولیں
2. کوڈ میں کنکشن URL کاؤنٹر فِٹ کے پتے سے میل کھاتا ہو
3. یقینی بنائیں فائر وال کنکشن کو بلاک نہیں کر رہا
4. کاؤنٹر فِٹ ایپ اور آپ کے کوڈ دونوں کو ری اسٹارٹ کریں

#### مسئلہ: کاؤنٹر فِٹ میں سینسر ظاہر نہیں ہو رہے
**علامات:** بنائے گئے سینسر کاؤنٹر فِٹ UI میں نظر نہیں آ رہے

**حل:**
1. کوڈ چلانے سے پہلے کاؤنٹر فِٹ UI میں سینسر بنائیں
2. براؤزر کا صفحہ ریفریش کریں
3. سینسر کی قسم چیک کریں کہ کوڈ کے مطابق ہو
4. براؤزر کی کیشے صاف کریں

## کنیکٹیویٹی کے مسائل

### وائی فائی کنکشن

#### مسئلہ: ڈیوائس وائی فائی سے کنیکٹ نہیں ہو پا رہا
**علامات:** کنکشن ٹائم آؤٹ، توثیق ناکام

**حل:**
1. **SSID اور پاس ورڈ چیک کریں:** تفصیلات درست ہوں
2. **وائی فائی بینڈ:** زیادہ تر IoT ڈیوائسز صرف 2.4GHz (5GHz نہیں) سپورٹ کرتے ہیں
3. **روٹر کی سیٹنگز:**
   - AP آ ئسولیشن بند کریں اگر فعال ہو
   - WPA2-PSK سیکیورٹی استعمال کریں (WPA3, WEP یا اوپن نیٹ ورکس سے بچیں)
   - یقینی بنائیں DHCP فعال ہے
4. **چھپے ہوئے نیٹ ورکس:** اگر SSID چھپا ہوا ہے تو اسے واضح طور پر ترتیب دیں
5. **سگنل کی طاقت:** ڈیوائس کو روٹر کے قریب لائیں
6. **مداخلت:** دوسرے ڈیوائسز، مائیکروویوز، یا دیواریں مداخلت کر سکتی ہیں

#### مسئلہ: وائی فائی کنکشن بار بار کٹتا رہتا ہے
**علامات:** غیر مستقل کنیکٹیویٹی

**حل:**
1. روٹر کی استحکام چیک کریں اور ریبوٹ کریں
2. ڈیوائس فرم ویئر اپ ڈیٹ کریں
3. DHCP کی بجائے سٹیٹک IP استعمال کریں
4. روٹر کے قریب ہو جائیں یا وائی فائی ایکسٹینڈر لگائیں
5. دوسرے ڈیوائسز کی مداخلت چیک کریں
6. یقینی بنائیں پاور سپلائی مکمل ہے (خاص طور پر ریسپبرری پائی کے لیے)

### کلاؤڈ سروسز

#### مسئلہ: Azure IoT Hub سے کنیکٹ نہیں ہو پا رہا
**خرابی:** توثیق ناکام، کنکشن مسترد

**حل:**
1. **تفصیلات کی تصدیق کریں:**
   - کنکشن سٹرنگ درست ہو
   - کنکشن سٹرنگ میں اضافی خالی جگہیں یا لائن بریک نہ ہوں
2. **ڈیوائس رجسٹریشن چیک کریں:** ڈیوائس IoT Hub میں رجسٹر ہونا چاہیے
3. **فائر وال/پروکسی:** MQTT (پورٹ 8883) یا HTTPS (پورٹ 443) آؤٹ باؤنڈ اجازت دیں
4. **IoT Hub خطہ:** یقینی بنائیں IoT Hub چل رہا ہے اور دیگر خطہ میں نہیں ہے جو لیٹنسی پیدا کرے
5. **کوٹہ کی حدیں:** مفت سطح کی حدوں کو چیک کریں
6. **کنکشن ٹیسٹ کریں:**
   ```bash
   az iot hub device-identity show-connection-string --hub-name YourIoTHub --device-id YourDevice
   ```

#### مسئلہ: Azure Functions فائر نہیں ہو رہی
**علامات:** پیغامات بھیجے جاتے ہیں لیکن فنکشن عمل میں نہیں آ رہی

**حل:**
1. چیک کریں کہ فنکشن ایپ چل رہی ہے (روکی ہوئی نہ ہو)
2. فنکشن ایپ کی سیٹنگز میں کنکشن سٹرنگ کی تصدیق کریں
3. Azure پورٹل میں فنکشن لاگز چیک کریں
4. ایونٹ ہب مطابقت پذیر اینڈ پوائنٹ درست طریقے سے ترتیب دیں
5. پیغام کا فارمیٹ فنکشن کی توقعات سے میل کھاتا ہو
6. فنکشن ایپ سروس پلان چیک کریں (consumption بمقابلہ dedicated)

### MQTT
#### مسئلہ: MQTT کنکشن ناکام ہو جاتا ہے  
**خرابی:** کنکشن مسترد کر دیا گیا، تصدیق ناکام

**حل:**  
1. **بروکر پتہ:** تصدیق کریں کہ بروکر URL/IP درست ہے  
2. **پورٹ:** پورٹ نمبر چیک کریں (غیر مرموز کے لیے 1883، TLS کے لیے 8883)  
3. **تصدیق:** اگر ضروری ہو تو یوزر نیم/پاسورڈ کی تصدیق کریں  
4. **TLS/SSL:** یقینی بنائیں کہ سرٹیفیکیٹس درست اور معتبر ہیں  
5. **فائر وال:** چیک کریں کہ پورٹ بلاک نہ ہو  
6. **MQTT کلائنٹ کے ساتھ جانچ کریں:** MQTT Explorer یا mosquitto_pub/sub استعمال کریں جانچ کے لیے  

#### مسئلہ: MQTT پیغامات وصول نہیں ہوتے  
**علامات:** پیغامات شائع ہو رہے ہیں مگر سبسکرائبرز کو وصول نہیں ہو رہے  

**حل:**  
1. **ٹاپک کے نام:** تصدیق کریں کہ سبسکرائبر کا ٹاپک مکمل طور پر پبلشر کے ٹاپک سے میل کھاتا ہے  
2. **QoS سطح:** 0 کی جگہ QoS 1 یا 2 آزمائیں  
3. **وائلڈ کارڈز:** چیک کریں کہ ٹاپک وائلڈ کارڈز درست استعمال ہوئے ہیں (`+` ایک سطح کے لیے، `#` ملٹی سطح کے لیے)  
4. **روک رکھے گئے پیغامات:** پبلشر آخری پیغام کو روک رکھنے کے لیے تاکہ سبسکرائبر اسے وصول کر سکیں  
5. **کنکشن کا وقت:** چیک کریں کہ سبسکرائبر پیغامات شائع ہونے سے پہلے کنیکٹ ہو جائے

## سینسر اور ایکچیویٹر کے مسائل

### گرویو سینسرز

#### مسئلہ: سینسر غلط ویلیوز دیتا ہے  
**علامات:** ریڈنگ صفر، -1 یا غیر منطقی قیمتیں  

**حل:**  
1. **کنیکشن چیک کریں:** یقین دہانی کریں کہ سینسر صحیح طریقے سے جڑا ہوا ہے  
2. **صحیح پورٹ:** چیک کریں کہ سینسر صحیح پورٹ کی قسم میں ہے:  
   - اینالاگ سینسرز → اینالاگ پورٹس (A0, A2, A4)  
   - ڈیجیٹل سینسرز → ڈیجیٹل پورٹس (D5, D16, D18، وغیرہ)  
   - I2C سینسرز → I2C پورٹس  
3. **کیلائبریشن:** بعض سینسرز کو کیلائبریٹ کرنا ضروری ہے (مثلاً مٹی کی نمی، روشنی)  
4. **پاور سائیکل:** سینسر کو انپلگ کر کے دوبارہ پلگ کریں  
5. **سینسر کا ڈیٹا شیٹ:** سینسر کی خصوصیات اور ضروریات چیک کریں  

#### مسئلہ: کپیسٹیو مٹی کی نمی سینسر ہمیشہ گیلا پڑھتا ہے  
**علامات:** سینسر خشک ہونے پر بھی زیادہ نمی دکھاتا ہے  

**حل:**  
1. **کیلائبریشن ضروری ہے:** مٹی کے سینسرز کو کیلائبریٹ کرنا ہوتا ہے:  
   - ہوا میں پڑھیں (خشک بیس لائن)  
   - پانی میں پڑھیں (گیلا بیس لائن)  
   - ان قیمتوں کے درمیان ریڈنگ کا نقشہ بنائیں  
2. **سینسر کی کوٹنگ چیک کریں:** نمی سینسر خراب ہو سکتا ہے اگر کوٹنگ خراب ہو جائے  
3. **جگہ کا انتخاب:** یقینی بنائیں کہ سینسر پوری طرح زمین میں ڈالا گیا ہے  

#### مسئلہ: درجہ حرارت/نمی سینسر کی ریڈنگ غلط ہے  
**علامات:** DHT11/DHT22 درجہ حرارت یا نمی غلط دکھاتا ہے  

**حل:**  
1. **سینسر کی پوزیشن:** براہ راست دھوپ، حرارت کے ذرائع، یا ہوا کے بہاؤ سے بچیں  
2. **گرم ہونے کا وقت:** سینسر کو بجلی لگانے کے بعد 2 سیکنڈ دیں  
3. **ریڈ فریکوئنسی:** DHT سینسرز کو ریڈ کے درمیان کم از کم 2 سیکنڈ کی ضرورت ہوتی ہے  
4. **کندینس کے لیے چیک کریں:** نمی کی وجہ سے ریڈنگ متاثر ہو سکتی ہے  
5. **سینسر کا معیار:** DHT11 DHT22 کے مقابلے میں کم درست ہے

   sudo reboot

3. **کیمرہ سیٹنگز:** کوڈ میں ایکسپوژر، ISO، وائٹ بیلنس کو ایڈجسٹ کریں  
4. **استحکام:** کیمرہ کو مستحکم رکھیں، ضرورت ہو تو ٹرائی پوڈ استعمال کریں  
5. **ریزولوشن:** کیمرہ کی زیادہ سے زیادہ ریزولوشن سے تجاوز نہ کریں

## ڈیولپمنٹ ماحول کے مسائل

### VS Code

#### مسئلہ: ٹرمینل خودکار طور پر ورچوئل انوائرنمنٹ کو ایکٹیویٹ نہیں کرتا  
**علامات:** ٹرمینل کھلتا ہے لیکن venv ایکٹیویٹ نہیں ہوتا  

**حل:**  
1. **پائتھون انٹرپریٹر سیٹ کریں:** کمانڈ پیلیٹ → "Python: Select Interpreter" → venv منتخب کریں  
2. **VS Code کو ری سٹارٹ کریں** انٹرپریٹر منتخب کرنے کے بعد  
3. **سیٹنگز چیک کریں:** `settings.json` میں شامل کریں:  
   ```json
   "python.terminal.activateEnvironment": true
   ```
  

#### مسئلہ: کوڈ ڈیوائس پر چل نہیں رہا  
**علامات:** کوڈ چلتا ہے مگر ڈیوائس پر کوئی عمل نہیں ہوتا  

**حل:**  
1. **کوڈ محفوظ ہے یا نہیں چیک کریں** (فائل ٹیب پر ڈاٹ دیکھیں)  
2. **چیک کریں کون سا پائتھون چل رہا ہے:** `which python` یا `where python`  
3. **Wio Terminal کے لیے:** یقینی بنائیں کہ کوڈ PlatformIO کے ذریعے اپلوڈ ہوا ہے (upload بٹن کلک کریں)  
4. **Raspberry Pi کے لیے:** Pi پر SSH کریں اور وہاں کوڈ چلائیں  
5. **آؤٹ پٹ ونڈو چیک کریں** خرابیوں کے لیے  

#### مسئلہ: IntelliSense لائبریری فنکشنز نہیں دکھا رہا  
**علامات:** امپورٹ کیے گئے ماڈیولز کے لئے کوئی آٹو کمپلیٹ نہیں آ رہا  

**حل:**  
1. یقینی بنائیں کہ لائبریری موجودہ ماحول میں انسٹال ہے  
2. VS Code ونڈو ری لوڈ کریں  
3. پائتھون انٹرپریٹر درست ہے  
4. اگر دستیاب ہو تو ٹائپ اسٹب انسٹال کریں: `pip install types-<library-name>`  

### پائتھون ورچوئل انوائرنمنٹس

#### مسئلہ: ورچوئل انوائرنمنٹ نہیں بن پا رہا  
**خرابی:** `The virtual environment was not created successfully`

**حل:**  
1. **venv ماڈیول انسٹال کریں:**  
   - Ubuntu/Debian: `sudo apt install python3-venv`  
   - macOS: پائتھون کے ساتھ شامل ہوتا ہے  
   - Windows: Python کو تمام کمپونینٹس کے ساتھ دوبارہ انسٹال کریں  
2. **پائتھون انسٹالیشن چیک کریں:** تصدیق کریں کہ پائتھون درست انسٹال ہے  
3. **مکمل راستہ استعمال کریں:** explicit python3 کال کے ساتھ `python3 -m venv .venv` آزمائیں  

#### مسئلہ: پیکجز غلط جگہ انسٹال ہو جاتے ہیں  
**علامات:** پیکج انسٹال کرنے کے بعد امپورٹ ایرر آتا ہے  

**حل:**  
1. **تصیح کریں کہ venv ایکٹیویٹ ہے:** کمانڈ پرامپٹ پر `(.venv)` نظر آنا چاہیے  
2. **pip کا مقام چیک کریں:** `which pip` کا آؤٹ پٹ `.venv/bin/pip` کی طرف اشارہ کرے  
3. **venv میں دوبارہ انسٹال کریں:** venv اکٹیویٹ کریں، پھر `pip install <package>` کریں  
4. **virtual environment میں sudo کا استعمال نہ کریں**  

#### مسئلہ: ورچوئل انوائرنمنٹ پورٹیبل نہیں ہے  
**علامات:** venv منتقل کرنے یا مختلف کمپیوٹر پر کام نہیں کرتا  

**حل:**  
1. **venv کو منتقل نہ کریں:** حذف کریں اور نئی جگہ پر دوبارہ بنائیں  
2. **requirements.txt استعمال کریں:**  
   ```bash
   pip freeze > requirements.txt
   pip install -r requirements.txt
   ```
  
3. **venv دوبارہ بنائیں:**  
   ```bash
   python3 -m venv .venv
   source .venv/bin/activate  # ونڈوز پر activate.bat یا
   pip install -r requirements.txt
   ```
  

### انحصارات

#### مسئلہ: پیکج انسٹالیشن ناکام  
**خرابی:** انسٹالیشن کے دوران مختلف pip ایررز  

**حل:**  
1. **pip اپڈیٹ کریں:**  
   ```bash
   pip install --upgrade pip
   ```
  
2. **بلڈ ٹولز انسٹال کریں:**  
   - Ubuntu/Debian: `sudo apt install build-essential python3-dev`

3. **انٹرنیٹ کنکشن چیک کریں**  
4. **مختلف پیکج انڈیکس آزمائیں:** `pip install --index-url https://pypi.org/simple/ <package>`  
5. **مخصوص ورژن انسٹال کریں:** `pip install <package>==<version>`  

#### مسئلہ: انحصاری تنازعات  
**خرابی:** `ERROR: pip's dependency resolver does not currently take into account all the packages that are installed`

**حل:**  
1. **ہر پروجیکٹ کے لیے نیا ورچوئل ماحول استعمال کریں**  
2. **پیکجز اپگریڈ کریں:** `pip install --upgrade <package>`  
3. **تنازعات تلاش کریں:** `pip check` استعمال کریں  
4. **مطابقت رکھنے والے ورژنز انسٹال کریں:** requirements.txt میں ورژن رینجز دیں

## کارکردگی کے مسائل

### مسئلہ: کوڈ سست چلتا ہے  
**علامات:** تاخیر، ٹائم آؤٹ، غیر جوابی رویہ  

**حل:**  
1. **سینسر ریڈ فریکوئنسی کم کریں:** سینسرز کو بار بار نہ پڑھیں  
2. **لوپس بہتر کریں:** busy-waiting سے بچیں، sleep() یا تاخیر استعمال کریں  
3. **میموری کے مسائل:**  
   - غیر ضروری ایپس بند کریں  
   - اسٹوریج کی جگہ خالی کریں  
   - `top` یا `htop` کے ساتھ مانیٹر کریں  
4. **SD کارڈ کی رفتار:** راسبیری پائی کے لیے تیز SD کارڈ یا SSD استعمال کریں  
5. **نیٹ ورک تاخیر:** نیٹ ورک کالز کے لیے async آپریشنز استعمال کریں  

### مسئلہ: میموری ختم ہونے کی غلطیاں  
**خرابی:** `MemoryError` یا سسٹم جم جانا  

**حل:**  
1. **راسبیری پائی کے لیے:**  
   - غیر ضروری ایپس بند کریں  
   - swap space بڑھائیں  
   - ہلکا OS استعمال کریں (Lite ورژن)  
   - RAM اپگریڈ کریں (Pi 4 میں 2/4/8GB آپشنز)  
2. **Wio Terminal کے لیے:**  
   - بفر سائز کم کریں  
   - چھوٹی تصاویر استعمال کریں  
   - سٹرنگ کے استعمال کو بہتر بنائیں  
   - میموری لیکز چیک کریں (میموری جو آزاد نہیں ہو رہی)  

### مسئلہ: ڈیٹا کا نقصان یا کرپشن  
**علامات:** پیغامات غائب، فائلیں خراب  

**حل:**  
1. **SD کارڈ کے مسائل:**  
   - معیاری SD کارڈ استعمال کریں (سستے یا جعلی سے بچیں)  
   - باقاعدہ بیک اپ بنائیں  
   - صحیح طریقے سے بند کریں (پاور فورس نہ پلگ کریں)  
2. **بفر اوور فلو:** کوڈ میں بفر سائز بڑھائیں  
3. **نیٹ ورک کی بھروسہ مندی:** ری ٹرائی لاجک اور ایرر ہینڈلنگ نافذ کریں  
4. **کوالٹی آف سروس:** اہم پیغامات کے لیے MQTT QoS 1 یا 2 استعمال کریں

## عام خرابی کے پیغام

### `ModuleNotFoundError: No module named 'X'`  
**وجہ:** پیکج انسٹال نہیں یا ورچوئل انوائرنمنٹ ایکٹیویٹ نہیں  

**حل:**  
```bash
pip install X
```
  
سب سے پہلے ورچوئل انوائرنمنٹ کو ایکٹیویٹ کریں۔  

### `Permission denied` لینکس/macOS پر  
**وجہ:** بلند سطح کی اجازت یا فائل پرمیشن کا مسئلہ  

**حل:**  
- سسٹم آپریشنز کے لیے: `sudo` استعمال کریں  
- pip کے لیے: venv ایکٹیویٹ کریں، sudo کا استعمال نہ کریں  
- سیریل پورٹ کے لیے: یوزر کو dialout گروپ میں شامل کریں: `sudo usermod -a -G dialout $USER`، پھر لاگ آوٹ/لاگ ان کریں  

### `OSError: [Errno 98] Address already in use`  
**وجہ:** پورٹ پہلے سے کسی اور عمل میں استعمال ہو رہی ہے  

**حل:**  
1. پورٹ استعمال کرنے والے عمل کو تلاش کریں: `lsof -i :<port>` یا `netstat -ano | findstr :<port>`  
2. عمل کو بند کریں یا کوڈ میں مختلف پورٹ استعمال کریں  

### `SSL: CERTIFICATE_VERIFY_FAILED`  
**وجہ:** SSL سرٹیفکیٹ کی تصدیق ناکام ہو رہی ہے  

**حل:**  
1. سرٹیفیکیٹس اپڈیٹ کریں: `pip install --upgrade certifi`  
2. سسٹم کا وقت درست ہے یا نہیں چیک کریں: `date`  
3. صرف ڈیولپمنٹ کے لیے (پیداوار نہیں): کوڈ میں تصدیق بند کریں  

### `IndentationError: unexpected indent`  
**وجہ:** پائتھون میں انڈینٹیشن کا مسئلہ (ٹیبز اور اسپیسز کا مکس)  

**حل:**  
1. مستقل انڈینٹیشن استعمال کریں (4 اسپیسز پائتھون کا معیار)  
2. ایڈیٹر کو کنفیگر کریں کہ ٹیبز کی جگہ اسپیسز استعمال کرے  
3. VS Code میں: `"editor.insertSpaces": true` اور `"editor.tabSize": 4` سیٹ کریں  

### `UnicodeDecodeError` یا `UnicodeEncodeError`  
**وجہ:** کریکٹر انکوڈنگ مسائل  

**حل:**  
```python
# فائلیں پڑھتے وقت
with open('file.txt', 'r', encoding='utf-8') as f:
    content = f.read()

# فائلیں لکھتے وقت
with open('file.txt', 'w', encoding='utf-8') as f:
    f.write(content)
```
