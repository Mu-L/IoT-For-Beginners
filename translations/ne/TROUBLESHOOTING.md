## इन्स्टलेशन समस्या

### Python इन्स्टलेशन

#### समस्या: Python संस्करण धेरै पुरानो छ
**त्रुटि:** `Python 3.6 वा माथि आवश्यक छ`

**समाधान:**
1. [python.org](https://www.python.org/downloads/) बाट नयाँ Python 3 डाउनलोड गर्नुहोस्
2. Windows मा इन्स्टलेशन गर्दा, "Add Python to PATH" विकल्पमा टिका लगाउनुहोस्
3. इन्स्टलेशन प्रमाणित गर्नुहोस्:
   ```bash
   python3 --version
   ```

#### समस्या: विभिन्न Python संस्करणहरूले द्विविधा सिर्जना गरिरहेका छन्
**लक्षणहरू:** गलत Python संस्करण चल्छ, प्याकेजहरू गलत स्थानमा इन्स्टल हुन्छन्

**समाधान:**
- **Windows:** `python` को सट्टा `py -3` प्रयोग गर्नुहोस् जसले Python 3 स्पष्ट रूपमा चलाउँछ
- **macOS/Linux:** `python` को सट्टा `python3` प्रयोग गर्नुहोस्
- सधैं प्रोजेक्टहरूको लागि भर्चुअल वातावरण बनाउनुहोस् र प्रयोग गर्नुहोस्

#### समस्या: pip कमाण्ड फेला परेन
**त्रुटि:** `'pip' आन्तरिक वा बाह्य कमाण्डको रूपमा मान्यता प्राप्त छैन`

**समाधान:**
1. `pip` को सट्टा `pip3` प्रयोग गरेर हेर्नुहोस्
2. वा `python -m pip` वा `python3 -m pip` प्रयोग गर्नुहोस्
3. Python PATH मा समावेश छ कि छैन जाँच्नुहोस् (Python पुनः इन्स्टल गर्दा विकल्प जाँच्नुहोस्)

### VS Code र एक्सटेन्सनहरू

#### समस्या: Pylance एक्सटेन्सनले काम गर्दैन
**लक्षणहरू:** Python IntelliSense, कोड पूर्ति, वा टाइप चेकिङ छैन

**समाधान:**
1. VS Code Command Palette खोल्नुहोस् (`Ctrl+Shift+P` वा `Cmd+Shift+P`)
2. "Python: Select Interpreter" चलाउनुहोस्
3. सही Python इन्टरप्रेटर रोज्नुहोस् (यदि भर्चुअल वातावरण छ भने त्यो)
4. VS Code विन्डो पुनः लोड गर्नुहोस्

#### समस्या: VS Code ले भर्चुअल वातावरण पत्ता लगाउँदैन
**लक्षणहरू:** गलत Python इन्टरप्रेटर चयन भएको छ

**समाधान:**
1. टर्मिनलमा भर्चुअल वातावरण सक्रिय गरिएको छ सुनिश्चित गर्नुहोस्
2. Command Palette खोलेर "Python: Select Interpreter" चलाउनुहोस्
3. `.venv` फोल्डरबाट इन्टरप्रेटर चयन गर्नुहोस्
4. तल बाँया Status Bar मा सही Python संस्करण देखिन्छ कि छैन जाँच्नुहोस्

### PlatformIO (Wio Terminal)

#### समस्या: PlatformIO इन्स्टलेशन असफल हुन्छ
**त्रुटि:** PlatformIO इन्स्टलेशनको क्रममा विभिन्न त्रुटिहरू

**समाधान:**
1. VS Code अद्यावधिक गरिएको छ कि छैन जाँच्नुहोस्
2. पहिले C/C++ एक्सटेन्सन इन्स्टल गर्नुहोस्
3. PlatformIO इन्स्टल गरेपछि VS Code पुनः सुरु गर्नुहोस्
4. इन्टरनेट कनेक्शन ठिक छ कि छैन जाँच्नुहोस् (PlatformIO ले ठूलो फाइलहरू डाउनलोड गर्छ)

#### समस्या: PlatformIO ले बोर्ड पत्ता लगाउँदैन
**लक्षणहरू:** Wio Terminal मा कोड अपलोड गर्न सकिँदैन

**समाधान:**
1. अर्को USB केबल प्रयोग गरेर हेर्नुहोस् (केही केबल चार्ज मात्र हुन सक्छन्)
2. Device Manager (Windows) वा `ls /dev/tty*` (macOS/Linux) मा जाँच गर्नुहोस्
3. USB ड्राइभर इन्स्टल वा अपडेट गर्नुहोस्
4. अर्को USB पोर्ट प्रयोग गर्नुहोस्
5. Wio Terminal को पावर स्विच छिटो दुई पटक सारेर बुटलोडर मोडमा जानुस्

#### समस्या: PlatformIO मा कम्पाइल गर्दा त्रुटि
**त्रुटि:** `fatal error: Arduino.h: No such file or directory`

**समाधान:**
1. प्रोजेक्टमा `.pio` फोल्डर मेटाउनुहोस्
2. Command Palette बाट "PlatformIO: Rebuild" चलाउनुहोस्
3. `platformio.ini` मा बोर्ड कन्फिगरेसन सही छ कि छैन सुनिश्चित गर्नुहोस्:
   ```ini
   [env:seeed_wio_terminal]
   platform = atmelsam
   board = seeed_wio_terminal
   framework = arduino
   ```

### Grove लाइब्रेरीहरू

#### समस्या: Raspberry Pi मा Grove लाइब्रेरी आयात असफल
**त्रुटि:** `ModuleNotFoundError: No module named 'grove'`

**समाधान:**
1. Grove लाइब्रेरीहरू पुन: इन्स्टल गर्नुहोस्:
   ```bash
   cd ~
   git clone https://github.com/Seeed-Studio/grove.py
   cd grove.py
   sudo pip3 install .
   ```
2. यदि भर्चुअल वातावरण प्रयोग गर्दै हुनुहुन्छ भने, ग्लोबली इन्स्टल गर्नु पर्ने हुन सक्छ वा लाइब्रेरीहरू कपी गर्नुहोस्
3. I2C सक्षम छ कि छैन जाँच्न: `sudo raspi-config nonint do_i2c 0`

#### समस्या: Grove सेंसर पत्ता लाग्दैन
**त्रुटि:** `IOError: [Errno 121] Remote I/O error`

**समाधान:**
1. भौतिक जडानहरू जाँच्नुहोस् (Grove केबल पूर्ण रूपमा जोडिएको छ कि छैन सुनिश्चित गर्नुहोस्)
2. सेंसर सही पोर्टमा जडान छ कि छैन जाँच्नुहोस् (analog, digital, I2C, UART)
3. `i2cdetect -y 1` चलाएर डिभाइस I2C बसमा छ कि छैन हेर्नुहोस्
4. अर्को Grove केबल प्रयोग गरेर हेर्नुहोस्
5. Grove Base Hat Raspberry Pi GPIO पिनहरूमा राम्रोसँग बसेको छ कि छैन जाँच्नुहोस्

## हार्डवेयर समस्या

### Raspberry Pi

#### समस्या: Raspberry Pi बुट हुँदैन
**लक्षणहरू:** कुनै डिस्प्ले छैन, LED काम गर्दैन, वा रेनबो स्क्रीन देखिन्छ

**समाधान:**
1. **पावर सप्लाई जाँच्नुहोस्:** Pi 4 का लागि आधिकारिक 5V 3A USB-C पावर सप्लाई प्रयोग गर्नुहोस्
2. **SD कार्ड समस्या:** 
   - SD कार्ड फर्म्याट गरेर Raspberry Pi OS पुनः इन्स्टल गर्नुहोस्
   - अर्को SD कार्ड प्रयोग गर्नुहोस् (सिफारिस गरिएका ब्रान्डहरू)
   - SD कार्ड राम्रोसँग इन्सर्ट गरिएको छ कि छैन सुनिश्चित गर्नुहोस्
3. **HDMI जडान जाँच्नुहोस्:** Pi 4 का दुबै HDMI पोर्टहरू प्रयास गर्नुहोस्, पावर नजिकको HDMI पोर्ट प्रयोग गर्नुहोस्

#### समस्या: Raspberry Pi मा SSH गर्न सकिँदैन
**लक्षणहरू:** कनेक्शन रिजेक्ट भयो वा टाइमआउट

**समाधान:**
1. SSH सक्षम गर्नुहोस्:
   - Raspberry Pi Imager मा SD कार्ड फ्ल्यास गर्दा advanced options मा SSH कन्फिगर गर्नुहोस्
   - वा बुट पार्टिसनमा नाम नहुने `ssh` नामक खाली फाइल बनाउनुहोस्
2. Pi को IP ठेगाना पत्ता लगाउनुहोस्:
   - आफ्नो राउटरको कनेक्टेड डिभाइस जाँच्नुहोस्
   - `ping raspberrypi.local` प्रयोग गर्नुहोस् (mDNS काम गरेमा)
   - `nmap` वा Angry IP Scanner जस्ता नेटवर्क स्क्यानिङ उपकरण प्रयोग गर्नुहोस्
3. नेटवर्क जाँच्नुहोस्:
   - Pi र तपाईंको कम्प्युटर एउटै नेटवर्कमा छ कि छैन सुनिश्चित गर्नुहोस्
   - WiFi सट्टा ethernet कनेक्शन प्रयास गर्नुहोस्
4. प्रयोगकर्ता नाम/पासवर्ड जाँच्नुहोस् (पूर्वनिर्धारित: प्रयोगकर्ता नाम `pi`, पासवर्ड `raspberry`)

#### समस्या: Grove Base Hat पत्ता लगिएको छैन
**लक्षणहरू:** सेंसरहरू काम गर्दैनन्, I2C त्रुटिहरू

**समाधान:**
1. Base Hat सबै GPIO पिनमा राम्रोसँग बसेको छ कि छैन सुनिश्चित गर्नुहोस्
2. Pi वा Base Hat मा तानिएका पिनहरू छन् कि छैन जाँच्नुहोस्
3. I2C इन्टरफेस सक्षम गर्नुहोस्:
   ```bash
   sudo raspi-config nonint do_i2c 0
   sudo reboot
   ```
4. I2C काम गरिरहेको छ कि छैन पुष्टि गर्नुहोस्: `i2cdetect -y 1`

#### समस्या: Raspberry Pi ढिलो चल्छ
**लक्षणहरू:** UI सुस्त, प्रतिक्रिया ढिलो

**समाधान:**
1. SD कार्डको गतिको जाँच गर्नुहोस् (Class 10 वा राम्रो, वा USB मार्फत SSD)
2. डिस्क स्पेस खाली गर्नुहोस्: `df -h` देखेर अनावश्यक फाइलहरू मेट्नुहोस्
3. `raspi-config` मा GPU मेमोरी कम गर्नुहोस् यदि क्यामेरा/डिस्प्ले धेरै प्रयोग नगरेको हो भने
4. अनावश्यक एप्लिकेशनहरू बन्द गर्नुहोस्
5. Pi 3 वा पुरानो प्रयोग गर्दै हुनुहुन्छ भने Pi 4 मा RAM बढाएर अपग्रेड विचार गर्नुहोस्

### Wio Terminal

#### समस्या: Wio Terminal को पर्दा खाली रहन्छ
**लक्षणहरू:** कोड अपलोड पछि कुनै डिस्प्ले आउँदैन

**समाधान:**
1. कोडले डिस्प्ले आरम्भ गर्छ कि गर्दैन जाँच्नुहोस् (TFT_eSPI लाइब्रेरी)
2. [Seeed Wiki](https://wiki.seeedstudio.com/Wio-Terminal-Getting-Started/) बाट Wio Terminal फर्मवेयर अपडेट गर्नुहोस्
3. डिस्प्ले आरम्भ गर्ने कोड थप्नुहोस्:
   ```cpp
   #include <TFT_eSPI.h>
   TFT_eSPI tft;
   tft.begin();
   tft.fillScreen(TFT_BLACK);
   ```
4. PlatformIO बाट परीक्षण स्केच अपलोड गरेर हार्डवेयर परीक्षण गर्नुहोस्

#### समस्या: Wio Terminal मा WiFi काम गर्दैन
**लक्षणहरू:** WiFi सँग जोड्न सकिदैन, नेटवर्क त्रुटिहरू

**समाधान:**
1. WiFi फर्मवेयर अपडेट गर्नुहोस्: [Wio Terminal WiFi firmware update guide](https://wiki.seeedstudio.com/Wio-Terminal-Network-Overview/) पालना गर्नुहोस्
2. WiFi क्रेडेन्सियलहरू जाँच गर्नुहोस्: SSID र पासवर्ड सही छन् कि छैनन् सुनिश्चित गर्नुहोस्
3. WiFi ब्यान्ड: Wio Terminal ले मात्र 2.4GHz WiFi समर्थन गर्छ (5GHz होइन)
4. सङ्केत शक्ति: राउटर नजिक लैजानुस्
5. राउटर सेटिङहरू: केही enterprise/WPA-Enterprise नेटवर्क काम नगर्न सक्छन्

#### समस्या: कम्प्युटरले Wio Terminal पत्ता लगाउँदैन
**लक्षणहरू:** USB डिभाइस भेटिन्न

**समाधान:**
1. फरक USB केबल प्रयास गर्नुहोस्: डाटा केबल प्रयोग गर्नुहोस्, चार्ज मात्र केबल होइन
2. बुटलोडर मोडमा जानुहोस्: पावर स्विच छिटो दुई पटक तल गर्नुहोस्
   - निलो LED पल्प हुनु पर्छ, डिभाइस Device Manager मा "Arduino" रूपमा देखिन्छ
3. ड्राइभर इन्स्टल गर्नुहोस् (Windows):
   - [Seeed USB driver](https://wiki.seeedstudio.com/Driver_for_Seeeduino/) डाउनलोड र इन्स्टल गर्नुहोस्
4. फरक USB पोर्ट प्रयास गर्नुहोस्: USB हबहरूबाट बच्नुहोस्, सिधा जडान गर्नुहोस्
5. सिस्टम USB ड्राइभर अपडेट गर्नुहोस्

#### समस्या: Wio Terminal मा सेंसर काम गर्दैन
**लक्षणहरू:** Grove सेंसर डेटा पढ्दैन

**समाधान:**
1. Grove केबल जडान जाँच्नुहोस्
2. सही Grove पोर्ट (बायाँ वा दायाँ) प्रयोग गरिरहनुभएको छ कि छैन पुष्टि गर्नुहोस्
3. सेंसरका लागि सही लाइब्रेरीहरू समावेश गर्नुहोस्
4. सेंसर पावर आवश्यकताहरू चेक गर्नुहोस्
5. लाइब्रेरीबाट उदाहरण कोड प्रयोग गरेर सेंसर परीक्षण गर्नुहोस्

### भर्चुअल उपकरण (CounterFit)

#### समस्या: CounterFit एप सुरु हुँदैन
**त्रुटि:** CounterFit सुरु गर्दा विभिन्न Python त्रुटिहरू

**समाधान:**
1. भर्चुअल वातावरण सक्रिय छ कि छैन सुनिश्चित गर्नुहोस्
2. CounterFit इन्स्टल/पुनःइन्स्टल गर्नुहोस्:
   ```bash
   pip install CounterFit
   ```
3. पोर्ट 5000 पहिले नै प्रयोगमा छैन कि होइन जाँच्नुहोस्:
   - Windows: `netstat -ano | findstr :5000`
   - macOS/Linux: `lsof -i :5000`
4. पोर्ट 5000 प्रयोग गरिरहेको प्रक्रिया बन्द गर्नुहोस् वा फरक पोर्ट प्रयोग गर्नुहोस्:
   ```bash
   counterfit --port 5001
   ```

#### समस्या: कोडबाट CounterFit सँग कनेक्ट गर्न सकिँदैन
**त्रुटि:** कनेक्शन रिजेक्ट भयो वा टाइमआउट

**समाधान:**
1. CounterFit चलिरहेको छ कि छैन जाँच्न: ब्राउजरमा `http://127.0.0.1:5000` खोल्नुहोस्
2. कोडमा कनेक्शन URL CounterFit ठेगानासँग मेल खान्छ कि छैन जाँच्नुहोस्
3. फायरवाल कनेक्शन ब्लक गर्दै छ कि छैन सुनिश्चित गर्नुहोस्
4. CounterFit एप र तपाईंको कोड दुबै पुनः सुरु गरेर प्रयास गर्नुहोस्

#### समस्या: CounterFit मा सेंसरहरू देखिन्नन्
**लक्षणहरू:** सिर्जना गरिएका सेंसर CounterFit UI मा देखिन्नन्

**समाधान:**
1. कोड चलाउनु अघि CounterFit UI मा सेंसरहरू सिर्जना गर्नुहोस्
2. ब्राउजर पेज रिफ्रेश गर्नुहोस्
3. सेंसर प्रकार कोडसँग मेल खान्छ कि छैन जाँच्नुहोस्
4. ब्राउजर क्यास सफा गर्नुहोस्

## कनेक्टिभिटी समस्या

### WiFi कनेक्शन

#### समस्या: डिभाइस WiFi सँग जडान गर्न सक्दैन
**लक्षणहरू:** कनेक्शन टाइमआउट, प्रमाणीकरण असफल

**समाधान:**
1. **SSID र पासवर्ड जाँच गर्नुहोस्:** क्रेडेन्सियलहरू सही छन् कि छैनन् सुनिश्चित गर्नुहोस्
2. **WiFi ब्यान्ड:** अधिकांश IoT डिभाइसहरूले मात्र 2.4GHz समर्थन गर्छन् (5GHz होइन)
3. **राउटर सेटिङहरू:**
   - यदि AP isolation सक्षम छ भने निष्क्रिय पार्नुहोस्
   - WPA2-PSK सुरक्षा प्रयोग गर्नुहोस् (WPA3, WEP वा खुला नेटवर्क टाल्नुहोस्)
   - DHCP सक्षम छ कि छैन सुनिश्चित गर्नुहोस्
4. **लुकेको नेटवर्कहरू:** यदि SSID लुकेको छ भने स्पष्ट रूपमा कन्फिगर गर्नु आवश्यक पर्छ
5. **सङ्केत शक्ति:** डिभाइसलाई राउटर नजिक लैजानुहोस्
6. **करपटी:** अन्य उपकरणहरू, माइक्रोवेभ, वा भित्ताहरूले करपटी गर्न सक्छन्

#### समस्या: WiFi कनेक्शन बारम्बार छुट्छ
**लक्षणहरू:** कनेक्टिभिटी विच्छेद

**समाधान:**
1. राउटर स्थिर छ कि छैन जाँच गर्नुहोस् र आवश्यकता परे पुनः सुरु गर्नुहोस्
2. डिभाइस फर्मवेयर अपडेट गर्नुहोस्
3. DHCP को सट्टा स्थिर IP प्रयोग गर्नुहोस्
4. राउटर नजिक जानुहोस् वा WiFi एक्सटेन्डर थप्नुहोस्
5. अन्य डिभाइसहरूबाट करपटी छ कि छैन जाँच्नुहोस्
6. पावर सप्लाई पर्याप्त छ कि छैन सुनिश्चित गर्नुहोस् (विशेष गरी Raspberry Pi को लागि)

### क्लाउड सेवा

#### समस्या: Azure IoT Hub सँग कनेक्ट गर्न सकिँदैन
**त्रुटि:** प्रमाणीकरण असफल, कनेक्शन रिजेक्ट भयो

**समाधान:**
1. **क्रेडेन्सियलहरू प्रमाणित गर्नुहोस्:**
   - कनेक्शन स्ट्रिङ सही छ कि छैन जाँच्नुहोस्
   - कनेक्शन स्ट्रिङमा थप स्पेस वा लाइन ब्रेक छैन भनेर सुनिश्चित गर्नुहोस्
2. **डिभाइस दर्ता जाँच्नुहोस्:** डिभाइस IoT Hub मा दर्ता गरिएको हुनुपर्छ
3. **फायरवाल/प्रोक्सी जाँच्नुहोस्:** बाहिर जाने MQTT (पोर्ट 8883) वा HTTPS (पोर्ट 443) अनुमति भएको छ कि छैन हेर्नुहोस्
4. **IoT Hub क्षेत्र:** IoT Hub चलिरहेको छ कि छैन र फरक क्षेत्रमा नभई प्रतिध्वनि (लेटेन्सी) नहोस् सुनिश्चित गर्नुहोस्
5. **कोटा सीमा:** निःशुल्क स्तरको सीमा पार भईसकेको छ कि छैन जाँच्नुहोस्
6. **कनेक्शन परीक्षण:**
   ```bash
   az iot hub device-identity show-connection-string --hub-name YourIoTHub --device-id YourDevice
   ```

#### समस्या: Azure Functions ट्रिगर हुँदैन
**लक्षणहरू:** मेसेजहरू पठाइन्छ तर फंक्शन Execute हुँदैन

**समाधान:**
1. Function App चलिरहेको छ (रोकिएको छैन) कि छैन जाँच्नुस्
2. Function App सेटिङमा कनेक्शन स्ट्रिङ सही छ कि छैन सुनिश्चित गर्नुहोस्
3. Azure Portal मा फंक्शन लगहरू जाँच गर्नुहोस्
4. Event Hub compatible endpoint सही रूपमा कन्फिगर छ कि छैन हेर्नुहोस्
5. मेसेज फर्म्याट फंक्शनसँग मिल्दोजुल्दो छ कि छैन जाँच्नुहोस्
6. Function App सेवा योजना (consumption वा dedicated) विचार गर्नुहोस्

### MQTT
#### समस्या: MQTT जडान असफल हुन्छ  
**त्रुटि:** जडान अस्वीकार गरियो, प्रमाणीकरण असफल

**समाधान:**  
1. **ब्रोकरे ठेगाना:** ब्रोकरे URL/IP सही छ भनेर जाँच गर्नुहोस्  
2. **पोर्ट:** पोर्ट नम्बर जाँच्नुहोस् (1883 बिना इन्क्रिप्टेड, 8883 TLS को लागि)  
3. **प्रमाणीकरण:** आवश्यक भएमा प्रयोगकर्ता नाम/पासवर्ड जाँच गर्नुहोस्  
4. **TLS/SSL:** प्रमाणपत्रहरू मान्य र विश्वासयुक्त छन् भनेर सुनिश्चित गर्नुहोस्  
5. **फायरवाल:** पोर्ट ब्लक गरिएको छैन भनेर जाँच गर्नुहोस्  
6. **MQTT क्लाइन्टसँग परीक्षण गर्नुहोस्:** MQTT Explorer वा mosquitto_pub/sub प्रयोग गरेर परीक्षण गर्नुहोस्  

#### समस्या: MQTT सन्देशहरू प्राप्त हुँदैनन्  
**लक्षणहरू:** सन्देशहरू प्रकाशित गरिएका छन् तर सब्सक्राइबरले प्राप्त गर्न सकेका छैनन्

**समाधान:**  
1. **टोपिक नामहरू:** सब्सक्राइबर टोपिकले प्रकाशक टोपिकसँग ठीकसँग मेल खान्छ भनी जाँच्नुहोस्  
2. **QoS स्तर:** QoS 0 को सट्टा QoS 1 वा 2 प्रयास गर्नुहोस्  
3. **वाइल्डकार्डहरू:** टोपिक वाइल्डकार्डहरू सहि प्रयोग भएको छ कि छैन जाँच्नुहोस् (`+` एकल तहको लागि, `#` बहु तहको लागि)  
4. **रिटेन्ड सन्देशहरू:** प्रकाशकले अन्तिम सन्देश राख्न रिटेन्ड फ्ल्याग सेट गर्न सक्छ  
5. **जडान समय:** सब्सक्राइबरले सन्देशहरू प्रकाशित हुनु अघि जडान हुन्छ भन्ने सुनिश्चित गर्नुहोस्

## सेन्सर र एक्ट्युएटर समस्याहरू  

### ग्रोभ सेन्सरहरू  

#### समस्या: सेन्सरले गलत मानहरू प्रतिक्रिया गर्छ  
**लक्षणहरू:** रिडिङहरू 0, -1, वा अर्थहीन मानहरू देखिन्छन्

**समाधान:**  
1. **जडानहरू जाँच्नुहोस्:** सेन्सर ठीकसँग जडान भएको सुनिश्चित गर्नुहोस्  
2. **सही पोर्ट:** सेन्सर सही पोर्ट प्रकारमा छ जाँच्नुहोस्:  
   - अनालग सेन्सरहरू → अनालग पोर्टहरू (A0, A2, A4)  
   - डिजिटल सेन्सरहरू → डिजिटल पोर्टहरू (D5, D16, D18 आदि)  
   - I2C सेन्सरहरू → I2C पोर्टहरू  
3. **क्यालिब्रेसन:** केहि सेन्सरहरूले क्यालिब्रेसन आवश्यक पर्दछ (माटोको नमी, प्रकाश)  
4. **पावर साइकल:** सेन्सर अन कनेक्सन विच्छेद गरेर फेरि जडान गर्नुहोस्  
5. **सेन्सर डेटा शीट:** सेन्सरका विशिष्टता र आवश्यकताहरू जाँच्नुहोस्  

#### समस्या: क्षैतिक माटो नमी सेन्सर सधैं भिजेको पढ्छ  
**लक्षणहरू:** सेन्सर सुक्खा हुँदा पनि उच्च नमी देखाउँछ

**समाधान:**  
1. **क्यालिब्रेसन आवश्यक:** माटो सेन्सरहरूले क्यालिब्रेसन आवश्यक पर्दछ:  
   - हावामा मान पढ्नुहोस् (सुक्खा आधाररेखा)  
   - पानीमा मान पढ्नुहोस् (भिजेको आधाररेखा)  
   - रिडिङहरू यी मानहरू बीच नक्सांकन गर्नुहोस्  
2. **सेन्सर कोटिंग जाँच्नुहोस्:** नमी सेन्सरको कोटिंग बिग्रिएको हुन सक्छ  
3. **स्थान:** सेन्सर माटोमा पूर्ण रूपमा जडान भएको सुनिश्चित गर्नुहोस्  

#### समस्या: तापमान/आर्द्रता सेन्सर रिडिङ गलत  
**लक्षणहरू:** DHT11/DHT22 मा तापमान वा आर्द्रता गलत देखाउँछ

**समाधान:**  
1. **सेन्सरको स्थान:** प्रत्यक्ष घाम, तातो स्रोत वा हावा प्रवाहबाट जोगिनुहोस्  
2. **वार्म-अप समय:** पावर अन भएपछि सेन्सरलाई २ सेकेन्ड दिनुहोस् पढ्न अघि  
3. **पढ्ने आवृत्ति:** DHT सेन्सरहरूले पढाइबीच कम्तिमा २ सेकेन्ड समय चाहिन्छ  
4. **संघनन जाँच्नुहोस्:** संघननले रिडिङलाई असर गर्न सक्छ  
5. **सेन्सर गुणस्तर:** DHT11 DHT22 भन्दा कम सही हुन्छ

### माइक्रोफोन र स्पिकर  

#### समस्या: अडियो इनपुट/आउटपुट छैन  
**लक्षणहरू:** माइक्रोफोन रेकर्ड गर्दैन, स्पिकर आवाज निकाल्दैन

**समाधान:**  
1. **जडानहरू जाँच्नुहोस्:** अडियो उपकरणहरू ठीकसँग जडान भएका छन् भनेर पुष्टि गर्नुहोस्  
2. **हार्डवेयर परीक्षण गर्नुहोस्:**  
   - स्पिकर: `speaker-test -t wav -c 2`

3. **भोल्युम सेटिङहरू:** जाँच गर्नुहोस् र समायोजन गर्नुहोस्:  
   ```bash
   alsamixer
   ```
4. **अडियो उपकरण चयन:** कोडमा सही अडियो उपकरण निर्दिष्ट गर्नुहोस्  
5. **ड्राइभर समस्या:** ALSA अपडेट गर्नुहोस् वा अडियो ड्राइभर पुन: स्थापना गर्नुहोस्  

#### समस्या: ReSpeaker ह्याट काम गरिरहेको छैन  
**लक्षणहरू:** अडियो उपकरण निदान भएको छैन

**समाधान:**  
1. **ड्राइभर स्थापना गर्नुहोस्:**  
   ```bash
   git clone https://github.com/HinTak/seeed-voicecard
   cd seeed-voicecard
   sudo ./install.sh
   sudo reboot
   ```
2. **स्थापना जाँच्नुहोस्:** `arecord -l` ले ReSpeaker देखाउनु पर्नेछ  
3. **फर्मवेयर अपडेट गर्नुहोस्:** केहि Pi OS भर्सनहरूले ड्राइभर अपडेट आवश्यक पर्छ  
4. **सिटिङ जाँच्नुहोस्:** ह्याट GPIO पिनहरूमा राम्रोसँग जडान भएको सुनिश्चित गर्नुहोस्

### VS कोड  

#### समस्या: टर्मिनलले भर्चुअल वातावरण स्वचालित सक्रिय गर्दैन  
**लक्षणहरू:** टर्मिनल खोलिन्छ तर venv सक्रिय छैन

**समाधान:**  
1. **Python इन्टरप्रिटर सेट गर्नुहोस्:** Command Palette → "Python: Select Interpreter" → venv रोज्नुहोस्  
2. इन्टरप्रिटर चयन पछि VS Code पुनः सुरु गर्नुहोस्  
3. `settings.json` मा थप्नुहोस्:  
   ```json
   "python.terminal.activateEnvironment": true
   ```
  
#### समस्या: उपकरणमा कोड चलिरहेको छैन  
**लक्षणहरू:** कोड चल्दैछ तर उपकरणमा केही हुँदैन

**समाधान:**  
1. **कोड सेभ गरिएको छ कि छैन जाँच्नुहोस्** (फाइल ट्याबमा डट हेर्नुहोस्)  
2. कुन Python चलिरहेको छ जाँच्नुहोस्: `which python` वा `where python`  
3. **Wio Terminal का लागि:** PlatformIO मार्फत कोड अपलोड गरिएको छ सुनिश्चित गर्नुहोस् (अपलोड बटन क्लिक गर्नुहोस्)  
4. **Raspberry Pi का लागि:** Pi मा SSH गरेर त्यहाँबाट कोड चलाउनुहोस्  
5. **आउटपुट विन्डोमा** त्रुटिहरू जाँच्नुहोस्  

#### समस्या: IntelliSense ले लाइब्रेरी फङ्क्शन देखाउँदैन  
**लक्षणहरू:** आयात गरिएका मोड्युलहरूको लागि स्वतः पूर्ति छैन

**समाधान:**  
1. लाइब्रेरी हालको वातावरणमा स्थापना गरिएको छ सुनिश्चित गर्नुहोस्  
2. VS Code विन्डो रीलोड गर्नुहोस्  
3. Python इन्टरप्रिटर सही छ कि छैन जाँच्नुहोस्  
4. उपलब्ध भएमा type stubs स्थापना गर्नुहोस्: `pip install types-<library-name>`  

### Python भर्चुअल वातावरणहरू  

#### समस्या: भर्चुअल वातावरण बनाउन सकिंदैन  
**त्रुटि:** `The virtual environment was not created successfully`

**समाधान:**  
1. **venv मोड्युल स्थापना गर्नुहोस्:**  
   - Ubuntu/Debian: `sudo apt install python3-venv`  
   - macOS: Python सँगै समावेश हुन्छ  
   - Windows: सबै कम्पोनेन्टसहित Python पुन: स्थापना गर्नुहोस्  
2. Python स्थापना सहि छ कि छैन जाँच गर्नुहोस्  
3. पूर्ण पथ प्रयोग गर्नुहोस्: `python3 -m venv .venv` स्पष्ट python3 कलसँग प्रयास गर्नुहोस्  

#### समस्या: प्याकेजहरू गलत स्थानमा स्थापना हुन्छन्  
**लक्षणहरू:** प्याकेज स्थापना पछि आयात त्रुटि

**समाधान:**  
1. **venv सक्रिय छ कि छैन जाँच्नुहोस्:** कमाण्ड प्रम्प्टमा `(.venv)` देखिनुपर्छ  
2. **pip स्थान जाँच्नुहोस्:** `which pip` `.venv/bin/pip` देखाउनुपर्छ  
3. **venv मा पुनः स्थापना गर्नुहोस्:** venv सक्रिय गरेर `pip install <package>` चलाउनुहोस्  
4. **venv मा sudo प्रयोग नगर्नुहोस्**  

#### समस्या: भर्चुअल वातावरण पोर्टेबल छैन  
**लक्षणहरू:** venv सार्दा वा फरक कम्प्युटरमा काम गर्दैन

**समाधान:**  
1. **venv सार्नु हुँदैन:** नयाँ स्थानमा मेटेर फेरि बनाउनुहोस्  
2. **requirements.txt प्रयोग गर्नुहोस्:**  
   ```bash
   pip freeze > requirements.txt
   pip install -r requirements.txt
   ```
3. **venv पुनः सिर्जना गर्नुहोस्:**  
   ```bash
   python3 -m venv .venv
   source .venv/bin/activate  # वा Windows मा activate.bat
   pip install -r requirements.txt
   ```
  
### निर्भरता  

#### समस्या: प्याकेज स्थापना असफल  
**त्रुटि:** स्थापना क्रममा विभिन्न pip त्रुटिहरू

**समाधान:**  
1. **pip अपडेट गर्नुहोस्:**  
   ```bash
   pip install --upgrade pip
   ```
2. **बिल्ड उपकरणहरू स्थापना गर्नुहोस्:**  
   - Ubuntu/Debian: `sudo apt install build-essential python3-dev`  
   - macOS: `xcode-select --install`  
   - Windows: Visual Studio Build Tools स्थापना गर्नुहोस्  
3. इन्टरनेट जडान जाँच्नुहोस्  
4. फरक प्याकेज इन्डेक्स प्रयोग गरेर प्रयास गर्नुहोस्: `pip install --index-url https://pypi.org/simple/ <package>`  
5. विशेष संस्करण स्थापना गर्नुहोस्: `pip install <package>==<version>`  

#### समस्या: निर्भरता द्वन्द्व  
**त्रुटि:** `ERROR: pip's dependency resolver does not currently take into account all the packages that are installed`

**समाधान:**  
1. परियोजनाको लागि नयाँ भर्चुअल वातावरण प्रयोग गर्नुहोस्  
2. प्याकेजहरू अपडेट गर्नुहोस्: `pip install --upgrade <package>`  
3. द्वन्द्वहरू पत्ता लगाउन `pip check` चलाउनुहोस्  
4. अनुकूल संस्करणहरू स्थापना गर्नुहोस्: requirements.txt मा संस्करण दायराहरू निर्दिष्ट गर्नुहोस्

## प्रदर्शन समस्याहरू  

### समस्या: कोड ढिलो चलिरहेको छ  
**लक्षणहरू:** ढिलाइ, टाइमआउट, अनुत्तरदायी व्यवहार

**समाधान:**  
1. सेन्सर पढ्ने आवृत्ति कम गर्नुहोस्: सेन्सरहरू धेरै पटक नपढ्नुहोस्  
2. लूपहरू अनुकूलन गर्नुहोस्: ब्य Busy-waiting बाट बच्नुहोस्, sleep() वा विलम्ब प्रयोग गर्नुहोस्  
3. मेमोरी समस्या:  
   - अनावश्यक एप्लिकेशन बन्द गर्नुहोस्  
   - भण्डारण ठाउँ खाली गर्नुहोस्  
   - Pi मा `top` वा `htop` सँग अनुगमन गर्नुहोस्  
4. SD कार्ड गति: Raspberry Pi को लागि छिटो SD कार्ड वा SSD प्रयोग गर्नुहोस्  
5. नेटवर्क ढिलाइ: नेटवर्क कलहरूका लागि async अपरेसनहरू प्रयोग गर्नुहोस्  

### समस्या: मेमोरी 부족 त्रुटि  
**त्रुटि:** `MemoryError` वा प्रणाली फ्रिज

**समाधान:**  
1. **Raspberry Pi का लागि:**  
   - अनावश्यक एप्लिकेशन बन्द गर्नुहोस्  
   - स्वाप स्पेस बढाउनुहोस्  
   - हल्का OS प्रयोग गर्नुहोस् (Lite भर्सन)  
   - RAM अपग्रेड गर्नुहोस् (Pi 4 मा 2/4/8GB विकल्प)  
2. **Wio Terminal का लागि:**  
   - बफर साइजहरू कम गर्नुहोस्  
   - साना छविहरू प्रयोग गर्नुहोस्  
   - स्ट्रिङ उपयोग अनुकूलन गर्नुहोस्  
   - मेमोरी लीकका लागि जाँच्नुहोस् (रिलिज नभएको मेमोरी)  

### समस्या: डाटा गुमाउनु वा भ्रष्ट हुनु  
**लक्षणहरू:** सन्देशहरू हराइरहेको, फाइलहरू भ्रष्ट

**समाधान:**  
1. SD कार्ड समस्या:  
   - गुणस्तरीय SD कार्ड प्रयोग गर्नुहोस् (सस्तो/काउन्टरफिटबाट बच्नुहोस्)  
   - नियमित ब्याकअप गर्नुहोस्  
   - सफा शटडाउन गर्नुहोस् (पावर नअटोर्नुहोस्)  
2. बफर ओभरफ्लो: कोडमा बफर साइज बढाउनुहोस्  
3. नेटवर्क विश्वसनीयता: पुन: प्रयास गर्ने तर्क र त्रुटि ह्यान्डलिङ कार्यान्वयन गर्नुहोस्  
4. सेवा गुणस्तर: महत्वपूर्ण सन्देशहरूको लागि MQTT QoS 1 वा 2 प्रयोग गर्नुहोस्

## सामान्य त्रुटि सन्देशहरू  

### `ModuleNotFoundError: No module named 'X'`  
**कारण:** प्याकेज स्थापना गरिएको छैन वा भर्चुअल वातावरण सक्रिय गरिएको छैन

**समाधान:**  
```bash
pip install X
```
सुरुमा भर्चुअल वातावरण सक्रिय भएको सुनिश्चित गर्नुहोस्।

### Linux/macOS मा `Permission denied`  
**कारण:** उच्च अनुमति आवश्यक वा फाइल अनुमतिमा समस्या

**समाधान:**  
- प्रणाली अपरेशनका लागि: `sudo` प्रयोग गर्नुहोस्  
- pip को लागि: venv सक्रिय गरेर sudo नगर्नुहोस्  
- सिरियल पोर्टका लागि: प्रयोगकर्तालाई dialout समूहमा थप्नुहोस्: `sudo usermod -a -G dialout $USER`, त्यसपछि logout/login गर्नुहोस्  

### `OSError: [Errno 98] Address already in use`  
**कारण:** पोर्ट अर्को प्रक्रियाले पहिले नै प्रयोग गरिरहेको छ

**समाधान:**  
1. पोर्ट प्रयोग गर्ने प्रक्रिया पत्ता लगाउनुहोस्: `lsof -i :<port>` वा `netstat -ano | findstr :<port>`  
2. प्रक्रिया रोक्नुहोस् वा कोडमा फरक पोर्ट प्रयोग गर्नुहोस्  

### `SSL: CERTIFICATE_VERIFY_FAILED`  
**कारण:** SSL प्रमाणपत्र मान्यता असफल

**समाधान:**  
1. प्रमाणपत्र अपडेट गर्नुहोस्: `pip install --upgrade certifi`  
2. प्रणाली समय सही छ कि छैन जाँच्नुहोस्: `date`  
3. विकासका लागि मात्र (उत्पादन होइन): कोडमा मान्यता अक्षम गर्नुहोस्  

### `IndentationError: unexpected indent`  
**कारण:** Python इन्डेन्टेशन समस्या (tabs/spaces मिश्रण)

**समाधान:**  
1. एकरूप इन्डेन्टेशन प्रयोग गर्नुहोस् (4 स्पेस Python मानक हो)  
2. एडिटरलाई tabs को सट्टा spaces प्रयोग गर्न सेट गर्नुहोस्  
3. VS Code मा: `"editor.insertSpaces": true` र `"editor.tabSize": 4` सेट गर्नुहोस्  

### `UnicodeDecodeError` वा `UnicodeEncodeError`  
**कारण:** क्यारक्टर इन्कोडिङ सम्बन्धी समस्या

**समाधान:**  
```python
# फाइलहरू पढ्दा
with open('file.txt', 'r', encoding='utf-8') as f:
    content = f.read()

# फाइलहरू लेख्दा
with open('file.txt', 'w', encoding='utf-8') as f:
    f.write(content)
```
