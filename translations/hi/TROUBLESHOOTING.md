## इंस्टॉलेशन समस्याएँ

### पाइथन इंस्टॉलेशन

#### समस्या: पाइथन संस्करण बहुत पुराना है
**त्रुटि:** `Python 3.6 या उससे ऊपर का संस्करण आवश्यक है`

**समाधान:**
1. नवीनतम Python 3 डाउनलोड करें [python.org](https://www.python.org/downloads/)
2. विंडोज पर इंस्टॉलेशन के दौरान, "Add Python to PATH" विकल्प चुनें
3. इंस्टॉलेशन सत्यापित करें:
   ```bash
   python3 --version
   ```

#### समस्या: कई पाइथन संस्करण टकराव कर रहे हैं
**लक्षण:** गलत पाइथन संस्करण चलता है, पैकेज गलत स्थान पर इंस्टॉल हो रहे हैं

**समाधान:**
- **विंडोज:** Python 3 को स्पष्ट रूप से कॉल करने के लिए `python` के बजाय `py -3` का उपयोग करें
- **macOS/Linux:** `python3` का उपयोग करें, `python` के बजाय
- प्रोजेक्ट्स के लिए हमेशा वर्चुअल एन्वायरनमेंट बनाएं और उपयोग करें

#### समस्या: pip कमांड नहीं मिला
**त्रुटि:** `'pip' एक आंतरिक या बाहरी कमांड के रूप में मान्यता प्राप्त नहीं है`

**समाधान:**
1. pip के बजाय `pip3` आज़माएँ
2. या `python -m pip` या `python3 -m pip` का उपयोग करें
3. सुनिश्चित करें कि Python PATH में जोड़ा गया है (पुनः इंस्टॉल करें और विकल्प जांचें)

### VS कोड और एक्सटेंशन्स

#### समस्या: Pylance एक्सटेंशन काम नहीं कर रहा
**लक्षण:** कोई Python IntelliSense, कोड पूर्णता या टाइप चेकिंग नहीं हो रहा

**समाधान:**
1. VS कोड कमांड पैलेट खोलें (`Ctrl+Shift+P` या `Cmd+Shift+P`)
2. "Python: Select Interpreter" चलाएं
3. सही Python इंटरप्रेटर चुनें (यदि वर्चुअल एन्वायरनमेंट उपयोग कर रहे हैं तो वही)
4. VS कोड विंडो को पुनः लोड करें

#### समस्या: VS कोड वर्चुअल एन्वायरनमेंट का पता नहीं लगाता
**लक्षण:** गलत Python इंटरप्रेटर चुना गया है

**समाधान:**
1. टर्मिनल में वर्चुअल एन्वायरनमेंट सक्रिय करें
2. कमांड पैलेट खोलें और "Python: Select Interpreter" चलाएं
3. `.venv` फोल्डर से इंटरप्रेटर चुनें
4. स्टेटस बार (नीचे बाएं) में सही Python संस्करण दिख रहा है, जांचें

### PlatformIO (Wio Terminal)

#### समस्या: PlatformIO इंस्टॉलेशन विफल होता है
**त्रुटि:** PlatformIO इंस्टॉलेशन के दौरान विभिन्न त्रुटियाँ

**समाधान:**
1. सुनिश्चित करें कि VS कोड अपडेटेड है
2. पहले C/C++ एक्सटेंशन इंस्टॉल करें
3. PlatformIO इंस्टॉल करने के बाद VS कोड रीस्टार्ट करें
4. अपनी इंटरनेट कनेक्शन जांचें (PlatformIO बड़े फ़ाइल डाउनलोड करता है)

#### समस्या: Board को PlatformIO द्वारा पता नहीं लगाया जाता
**लक्षण:** Wio Terminal पर कोड अपलोड नहीं हो रहा

**समाधान:**
1. अलग USB केबल आज़माएँ (कुछ केबल केवल चार्जिंग के लिए होते हैं)
2. डिवाइस मैनेजर (विंडोज) या `ls /dev/tty*` (macOS/Linux) जांचें
3. USB ड्राइवर इंस्टॉल या अपडेट करें
4. अलग USB पोर्ट आज़माएँ
5. Wio Terminal पर पावर स्विच दो बार जल्दी स्लाइड करें ताकि बूटलोडर मोड में जाएँ

#### समस्या: PlatformIO में कम्पाइल त्रुटियाँ
**त्रुटि:** `fatal error: Arduino.h: No such file or directory`

**समाधान:**
1. अपने प्रोजेक्ट में `.pio` फोल्डर हटाएं
2. कमांड पैलेट से "PlatformIO: Rebuild" चलाएं
3. सुनिश्चित करें कि `platformio.ini` में सही बोर्ड कॉन्फ़िगरेशन है:
   ```ini
   [env:seeed_wio_terminal]
   platform = atmelsam
   board = seeed_wio_terminal
   framework = arduino
   ```

### Grove लाइब्रेरीज

#### समस्या: Raspberry Pi पर Grove लाइब्रेरी इम्पोर्ट विफल
**त्रुटि:** `ModuleNotFoundError: No module named 'grove'`

**समाधान:**
1. Grove लाइब्रेरीज पुनः इंस्टॉल करें:
   ```bash
   cd ~
   git clone https://github.com/Seeed-Studio/grove.py
   cd grove.py
   sudo pip3 install .
   ```
2. यदि वर्चुअल एन्वायरनमेंट उपयोग कर रहे हैं, तो वैश्विक रूप से इंस्टॉल करें या लाइब्रेरीज कॉपी करें
3. I2C सक्षम है यह सत्यापित करें: `sudo raspi-config nonint do_i2c 0`

#### समस्या: Grove सेंसर पता नहीं चलता
**त्रुटि:** `IOError: [Errno 121] Remote I/O error`

**समाधान:**
1. भौतिक कनेक्शनों की जांच करें (सुनिश्चित करें Grove केबल पूरी तरह से लगा हुआ है)
2. सेंसर सही पोर्ट से जुड़ा है यह जांचें (एनालॉग, डिजिटल, I2C, UART)
3. `i2cdetect -y 1` चलाकर देखें कि डिवाइस I2C बस पर दिख रहा है या नहीं
4. अलग Grove केबल आज़माएँ
5. सुनिश्चित करें कि Grove Base Hat ठीक से Raspberry Pi GPIO पिनों पर लगा है

## हार्डवेयर समस्याएँ

### रास्पबेरी पाई

#### समस्या: रास्पबेरी पाई बूट नहीं हो रहा
**लक्षण:** कोई डिस्प्ले, LED गतिविधि नहीं, या रेनबो स्क्रीन

**समाधान:**
1. **पावर सप्लाई जांचें:** Pi 4 के लिए आधिकारिक 5V 3A USB-C पावर सप्लाई का उपयोग करें
2. **SD कार्ड समस्याएं:**
   - SD कार्ड रीफॉर्मेट करें और Raspberry Pi OS पुनः इंस्टॉल करें
   - अलग SD कार्ड आज़माएँ (अनुशंसित ब्रांड का उपयोग करें)
   - सुनिश्चित करें SD कार्ड ठीक से लगा हुआ है
3. **HDMI कनेक्शन जांचें:** Pi 4 के दोनों HDMI पोर्ट्स आज़माएँ, पावर के पास वाला HDMI पोर्ट उपयोग करें

#### समस्या: Raspberry Pi में SSH नहीं हो रहा
**लक्षण:** कनेक्शन अस्वीकृत या टाइमआउट

**समाधान:**
1. SSH सक्षम करें:
   - SD कार्ड फ्लैश करते समय Raspberry Pi Imager में एडवांस विकल्प से SSH कॉन्फ़िगर करें
   - या बूट पार्टीशन में `ssh` नाम की खाली फ़ाइल बनाएं (कोई एक्सटेंशन नहीं)
2. Pi का IP पता ढूंढ़ें:
   - अपने राउटर के कनेक्टेड डिवाइस देखें
   - `ping raspberrypi.local` (यदि mDNS काम कर रहा हो)
   - नेटवर्क स्कैनिंग टूल्स जैसे `nmap` या Angry IP Scanner का उपयोग करें
3. नेटवर्क जांचें:
   - सुनिश्चित करें Pi आपके कंप्यूटर के समान नेटवर्क पर है
   - वाईफ़ाई के बजाय ईथरनेट कनेक्शन आज़माएँ
4. उपयोगकर्ता नाम/पासवर्ड सत्यापित करें (डिफॉल्ट: यूजरनेम `pi`, पासवर्ड `raspberry`)

#### समस्या: Grove Base Hat पहचाना नहीं जाता
**लक्षण:** सेंसर काम नहीं कर रहे, I2C त्रुटियाँ

**समाधान:**
1. सुनिश्चित करें Base Hat सभी GPIO पिनों पर ठीक से लगा है
2. Pi या Base Hat पर मोड़े हुए पिन जांचें
3. I2C इंटरफ़ेस सक्षम करें:
   ```bash
   sudo raspi-config nonint do_i2c 0
   sudo reboot
   ```
4. I2C काम कर रहा है यह सत्यापित करें: `i2cdetect -y 1`

#### समस्या: रास्पबेरी पाई धीमा चल रहा है
**लक्षण:** UI में लैग, धीमा प्रतिक्रिया

**समाधान:**
1. SD कार्ड की गति जांचें (क्लास 10 या बेहतर, या USB के माध्यम से SSD)
2. डिस्क स्पेस खाली करें: `df -h` से जांचें, अनावश्यक फ़ाइलें हटाएं
3. यदि कैमरा/डिस्प्ले भारी उपयोग नहीं होता, तो `raspi-config` में GPU मेमोरी कम करें
4. अनावश्यक एप्लिकेशन बंद करें
5. यदि Pi 3 या पुराना उपयोग कर रहे हैं तो Pi 4 में अधिक RAM वाला अपग्रेड करें

### Wio Terminal

#### समस्या: Wio Terminal स्क्रीन खाली रहती है
**लक्षण:** कोड अपलोड के बाद कोई डिस्प्ले आउटपुट नहीं

**समाधान:**
1. जांचें कि कोड डिस्प्ले (TFT_eSPI लाइब्रेरी) इनिशियलाइज़ करता है
2. Wio Terminal फर्मवेयर अपडेट करें [Seeed Wiki](https://wiki.seeedstudio.com/Wio-Terminal-Getting-Started/)
3. डिस्प्ले इनिशियलाइजेशन कोड जोड़ें:
   ```cpp
   #include <TFT_eSPI.h>
   TFT_eSPI tft;
   tft.begin();
   tft.fillScreen(TFT_BLACK);
   ```
4. हार्डवेयर टेस्ट करने के लिए PlatformIO से उदाहरण स्केच अपलोड करें

#### समस्या: Wio Terminal पर WiFi काम नहीं कर रहा
**लक्षण:** WiFi से कनेक्ट नहीं हो पा रहा, नेटवर्क त्रुटियाँ

**समाधान:**
1. **WiFi फर्मवेयर अपडेट करें:** [Wio Terminal WiFi firmware update guide](https://wiki.seeedstudio.com/Wio-Terminal-Network-Overview/) देखें
2. **WiFi क्रेडेंशियल्स जांचें:** सुनिश्चित करें SSID और पासवर्ड सही हैं
3. **WiFi बैंड:** Wio Terminal केवल 2.4GHz WiFi को सपोर्ट करता है (5GHz नहीं)
4. **सिग्नल स्ट्रेंथ:** राउटर के करीब जाएँ
5. **राउटर सेटिंग्स:** कुछ एंटरप्राइज़/WPA-Enterprise नेटवर्क काम नहीं करते

#### समस्या: Wio Terminal कंप्यूटर द्वारा पहचाना नहीं जाता
**लक्षण:** USB डिवाइस नहीं दिख रहा

**समाधान:**
1. **अलग USB केबल आज़माएँ:** डेटा केबल उपयोग करें, चार्ज-ओनली नहीं
2. **बूटलोडर मोड में जाएं:** पावर स्विच नीचे दो बार जल्दी स्लाइड करें
   - नीला LED पल्स करेगा, डिवाइस डिवाइस मैनेजर में "Arduino" के रूप में दिखेगा
3. **ड्राइवर इंस्टॉल करें (विंडोज):**
   - डाउनलोड और इंस्टॉल करें [Seeed USB driver](https://wiki.seeedstudio.com/Driver_for_Seeeduino/)
4. **अलग USB पोर्ट आज़माएँ:** USB हब से बचें, डायरेक्ट कनेक्शन करें
5. सिस्टम USB ड्राइवर अपडेट करें

#### समस्या: Wio Terminal पर सेंसर काम नहीं कर रहे
**लक्षण:** Grove सेंसर डेटा नहीं पढ़ रहे

**समाधान:**
1. Grove केबल कनेक्शन्स जांचें
2. सही Grove पोर्ट (बाएं या दाएं) उपयोग कर रहे हैं यह सत्यापित करें
3. सेंसर के लिए सही लाइब्रेरी शामिल करें
4. सेंसर की पावर आवश्यकताएं जांचें
5. लाइब्रेरी के उदाहरण कोड से सेंसर टेस्ट करें

### वर्चुअल डिवाइस (CounterFit)

#### समस्या: CounterFit ऐप शुरू नहीं हो रहा
**त्रुटि:** CounterFit शुरू करते समय विभिन्न Python त्रुटियाँ

**समाधान:**
1. सुनिश्चित करें कि वर्चुअल एन्वायरनमेंट सक्रिय है
2. CounterFit इंस्टॉल/पुनः इंस्टॉल करें:
   ```bash
   pip install CounterFit
   ```
3. जांचें कि पोर्ट 5000 पहले से उपयोग में नहीं है:
   - विंडोज: `netstat -ano | findstr :5000`
   - macOS/Linux: `lsof -i :5000`
4. पोर्ट 5000 का उपयोग कर रहे प्रोसेस को बंद करें या अलग पोर्ट उपयोग करें:
   ```bash
   counterfit --port 5001
   ```

#### समस्या: कोड से CounterFit से कनेक्ट नहीं हो पाता
**त्रुटि:** कनेक्शन अस्वीकार या टाइमआउट

**समाधान:**
1. सुनिश्चित करें CounterFit चल रहा है: ब्राउजर में खोलें `http://127.0.0.1:5000`
2. कोड में कनेक्शन URL CounterFit एड्रेस से मेल खाता है
3. सुनिश्चित करें फ़ायरवॉल कनेक्शन ब्लॉक नहीं कर रहा
4. CounterFit ऐप और आपका कोड दोनों को पुनः शुरू करें

#### समस्या: CounterFit में सेंसर नहीं दिख रहे
**लक्षण:** बनाए गए सेंसर CounterFit UI में नहीं दिख रहे

**समाधान:**
1. कोड चलाने से पहले CounterFit UI में सेंसर बनाएं
2. ब्राउज़र पेज रिफ्रेश करें
3. सेंसर प्रकार कोड की अपेक्षा के अनुसार है यह जांचें
4. ब्राउज़र कैश साफ़ करें

## कनेक्टिविटी समस्याएँ

### वाईफ़ाई कनेक्शन

#### समस्या: डिवाइस वाईफ़ाई से कनेक्ट नहीं हो रहा
**लक्षण:** कनेक्शन टाइमआउट, प्रमाणीकरण विफल

**समाधान:**
1. **SSID और पासवर्ड जांचें:** क्रेडेंशियल सही हैं यह पुष्टि करें
2. **वाईफ़ाई बैंड:** अधिकांश IoT डिवाइसेज केवल 2.4GHz (5GHz नहीं) का समर्थन करते हैं
3. **राउटर सेटिंग्स:**
   - AP आइसोलेशन अक्षम करें यदि सक्षम हो
   - WPA2-PSK सुरक्षा का प्रयोग करें (WPA3, WEP या खुले नेटवर्क से बचें)
   - DHCP सक्षम करें
4. **छुपे हुए नेटवर्क:** यदि SSID छुपा है, तो इसे स्पष्ट रूप से कॉन्फ़िगर करना पड़ सकता है
5. **सिग्नल स्ट्रेंथ:** डिवाइस को राउटर के करीब ले जाएँ
6. **बाधा:** अन्य डिवाइसेज, माइक्रोवेव, या दीवारें हस्तक्षेप कर सकती हैं

#### समस्या: वाईफ़ाई कनेक्शन बार-बार टूट रहा है
**लक्षण:** कनेक्टिविटी बीच-बीच में होती है

**समाधान:**
1. राउटर स्थिरता जांचें और पुनः चालू करें
2. डिवाइस फर्मवेयर अपडेट करें
3. DHCP की जगह स्थिर IP उपयोग करें
4. राउटर से दूरी कम करें या वाईफ़ाई एक्सटेंटर जोड़ें
5. अन्य डिवाइसेज से हस्तक्षेप जांचें
6. पावर सप्लाई पर्याप्त है यह सुनिश्चित करें (खासकर Raspberry Pi के लिए)

### क्लाउड सेवाएँ

#### समस्या: Azure IoT Hub से कनेक्ट नहीं हो पा रहा
**त्रुटि:** प्रमाणिकरण विफल, कनेक्शन अस्वीकार

**समाधान:**
1. **क्रेडेंशियल सत्यापित करें:**
   - कनेक्शन स्ट्रिंग सही है यह जांचें
   - कनेक्शन स्ट्रिंग में अतिरिक्त स्पेस या लाइन ब्रेक न हों
2. **डिवाइस पंजीकरण जांचें:** डिवाइस IoT Hub में पंजीकृत होना चाहिए
3. **फ़ायरवॉल/प्रॉक्सी:** आउटबाउंड MQTT (पोर्ट 8883) या HTTPS (पोर्ट 443) अनुमति दें
4. **IoT Hub क्षेत्र:** सुनिश्चित करें कि IoT Hub चल रहा है और अलग क्षेत्र में नहीं है जिससे विलंबता हो
5. **कोटा सीमाएँ:** जांचें कि फ्री टियर सीमाएँ पूरी नहीं हुई हैं
6. **कनेक्शन टेस्ट करें:**
   ```bash
   az iot hub device-identity show-connection-string --hub-name YourIoTHub --device-id YourDevice
   ```

#### समस्या: Azure Functions ट्रिगर नहीं हो रहे
**लक्षण:** संदेश भेजे गए लेकिन फ़ंक्शन निष्पादित नहीं हो रहा

**समाधान:**
1. सुनिश्चित करें कि Function App चल रहा है (रुकावा नहीं है)
2. Function App सेटिंग्स में कनेक्शन स्ट्रिंग जांचें
3. Azure पोर्टल में फ़ंक्शन लॉग जांचें
4. सुनिश्चित करें Event Hub कम्पैटिबल एंडपॉइंट सही कॉन्फ़िगर किया गया है
5. संदेश स्वरूप फ़ंक्शन की अपेक्षा के अनुरूप है यह देखें
6. Function App सेवा योजना जांचें (कंजंप्शन बनाम डेडिकेटेड)

### MQTT
#### समस्या: MQTT कनेक्शन असफल
**त्रुटि:** कनेक्शन अस्वीकार किया गया, प्रमाणीकरण विफल

**समाधान:**
1. **ब्रोकर पता:** ब्रोकर URL/IP सही है यह सुनिश्चित करें
2. **पोर्ट:** पोर्ट नंबर जांचें (अअनएन्क्रिप्टेड के लिए 1883, TLS के लिए 8883)
3. **प्रमाणीकरण:** यदि आवश्यक हो तो उपयोगकर्ता नाम/पासवर्ड सत्यापित करें
4. **TLS/SSL:** सुनिश्चित करें प्रमाणपत्र मान्य और भरोसेमंद हैं
5. **फ़ायरवॉल:** जांचें कि पोर्ट ब्लॉक नहीं है
6. **MQTT क्लाइंट से परीक्षण करें:** परीक्षण के लिए MQTT Explorer या mosquitto_pub/sub का उपयोग करें

#### समस्या: MQTT संदेश प्राप्त नहीं हो रहे
**लक्षण:** संदेश प्रकाशित हो रहे हैं लेकिन सब्सक्राइबर द्वारा प्राप्त नहीं हो रहे

**समाधान:**
1. **विषय नाम:** सत्यापित करें कि सब्सक्राइबर का विषय बिल्कुल प्रकाशक के विषय से मेल खाता है
2. **QoS स्तर:** 0 के बजाय QoS 1 या 2 का प्रयास करें
3. **वाइल्डकार्ड:** जांचें कि विषय वाइल्डकार्ड सही ढंग से इस्तेमाल हो रहे हैं (`+` एक स्तर के लिए, `#` बहु-स्तर के लिए)
4. **रिटेन किए गए संदेश:** प्रकाशक अंतिम संदेश को रखने के लिए रिटेन फ़्लैग सेट कर सकता है
5. **कनेक्शन समय:** सुनिश्चित करें सब्सक्राइबर संदेश प्रकाशित होने से पहले कनेक्ट हो

## सेंसर और अधियंत्रक समस्याएँ

### ग्रोव सेंसर

#### समस्या: सेंसर गलत मान लौटाता है
**लक्षण:** पढ़ाई 0, -1, या अव्यवहारिक मान हैं

**समाधान:**
1. **कनेक्शन जांचें:** सुनिश्चित करें कि सेंसर ठीक से जुड़ा है
2. **सही पोर्ट:** जांचें कि सेंसर सही पोर्ट प्रकार में है:
   - एनालॉग सेंसर → एनालॉग पोर्ट (A0, A2, A4)
   - डिजिटल सेंसर → डिजिटल पोर्ट (D5, D16, D18, आदि)
   - I2C सेंसर → I2C पोर्ट
3. **कैलिब्रेशन:** कुछ सेंसर को कैलिब्रेशन की जरूरत होती है (मृदा नमी, प्रकाश)
4. **पावर साइकल:** सेंसर को डिस्कनेक्ट कर फिर से कनेक्ट करें
5. **सेंसर डेटाशीट:** सेंसर स्पेसिफिकेशन और आवश्यकताओं की जांच करें

#### समस्या: धार्रीय मृदा नमी सेंसर हमेशा गीला दिखाता है
**लक्षण:** सेंसर सूखा होने पर भी उच्च नमी पढ़ता है

**समाधान:**
1. **कैलिब्रेशन आवश्यक:** मिट्टी सेंसर को कैलिब्रेशन की जरूरत होती है:
   - हवा में मान पढ़ें (सूखा बेसलाइन)
   - पानी में मान पढ़ें (गीला बेसलाइन)
   - इन मानों के बीच मान मैप करें
2. **सेंसर कोटिंग जांचें:** नमी सेंसर की कोटिंग खराब हो सकती है
3. **स्थान:** सुनिश्चित करें कि सेंसर पूरी तरह मिट्टी में डाला गया है

#### समस्या: तापमान/आर्द्रता सेंसर की गलत पढ़ाई
**लक्षण:** DHT11/DHT22 गलत तापमान या आर्द्रता दिखाता है

**समाधान:**
1. **सेंसर स्थान:** सीधे धूप, गर्मी के स्रोत, या हवा के प्रवाह से बचें
2. **वार्म-अप समय:** पावर ऑन के बाद सेंसर को पढ़ने से पहले 2 सेकंड दें
3. **पढ़ने की आवृत्ति:** DHT सेंसर को पढ़ाई के बीच कम से कम 2 सेकंड चाहिए
4. **संघनन जांचें:** संघनन पढ़ाई को प्रभावित कर सकता है
5. **सेंसर गुणवत्ता:** DHT11 DHT22 से कम सटीक है

### कैमरा

#### समस्या: रास्पबेरी पाई पर कैमरा डिटेक्ट नहीं हो रहा
**त्रुटि:** `mmal: mmal_vc_component_create: failed to create component 'vc.ril.camera'`

**समाधान:**
1. **कैमरा इंटरफ़ेस सक्षम करें:**
   ```bash
   sudo raspi-config
   ```
  
   Interface Options → Camera → Enable पर जाएँ
2. **रिबन केबल जांचें:** सुनिश्चित करें कि कैमरा केबल ठीक से लगा है
   - पाई ज़ीरो पर नीला भाग USB पोर्ट की ओर होना चाहिए
   - पाई 4 पर नीला भाग USB पोर्ट से दूर होना चाहिए
3. **फर्मवेयर अपडेट करें:**
   ```bash
   sudo apt update
   sudo apt full-upgrade
   sudo reboot
   ```
   
4. **कैमरा का परीक्षण करें:**
   ```bash
   raspistill -o test.jpg
   ```


#### समस्या: कैमरा चित्र खराब गुणवत्ता के हैं
**लक्षण:** धुंधले, अंधेरे, या फीके चित्र

**समाधान:**
1. **फोकस:** लेंस से सुरक्षात्मक फिल्म हटाएं, यदि संभव हो तो फोकस समायोजित करें
2. **प्रकाश:** पर्याप्त प्रकाश सुनिश्चित करें
3. **कैमरा सेटिंग्स:** कोड में एक्सपोज़र, ISO, व्हाइट बैलेंस समायोजित करें
4. **स्थिरता:** कैमरा स्थिर रखें, आवश्यकता हो तो ट्राइपॉड का उपयोग करें
5. **रिज़ॉल्यूशन:** कैमरे की अधिकतम रिज़ॉल्यूशन से अधिक न करें

### माइक्रोफोन और स्पीकर

#### समस्या: ऑडियो इनपुट/आउटपुट नहीं हो रहा
**लक्षण:** माइक्रोफोन रिकॉर्ड नहीं कर रहा, स्पीकर आवाज़ नहीं दे रहा

**समाधान:**
1. **कनेक्शन जांचें:** ऑडियो डिवाइस ठीक से जुड़े हैं यह सुनिश्चित करें
2. **हार्डवेयर परीक्षण:**
   - स्पीकर: `speaker-test -t wav -c 2`
   - माइक्रोफोन: `arecord -l` सूची के लिए, `arecord test.wav` रिकॉर्ड करने के लिए
3. **वॉल्यूम सेटिंग्स:** जांचें और समायोजित करें:
   ```bash
   alsamixer
   ```
   
4. **ऑडियो डिवाइस चुनें:** कोड में सही ऑडियो डिवाइस निर्दिष्ट करें
5. **ड्राइवर समस्या:** ALSA अपडेट करें या ऑडियो ड्राइवर पुनः इंस्टॉल करें

#### समस्या: ReSpeaker हैट काम नहीं कर रहा
**लक्षण:** ऑडियो डिवाइस डिटेक्ट नहीं हो रहा

**समाधान:**
1. **ड्राइवर इंस्टॉल करें:**
   ```bash
   git clone https://github.com/HinTak/seeed-voicecard
   cd seeed-voicecard
   sudo ./install.sh
   sudo reboot
   ```
  
2. **इंस्टॉलेशन जांचें:** `arecord -l` में ReSpeaker दिखना चाहिए
3. **फर्मवेयर अपडेट करें:** कुछ पाई OS संस्करणों को ड्राइवर अपडेट की जरूरत होती है
4. **सेटिंग जांचें:** हैट GPIO पिन से ठीक से जुड़ा है यह सुनिश्चित करें

## विकास वातावरण समस्याएँ

### VS कोड

#### समस्या: टर्मिनल वर्चुअल एनवायरनमेंट को स्वतः सक्रिय नहीं कर रहा
**लक्षण:** टर्मिनल खुलता है लेकिन venv सक्रिय नहीं होता

**समाधान:**
1. **पायथन व्याख्याता सेट करें:** कमांड पैलेट → "Python: Select Interpreter" → venv चुनें
2. चयन के बाद VS कोड पुनः प्रारंभ करें
3. **सेटिंग्स जांचें:** `settings.json` में जोड़ें:
   ```json
   "python.terminal.activateEnvironment": true
   ```


#### समस्या: डिवाइस पर कोड नहीं चल रहा
**लक्षण:** कोड चलता है लेकिन डिवाइस पर कोई प्रतिक्रिया नहीं होती

**समाधान:**
1. **सुनिश्चित करें कोड सेव है** (फाइल टैब पर डॉट देखें)
2. **चेक करें कौन सा पायथन चल रहा है:** `which python` या `where python`
3. **Wio Terminal के लिए:** कोड PlatformIO से अपलोड हुआ हो
4. **Raspberry Pi के लिए:** SSH से पाई में लॉगिन कर कोड चलाएं
5. आउटपुट विंडो में त्रुटियां देखें

#### समस्या: IntelliSense पुस्तकालय फ़ंक्शन नहीं दिखा रहा
**लक्षण:** आयातित मॉड्यूल के लिए ऑटोकंप्लीट नहीं

**समाधान:**
1. सुनिश्चित करें लाइब्रेरी वर्तमान वातावरण में इंस्टॉल है
2. VS कोड विंडो रीलोड करें
3. पायथन इंटरप्रेटर सही है जांचें
4. उपलब्ध होने पर टाइप स्टब इंस्टॉल करें: `pip install types-<library-name>`

### पायथन वर्चुअल एनवायरनमेंट

#### समस्या: वर्चुअल एनवायरनमेंट बना नहीं पा रहे
**त्रुटि:** `The virtual environment was not created successfully`

**समाधान:**
1. **venv मॉड्यूल इंस्टॉल करें:**
   - Ubuntu/Debian: `sudo apt install python3-venv`
   - macOS: Python के साथ शामिल होता है
   - Windows: सभी कंपोनेंट्स के साथ Python फिर से इंस्टॉल करें
2. **पायथन इंस्टॉलेशन जांचें:** पायथन ठीक इंस्टॉल है यह सुनिश्चित करें
3. **पथ स्पष्ट करें:** `python3 -m venv .venv` की तरह स्पष्ट कॉल का उपयोग करें

#### समस्या: पैकेज गलत स्थान पर इंस्टॉल हो रहे हैं
**लक्षण:** पैकेज इंस्टॉल के बाद इम्पोर्ट त्रुटि

**समाधान:**
1. **venv सक्रिय है जांचें:** कमांड प्रॉम्प्ट पर `(.venv)` दिखना चाहिए
2. **pip स्थान जांचें:** `which pip` `.venv/bin/pip` की ओर इशारा करे
3. venv सक्रिय कर पुनः इंस्टॉल करें: `pip install <package>`
4. वर्चुअल एनवायरनमेंट में pip के साथ sudo का उपयोग न करें

#### समस्या: वर्चुअल एनवायरनमेंट पोर्टेबल नहीं है
**लक्षण:** वर्चुअल एनवायरनमेंट स्थानांतरण के बाद या अन्य कंप्यूटर पर काम नहीं करता

**समाधान:**
1. **venv को न स्थानांतरित करें:** हटाएं और नए स्थान पर फिर बनाएं
2. **requirements.txt उपयोग करें:**
   ```bash
   pip freeze > requirements.txt
   pip install -r requirements.txt
   ```
  
3. **venv पुनः बनाएँ:**
   ```bash
   python3 -m venv .venv
   source .venv/bin/activate  # या Windows पर activate.bat
   pip install -r requirements.txt
   ```


### निर्भरता

#### समस्या: पैकेज इंस्टॉलेशन विफल
**त्रुटि:** इंस्टॉलेशन के दौरान विभिन्न pip त्रुटियाँ

**समाधान:**
1. **pip अपडेट करें:**
   ```bash
   pip install --upgrade pip
   ```
  
2. **बिल्ड टूल्स इंस्टॉल करें:**
   - Ubuntu/Debian: `sudo apt install build-essential python3-dev`
   - macOS: `xcode-select --install`
   - Windows: Visual Studio Build Tools इंस्टॉल करें
3. इंटरनेट कनेक्शन जांचें
4. अलग पैकेज इंडेक्स आज़माएं: `pip install --index-url https://pypi.org/simple/ <package>`
5. विशिष्ट संस्करण इंस्टॉल करें: `pip install <package>==<version>`

#### समस्या: निर्भरता संघर्ष
**त्रुटि:** `ERROR: pip's dependency resolver does not currently take into account all the packages that are installed`

**समाधान:**
1. प्रत्येक प्रोजेक्ट के लिए ताजा वर्चुअल एनवायरनमेंट का उपयोग करें
2. पैकेज अपडेट करें: `pip install --upgrade <package>`
3. संघर्ष जांचें: `pip check` चलाएं
4. संगत संस्करण इंस्टॉल करें: requirements.txt में संस्करण सीमा निर्दिष्ट करें

## प्रदर्शन समस्याएँ

### समस्या: कोड धीरे चलता है
**लक्षण:** विलंब, टाइमआउट, अप्रतिसाद व्यवहार

**समाधान:**
1. सेंसर पढ़ने की आवृत्ति कम करें: बहुत बार न पढ़ें
2. लूप ऑप्टिमाइज़ करें: व्यस्त प्रतीक्षा से बचें, sleep() या डिले का उपयोग करें
3. मेमोरी मुद्दे:
   - अनावश्यक ऐप बंद करें
   - संग्रहण स्थान खाली करें
   - Pi पर `top` या `htop` से मॉनिटर करें
4. SD कार्ड गति: रास्पबेरी पाई के लिए तेज़ SD कार्ड या SSD का उपयोग करें
5. नेटवर्क विलंब: नेटवर्क कॉल के लिए असिंक्रोनस ऑपरेशन प्रयोग करें

### समस्या: मेमोरी खत्म होना
**त्रुटि:** `MemoryError` या सिस्टम फ्रीज़िंग

**समाधान:**
1. **रास्पबेरी पाई के लिए:**
   - अनावश्यक ऐप्स बंद करें
   - स्वैप स्पेस बढ़ाएं
   - हल्का OS (लाइट संस्करण) प्रयोग करें
   - RAM अपग्रेड करें (पाई 4 में 2/4/8GB विकल्प हैं)
2. **Wio Terminal के लिए:**
   - बफर साइज़ कम करें
   - छोटे चित्रों का उपयोग करें
   - स्ट्रिंग उपयोग अनुकूलित करें
   - मेमोरी लीक जांचें (अविवाहित मेमोरी)

### समस्या: डेटा हानि या भ्रष्टाचार
**लक्षण:** संदेश गायब, फाइलें खराब हो गईं

**समाधान:**
1. **SD कार्ड मुद्दे:**
   - गुणवत्ता वाले SD कार्ड का उपयोग करें (सस्ते/नकली से बचें)
   - नियमित बैकअप करें
   - स्वच्छ शटडाउन करें (पावर ना खींचें)
2. **बफर ओवरफ़्लो:** कोड में बफर साइज़ बढ़ाएं
3. **नेटवर्क विश्वसनीयता:** पुनः प्रयास तर्क और त्रुटि हैंडलिंग लागू करें
4. **सेवा की गुणवत्ता:** महत्वपूर्ण संदेशों के लिए MQTT QoS 1 या 2 का उपयोग करें

## सामान्य त्रुटि संदेश

### `ModuleNotFoundError: No module named 'X'`
**कारण:** पैकेज इंस्टॉल नहीं है या वर्चुअल एनवायरनमेंट सक्रिय नहीं है

**समाधान:**
```bash
pip install X
```
  
पहले वर्चुअल एनवायरनमेंट सक्रिय करें।

### Linux/macOS पर `Permission denied`
**कारण:** उन्नत अनुमति या फाइल अनुमति समस्या

**समाधान:**
- सिस्टम ऑपरेशन के लिए: `sudo` का उपयोग करें
- pip के लिए: वर्चुअल एनवायरनमेंट सक्रिय कर sudo का उपयोग न करें
- सीरियल पोर्ट के लिए: उपयोगकर्ता को dialout समूह में जोड़ें: `sudo usermod -a -G dialout $USER`, फिर लॉग आउट/लॉग इन करें

### `OSError: [Errno 98] Address already in use`
**कारण:** पोर्ट पहले से किसी अन्य प्रक्रिया द्वारा उपयोग में है

**समाधान:**
1. पोर्ट उपयोग कर रही प्रक्रिया खोजें: `lsof -i :<port>` या `netstat -ano | findstr :<port>`
2. प्रक्रिया को समाप्त करें या अपने कोड में अलग पोर्ट उपयोग करें

### `SSL: CERTIFICATE_VERIFY_FAILED`
**कारण:** SSL प्रमाणपत्र सत्यापन विफल

**समाधान:**
1. प्रमाणपत्र अपडेट करें: `pip install --upgrade certifi`
2. सिस्टम समय सही जांचें: `date`
3. केवल विकास के लिए (प्रोडक्शन नहीं): कोड में सत्यापन अक्षम करें

### `IndentationError: unexpected indent`
**कारण:** पायथन इंडेंटेशन समस्या (टैब/स्पेस मिलाना)

**समाधान:**
1. स्थायी इंडेंटेशन का उपयोग करें (4 स्पेस पायथन मानक है)
2. संपादक को टैब की जगह स्पेस उपयोग करने के लिए सेट करें
3. VS कोड में: `"editor.insertSpaces": true` और `"editor.tabSize": 4` सेट करें

### `UnicodeDecodeError` या `UnicodeEncodeError`
**कारण:** वर्णांकन त्रुटि

**समाधान:**
```python
# फ़ाइलें पढ़ते समय
with open('file.txt', 'r', encoding='utf-8') as f:
    content = f.read()

# फ़ाइलें लिखते समय
with open('file.txt', 'w', encoding='utf-8') as f:
    f.write(content)
```
