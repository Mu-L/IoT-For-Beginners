# IoT အဖော် - သင်ကြားမှုအစီအစဉ်

Microsoft ရဲ့ Azure Cloud Advocates အဖွဲ့သည် IoT အခြေခံများအကြောင်း ဆွေးနွေးသင်ကြားပေးသည့် ၁၂ ပတ်၊ ၂၄ သင်ခန်းစာအစီအစဉ်တစ်ခုကို ပျော်ရွှင်စွာတင်ဆက်ပေးပါသည်။ သင်ခန်းစာတိုင်းတွင် သင်ခန်းစာမတိုင်မီနှင့် သင်ခန်းစာပြီးနောက် စစ်ဆေးမေးခွန်းများ၊ သင်ခန်းစာကို ပြီးမြောက်စေရန် ရေးသားချက်များ၊ ဖြေရှင်းချက်တစ်ခု၊ အလုပ်တာဝန်တစ်ခုပြုလုပ်ခြင်းနှင့် အခြား အရာများပါဝင်သည်။ ကျွန်ုပ်တို့၏ ပရောဂျက်တီစဉ်အခြေပြု သင်ကြားမှုနည်းစနစ်သည် သင်ယူသူများ အတည်ပြုနိုင်စေရန် စီမံထားသည့် နည်းဖြစ်သည်။

ဒီပရောဂျက်များတွင် စိုက်ပျိုးရေးမှ စတင်ပြီး အစားအစာများ ကိုးကားမှုလမ်းကြောင်းကို ဖော်ပြထားသည်။ ဒီမှာ စိုက်ပျိုးရေး, ပို့ဆောင်ရေး, ထုတ်လုပ်မှု, လက်လီရောင်းအား၊ စားသုံးသူ အစိတ်အပိုင်းများ ပါဝင်သည် - IoT စက်ပစ္စည်းများအတွက် လူကြိုက်များသော လုပ်ငန်း ကဏ္ဍများ ဖြစ်ကြသည်။

## သင်ကြားမှုနည်းပညာ

ဒီ သင်ကြားမှုအစီအစဉ်ကို တည်ဆောက်ရာတွင် ပရောဂျက်အခြေပြုမှုနှင့် အကြိမ်ကြိမ် စစ်ဆေးမေးခွန်းထည့်သွင်းမှုတို့ကို ရွေးချယ်ထားသည်။ အစီအစဉ်ကုန်ဆုံးချိန်တွင် ကျောင်းသားများသည် အပင်များ စစ်ဆေးခြင်းနှင့် ရေကျွန်းပေးခြင်းစနစ်၊ ကားတစ်စီးကိန်းထိန်းခြင်းစနစ်၊ စားသောက်ကုန်စာရင်းစောင့်ကြည့်တဲ့ စကျန်းမာရေးစက်ရုံ setup, အသံထိန်းချုပ်နိုင်သော မီးဖိုချောင် အချိန်ကိရိယာတို့ကို တည်ဆောက်ပြီး IoT အခြေခံများ၊ စက်ပစ္စည်းကုဒ်ရေးသားခြင်း၊ Cloud နဲ့ချိတ်ဆက်ခြင်း၊ telemetry ကို ခွဲခြမ်းစိတ်ဖြာခြင်း နဲ့ အားနည်းယာဉ်ပေါ် AI အသုံးပြုနည်းတို့ကို သင်ယူကြပါလိမ့်မယ်။

သင်ခန်းစာများကို ပရောဂျက်အခြေပြုစေခြင်းအားဖြင့် ကျောင်းသားများအတွက် စိတ်ဝင်စားစေရန်နဲ့ အတွေးအမြင်များ ပိုမိုမြှင့်တင်နိုင်စေသည်။

ထို့အပြင် သင်တန်းမတိုင်မီ စစ်ဆေးမေးခွန်းတစ်ခုက ကျောင်းသားအတွက် ပညာသင်ယူချိန်ရောက်ရှိကြောင်း သတ်မှတ်သည့် ရည်ရွယ်ချက်ကို ရရှိစေပြီး သင်တန်းပြီးသည်နှင့် နောက်ထပ် စစ်ဆေးမေးခွန်းက ပိုမိုသိမြင်မှု ရရှိစေရန် အထောက်အကူပြုသည်။ ဒီအစီအစဉ်ကို သဘာဝကျပြီး ပျော်ရွှင်စရာအနေဖြင့် ဒီအစီအစဉ်တစ်လျှောက်လုံး သို့မဟုတ် တစိတ်တပိုင်း ပေါင်းစပ်ပြီး အသုံးပြုနိုင်သည်။ ပရောဂျက်များသည် သေးငယ်စေပြီး ၁၂ ပတ်စဉ်အတွင်း အဆင့်မြင့်လာသည်။

ပရောဂျက်တိုင်းသည် ကျောင်းသားများနှင့် စိတ်ဝင်စားသူများအတွက် ရရှိနိုင်သော လက်တွေ့ ဟာ့ဒ်ဝဲပစ္စည်းများပတ်လည်မှာ တည်ရှိသည်။ ပရောဂျက်တိုင်းသည် အဆိုပါ ပရောဂျက်ကဏ္ဍကို လေ့လာပေးပြီး သက်ဆိုင်ရာ နောက်ခံความรู้ကို ပေးသည်။ အောင်မြင်သော developer ဖြစ်ရန် မိမိဖြေရှင်းရန် လုပ်ဆောင်မည့် ကဏ္ဍကို နားလည်မှုရှိခြင်းသည် ကူညီပေးသည်။ ဒီနောက်ခံနားလည်မှုအားဖြင့် ကျောင်းသားများသည် မိမိတို့ IoT ဖြေရှင်းချက်များနှင့် သင်ယူချက်များကို လက်တွေ့ကိစ္စများ သဘောတူ၊ အသုံးပြုသူအဆုံး မူအပေါ် တွေးခွင့်ရရှိပြီး သင်ယူနိုင်စေသည်။

## ဟာ့ဒ်ဝဲ
ကြိုက်နှစ်သက်မှု၊ ပရိုဂရမ်မင်းဘာသာစကားလေ့လာမှု သို့မဟုတ် သင်ကြားလိုချင်သည့်အရာများ၊ ရရှိနိုင်မှုအပေါ် မူတည်ပြီး စိစစ်ကြည့်ရမည့် IoT hardware နှစ်မျိုးရှိပါသည်။ Hardware မရှိသေးသူများ သို့မဟုတ် ဝယ်ယူမှုမပြုမီ ပို၍ လေ့လာလိုသူများအတွက် 'virtual hardware' ဗားရှင်းကိုလည်း ပံ့ပိုးပေးထားပါသည်။ ကျွန်ုပ်တို့၏ မိတ်ဆွေ Seeed Studio မှ ရရှိနိုင်သော အတူတူ kit များ ဝယ်ယူနိုင်ပြီး 'shopping list' အပါအဝင် ပိုမိုသိရှိလိုပါက [hardware page](./hardware.md) တွင် ဖတ်ရှုနိုင်ပါသည်။

- သင်ခန်းစာမတိုင်မီ အပူအအေး စစ်ဆေးမှု ဖြစ်နိုင်ခြေ

- သိကောင်းစရာ စစ်ဆေးခြင်းများ
- စိန်ခေါ်မှုတစ်ခု
- ထပ်တိုးစာဖတ်ခြင်း
- တာဝန်ပေးစာအုပ်
- [သင်ခန်းစာပြီးနောက် စစ်ဆေးမှု](https://ff-quizzes.netlify.app/en/)

> **စစ်ဆေးမှုများအကြောင်း မှတ်ချက်**: စစ်ဆေးမှုအားလုံးကို quiz-app ဖိုလ်ဒါတွင် စုစည်းထားပြီး၊ မေးခွန်းသုံးမေးခွန်း ရှိသည့် စစ်ဆေးမှု ၄၈ ခု ရှိပါသည်။ သင်ခန်းစာများတွင် ဖက်ယာထားပြီး လူကြီးမင်းတို့နှင့် တိုက်ရိုက်စစ်ဆေးနိုင်သည်။ quiz-app ကိုကိုယ်တိုင် ဒေသန္တရ ပြေးနိုင်သည့်အပြင် Azure တွင်လည်း တင်နိုင်ပါသည်။ quiz-app ဖိုလ်ဒါအတွင်း လမ်းညွှန်ချက်များနှင့် အညီ လုပ်ဆောင်ပါ။ စစ်ဆေးခြင်းများကို တဖြည်းဖြည်း ပြည်တွင်းဘာသာစကားများသို့ ပြောင်းလဲနေပါသည်။

## အွန်လိုင်းမရှိဘဲ အသုံးပြုနိုင်မှု

[Docsify](https://docsify.js.org/#/) ကိုအသုံးပြု၍ ဒီစာတမ်းများကို အွန်လိုင်းမရှိဘဲ အကောင်းဆုံးသုံးနိုင်ပါသည်။ repo ကို fork လုပ်ပြီး [Docsify](https://docsify.js.org/#/quickstart) ကို ကိုယ်ပိုင်စက်တွင် ထည့်သွင်းပြီး repo root ဖိုလ်ဒါတွင် `docsify serve` ဟုရိုက်ထည့်ပါ။ ရလဒ်အနေဖြင့် 3000 port တွင် ရှိသည့် localhost: `localhost:3000` တွင် ဝဘ်ဆိုဒ်ကို လုပ်ဆောင်ပေးပါမည်။
