## इन्स्टॉलेशन समस्या

### पायथन इन्स्टॉलेशन

#### समस्या: पायथन आवृत्ती खूप जुनी आहे
**त्रुटी:** `Python 3.6 किंवा त्याहून अधिक आवृत्ती आवश्यक आहे`

**उपाय:**
1. [python.org](https://www.python.org/downloads/) वरून नवीनतम Python 3 डाउनलोड करा
2. Windows वर इन्स्टॉलेशन दरम्यान, "Add Python to PATH" तपासा
3. इन्स्टॉलेशन पडताळा:
   ```bash
   python3 --version
   ```

#### समस्या: अनेक पायथन आवृत्त्यांमुळे संघर्ष
**लक्षणे:** चुकीची पायथन आवृत्ती चालते, पॅकेजेस चुकीच्या स्थानावर इन्स्टॉल होतात

**उपाय:**
- **Windows:** `python` ऐवजी स्पष्टपणे Python 3 वापरण्यासाठी `py -3` वापरा
- **macOS/Linux:** `python` ऐवजी `python3` वापरा
- प्रोजेक्टसाठी नेहमी व्हर्च्युअल एन्व्हायर्नमेंट तयार करा आणि वापरा

#### समस्या: pip आदेश सापडला नाही
**त्रुटी:** `'pip' हा अंतर्गत किंवा बाह्य आदेश म्हणून ओळखला जात नाही`

**उपाय:**
1. `pip` ऐवजी `pip3` वापरून पहा
2. किंवा `python -m pip` किंवा `python3 -m pip` वापरा
3. पायथन PATH मध्ये जोडलेले आहे याची खात्री करा (पायथन रीइन्स्टॉल करा आणि पर्याय तपासा)

### VS कोड आणि एक्सटेंशन्स

#### समस्या: Pylance एक्सटेंशन काम करत नाही
**लक्षणे:** Python IntelliSense, कोड पूर्णता किंवा टाइप तपासणी नाही

**उपाय:**
1. VS Code आदेश पॅलेट उघडा (`Ctrl+Shift+P` किंवा `Cmd+Shift+P`)
2. "Python: Select Interpreter" चालवा
3. योग्य Python इंटरप्रिटरसाठी निवडा (व्हर्च्युअल एन्व्हायर्नमेंट वापरत असल्यास)
4. VS Code विंडो रीलोड करा

#### समस्या: VS Code व्हर्च्युअल एन्व्हायर्नमेंट ओळखत नाही
**लक्षणे:** चुकीचा Python इंटरप्रिटर निवडला गेला आहे

**उपाय:**
1. टर्मिनलमध्ये व्हर्च्युअल एन्व्हायर्नमेंट सक्रिय केले असल्याची खात्री करा
2. आदेश पॅलेट उघडा आणि "Python: Select Interpreter" चालवा
3. `.venv` फोल्डरमधील इंटरप्रिटर निवडा
4. स्टेटस बार (खाली डावे) योग्य Python आवृत्ती दर्शवत आहे की नाही तपासा

### प्लॅटफॉर्मआयओ (Wio टर्मिनल)

#### समस्या: PlatformIO इन्स्टॉलेशन अयशस्वी
**त्रुटी:** PlatformIO इन्स्टॉलेशन दरम्यान विविध त्रुटी

**उपाय:**
1. VS Code अद्ययावत आहे याची खात्री करा
2. प्रथम C/C++ एक्सटेंशन इन्स्टॉल करा
3. PlatformIO इन्स्टॉल केल्यानंतर VS Code पुन्हा सुरू करा
4. तुमचा इंटरनेट कनेक्शन तपासा (PlatformIO मोठ्या फाइल्स डाउनलोड करते)

#### समस्या: PlatformIO कडून बोर्ड सापडत नाही
**लक्षणे:** Wio टर्मिनलवर कोड अपलोड करता येत नाही

**उपाय:**
1. वेगळा USB केबल वापरून पाहा (काही केबल फक्त चार्जसाठी असतात)
2. Device Manager (Windows) किंवा `ls /dev/tty*` (macOS/Linux) तपासा
3. USB ड्रायव्हर्स इन्स्टॉल किंवा अपडेट करा
4. वेगळ्या USB पोर्टचा प्रयत्न करा
5. Wio टर्मिनलवरील पॉवर स्विच दोनदा लवकर स्लाइड करून बूटलोडर मोडमध्ये जा

#### समस्या: PlatformIO मध्ये संकलन त्रुटी
**त्रुटी:** `fatal error: Arduino.h: No such file or directory`

**उपाय:**
1. प्रोजेक्टमधील `.pio` फोल्डर हटवा
2. आदेश पॅलेटमधून "PlatformIO: Rebuild" चालवा
3. `platformio.ini` मध्ये योग्य बोर्ड कॉन्फिगरेशन आहे की नाही याची खात्री करा:
   ```ini
   [env:seeed_wio_terminal]
   platform = atmelsam
   board = seeed_wio_terminal
   framework = arduino
   ```

### ग्रोव्ह लायब्ररीज

#### समस्या: रास्पबेरी पायवर Grove लायब्ररी इम्पोर्ट अयशस्वी
**त्रुटी:** `ModuleNotFoundError: No module named 'grove'`

**उपाय:**
1. Grove लायब्ररीज पुन्हा इन्स्टॉल करा:
   ```bash
   cd ~
   git clone https://github.com/Seeed-Studio/grove.py
   cd grove.py
   sudo pip3 install .
   ```
2. व्हर्च्युअल एन्व्हायर्नमेंट वापरत असल्यास, ग्लोबली इन्स्टॉल करावे किंवा लायब्ररीज कॉपी कराव्या लागतील
3. I2C सक्रिय आहे का तपासा: `sudo raspi-config nonint do_i2c 0`

#### समस्या: Grove सेन्सर सापडत नाही
**त्रुटी:** `IOError: [Errno 121] Remote I/O error`

**उपाय:**
1. भौतिक कनेक्शन्स तपासा (Grove केबल नीट बसलेली आहे याची खात्री करा)
2. सेन्सर योग्य पोर्टवर (एनालॉग, डिजिटल, I2C, UART) जोडलेला आहे का तपासा
3. `i2cdetect -y 1` चालवून डिव्हाइस I2C बसवर दिसतो का ते तपासा
4. वेगळा Grove केबल वापरून पहा
5. Grove बेस हॅट रास्पबेरी पाय GPIO पिन्सवर नीट बसलेला आहे का याची खात्री करा

## हार्डवेअर समस्या

### रास्पबेरी पाय

#### समस्या: रास्पबेरी पाय बूट होत नाही
**लक्षणे:** डिस्प्ले नाही, LED क्रिया नाही किंवा रेनबो स्क्रीन

**उपाय:**
1. **पॉवर सप्लाय तपासा:** Pi 4 साठी अधिकृत 5V 3A USB-C पॉवर सप्लाय वापरा
2. **SD कार्ड समस्या:**
   - SD कार्ड फॉरमॅट करा आणि Raspberry Pi OS पुन्हा इन्स्टॉल करा
   - वेगळे SD कार्ड वापरून पहा (शिफारस केलेले ब्रँड वापरा)
   - SD कार्ड नीट घातलेले आहे का याची खात्री करा
3. **HDMI कनेक्शन तपासा:** Pi 4 वरील दोन्ही HDMI पोर्ट वापरून पहा, पॉवरजवळचा HDMI पोर्ट वापरा

#### समस्या: रास्पबेरी पायवर SSH करता येत नाही
**लक्षणे:** कनेक्शन नाकारले गेले किंवा टाइमआऊट

**उपाय:**
1. SSH सक्षम करा:
   - Raspberry Pi Imager वापरताना SD कार्ड फ्लॅश करताना SSH अ‍ॅडव्हान्स्ड पर्यायामध्ये सेटिंग करा
   - किंवा बूट पार्टीशनमध्ये `ssh` नावाची (विस्तारांश न ठेवता) रिकामी फाइल तयार करा
2. पाय IP पत्ता शोधा:
   - तुमच्या राऊटरच्या कनेक्टेड डिव्हाइसेस तपासा
   - `ping raspberrypi.local` वापरा (जर mDNS काम करत असेल तर)
   - नेटवर्क स्कॅनिंग टूल्स जसे `nmap` किंवा Angry IP Scanner वापरा
3. नेटवर्क तपासा:
   - पाय आणि संगणक एकाच नेटवर्कवर आहे याची खात्री करा
   - WiFi ऐवजी इथरनेट कनेक्शन वापरून पहा
4. वापरकर्ता नाव/पासवर्ड सत्यापित करा (डिफॉल्ट: वापरकर्ता `pi`, पासवर्ड `raspberry`)

#### समस्या: Grove बेस हॅट ओळखली जात नाही
**लक्षणे:** सेन्सर्स काम करत नाहीत, I2C त्रुटी

**उपाय:**
1. बेस हॅट सर्व GPIO पिन्सवर नीट बसलेला आहे याची खात्री करा
2. पाय किंवा बेस हॅटवरील वाकलेले पिन तपासा
3. I2C इंटरफेस सक्षम करा:
   ```bash
   sudo raspi-config nonint do_i2c 0
   sudo reboot
   ```
4. I2C काम करत आहे का ते तपासा: `i2cdetect -y 1`

#### समस्या: रास्पबेरी पाय हळूहळू चालतोय
**लक्षणे:** UI उशीराने प्रतिसाद देते, सिस्टम मंदावलेले आहे

**उपाय:**
1. SD कार्ड वेग तपासा (Class 10 किंवा त्याहून चांगले, USB द्वारे SSD वापरा)
2. डिस्क स्पेस मोकळी करा: `df -h` वापरून तपासा, अनावश्यक फाइल्स काढा
3. `raspi-config` मध्ये GPU मेमरी कमी करा जर कॅमेरा/डिस्प्ले जास्त वापरत नसेल तर
4. अनावश्यक ऍप्लिकेशन्स बंद करा
5. Pi 3 किंवा जुन्या मॉडेल वापरत असल्यास Pi 4  जास्त RAM सह विचार करा

### Wio टर्मिनल

#### समस्या: Wio टर्मिनल स्क्रीन रिकामी आहे
**लक्षणे:** कोड अपलोड केल्यानंतर डिस्प्ले आउटपुट नाही

**उपाय:**
1. कोड डिस्प्लेला सुरुवात करतोय का ते तपासा (TFT_eSPI लायब्ररी)
2. [Seeed Wiki](https://wiki.seeedstudio.com/Wio-Terminal-Getting-Started/) वरून Wio टर्मिनल फर्मवेअर अपडेट करा
3. डिस्प्ले प्रारंभ करण्याचा कोड जोडा:
   ```cpp
   #include <TFT_eSPI.h>
   TFT_eSPI tft;
   tft.begin();
   tft.fillScreen(TFT_BLACK);
   ```
4. प्लॅटफॉर्मआयओमधून उदाहरण स्केच अपलोड करून हार्डवेअर तपासा

#### समस्या: Wio टर्मिनलवर WiFi काम करत नाही
**लक्षणे:** WiFi कनेक्ट करता येत नाही, नेटवर्क त्रुटी

**उपाय:**
1. **WiFi फर्मवेअर अपडेट करा:** [Wio Terminal WiFi firmware update guide](https://wiki.seeedstudio.com/Wio-Terminal-Network-Overview/) अनुसरा
2. **WiFi प्रमाणपत्र तपासा:** SSID आणि पासवर्ड बरोबर आहेत याची खात्री करा
3. **WiFi बँड:** Wio टर्मिनल केवळ 2.4GHz WiFi समर्थित (5GHz नाही)
4. **सिग्नल क्षमता:** राऊटरजवळ जा
5. **राऊटर सेटिंग्स:** काही एंटरप्राइझ/WPA-Enterprise नेटवर्क काम करू शकत नाहीत

#### समस्या: Wio टर्मिनल संगणकाने ओळखला नाही
**लक्षणे:** USB डिव्हाइस सापडत नाही

**उपाय:**
1. **वेगळा USB केबल वापरा:** डेटा केबल वापरा, फक्त चार्जसाठी नसलेली केबल वापरा
2. **बूटलोडर मोडमध्ये जा:** पॉवर स्विच दोनदा लवकर स्लाइड करा
   - निळे LED पल्स करेल, डिव्हाइस Device Manager मध्ये "Arduino" म्हणून दिसेल
3. **ड्रायव्हर्स इंस्टॉल करा (Windows):**
   - [Seeed USB driver](https://wiki.seeedstudio.com/Driver_for_Seeeduino/) डाउनलोड आणि इन्स्टॉल करा
4. **वेगळ्या USB पोर्टचा प्रयत्न करा:** USB हब टाळा, थेट कनेक्शन वापरा
5. **सिस्टम USB ड्रायव्हर्स अपडेट करा**

#### समस्या: Wio टर्मिनलवर सेन्सर्स काम करत नाहीत
**लक्षणे:** Grove सेन्सर्स डेटा वाचत नाहीत

**उपाय:**
1. Grove केबल कनेक्शन तपासा
2. योग्य Grove पोर्ट (डावा किंवा उजवा) वापरत आहात का ते तपासा
3. सेन्सरसाठी योग्य लायब्ररी समाविष्ट करा
4. सेन्सरची पॉवर आवश्यकतांची खात्री करा
5. लायब्ररीमधील उदाहरण कोडने सेन्सर तपासून पहा

### व्हर्च्युअल डिव्हाइस (काउंटरफिट)

#### समस्या: काउंटरफिट अॅप सुरू होत नाही
**त्रुटी:** CounterFit सुरू करताना विविध पायथन त्रुटी

**उपाय:**
1. व्हर्च्युअल एन्व्हायर्नमेंट सक्रिय आहे याची खात्री करा
2. CounterFit इन्स्टॉल / रीइन्स्टॉल करा:
   ```bash
   pip install CounterFit
   ```
3. पोर्ट 5000 आधीच वापरात नाही याची तपासणी करा:
   - Windows: `netstat -ano | findstr :5000`
   - macOS/Linux: `lsof -i :5000`
4. पोर्ट 5000 वापरणारा प्रोसेस संपवा किंवा वेगळा पोर्ट वापरा:
   ```bash
   counterfit --port 5001
   ```

#### समस्या: कोडमधून काउंटरफिटशी कनेक्ट करता येत नाही
**त्रुटी:** कनेक्शन नाकारले गेले किंवा टाइमआऊट

**उपाय:**
1. CounterFit चालू आहे का ते तपासा: ब्राउझरमध्ये `http://127.0.0.1:5000` उघडा
2. कोडमधील कनेक्शन URL काउंटरफिट पत्त्यास अनुरूप आहे का ते तपासा
3. फायरवॉल कनेक्शन ब्लॉक करत नाही याची खात्री करा
4. CounterFit अॅप आणि तुमचा कोड दोन्ही रीस्टार्ट करा

#### समस्या: काउंटरफिटमध्ये सेन्सर्स दिसत नाहीत

## कनेक्टिव्हिटी समस्या

### वाईफाई कनेक्शन

#### समस्या: डिव्हाइस WiFi शी कनेक्ट होऊ शकत नाही
**लक्षणे:** कनेक्शन टाइमआऊट, प्रमाणीकरण अयशस्वी

**उपाय:**
1. **SSID आणि पासवर्ड तपासा:** प्रमाणीकरण माहिती बरोबर आहे का तपासा
2. **WiFi बँड:** बहुतेक IoT डिव्हाइसेस फक्त 2.4GHz पुरवतात (5GHz नाही)
3. **राऊटर सेटिंग्स:**
   - AP आयसोलेशन असेल तर डिसेबल करा
   - WPA2-PSK सुरक्षा वापरा (WPA3, WEP किंवा उघडे नेटवर्क टाळा)
   - DHCP सक्षम आहे याची खात्री करा
4. **लपलेली नेटवर्क्स:** SSID लपवलेले असल्यास, ते स्पष्टपणे कॉन्फिगर करावे लागेल
5. **सिग्नल क्षमता:** डिव्हाइस राऊटरजवळ ठेवा
6. **व्यत्यय:** इतर डिव्हाइसेस, मायक्रोवेव किंवा भिंतींमुळे व्यत्यय होऊ शकतो

#### समस्या: WiFi कनेक्शन वारंवार संपते
**लक्षणे:** अनियमित कनेक्टिव्हिटी

**उपाय:**
1. राऊटर स्थिरतेची तपासणी करा आणि रीबूट करा
2. डिव्हाइस फर्मवेअर अपडेट करा
3. DHCP ऐवजी स्थिर IP वापरा
4. राऊटरजवळ जा किंवा WiFi एक्स्टेंडर जोडा
5. इतर डिव्हाइसेसमधून व्यत्यय पाहा
6. पॉवर सप्लाय पुरेशा आहे का तपासा (विशेषतः रास्पबेरी पायसाठी)

### क्लाउड सेवा

#### समस्या: Azure IoT Hub शी कनेक्ट करता येत नाही
**त्रुटी:** प्रमाणीकरण अयशस्वी, कनेक्शन नाकारले

**उपाय:**
1. **प्रमाणीकरण तपासा:**
   - कनेक्शन स्ट्रिंग बरोबर आहे का तपासा
   - कनेक्शन स्ट्रिंगमध्ये अतिरिक्त जागा किंवा ओळीतील ब्रेक नाहीत याची खात्री करा
2. **डिव्हाइस नोंदणी तपासा:** डिव्हाइस IoT Hub मध्ये नोंदणीकृत आहे का
3. **फायरवॉल / प्रॉक्सी:** आउटबाउंड MQTT (पोर्ट 8883) किंवा HTTPS (पोर्ट 443) परवानगी आहे का तपासा
4. **IoT Hub क्षेत्र:** IoT Hub चालू आहे आणि वेगळ्या प्रदेशात नाही ज्यामुळे विलंब होतो ते तपासा
5. **कोटा मर्यादां:** फ्री टियर मर्यादा ओलांडल्या नाहीत का ते तपासा
6. **कनेक्शन तपासणी करा:**
   ```bash
   az iot hub device-identity show-connection-string --hub-name YourIoTHub --device-id YourDevice
   ```

#### समस्या: Azure Functions ट्रिगर होत नाहीत
**लक्षणे:** मेसेज पाठवले जात आहेत पण फंक्शन कार्यरत नाही

**उपाय:**
1. फंक्शन अॅप चालू आहे का तपासा (थांबवलेले नाही)
2. फंक्शन अॅप सेटिंग्जमधील कनेक्शन स्ट्रिंग तपासा
3. Azure पोर्टलमधील फंक्शन लॉग तपासा
4. इव्हेंट हब कंपॅटिबल एंडपॉइंट योग्यरित्या कॉन्फिगर केले आहे का तपासा
5. मेसेज फॉरमॅट फंक्शनच्या अपेक्षांशी जुळतो का पहा
6. फंक्शन अॅप सेवा योजना तपासा (कन्सम्प्शन विरुद्ध डेडिकेटेड)

### MQTT
#### समस्या: MQTT कनेक्शन अयशस्वी होते  
**त्रुटी:** कनेक्शन नाकारले, प्रमाणीकरण अयशस्वी

**उपाय:**  
1. **ब्रोक्हर पत्ता:** ब्रोक्हर URL/IP योग्य आहे का तपासा  
2. **पोर्ट:** पोर्ट क्रमांक तपासा (1883 एनक्रिप्टेड नसलेल्यासाठी, 8883 TLS साठी)  
3. **प्रमाणीकरण:** वापरकर्ता नाव/पासवर्ड आवश्यक असल्यास तपासा  
4. **TLS/SSL:** प्रमाणपत्रे वैध आणि विश्वासार्ह आहेत याची खात्री करा  
5. **फायर्वॉल:** पोर्ट ब्लॉक केलेले नाही याची तपासणी करा  
6. **MQTT क्लायंटसह चाचणी करा:** MQTT Explorer किंवा mosquitto_pub/sub वापरून चाचणी करा  

#### समस्या: MQTT संदेश प्राप्त होत नाहीत  
**लक्षणे:** संदेश प्रकाशित केले जातात पण सबस्क्राइबर्सकडे मिळत नाहीत

**उपाय:**  
1. **टॉपिक नावे:** सबस्क्रायबर टॉपिक प्रकाशित करणाऱ्या टॉपिकशी अगदी जुळते का तपासा  
2. **QoS पातळी:** 0 ऐवजी QoS 1 किंवा 2 वापरून पहा  
3. **वाइल्डकार्ड्स:** टॉपिक वाइल्डकार्ड बरोबर वापरले आहेत का तपासा (`+` सिंगल लेवलसाठी, `#` मल्टी-लेवलसाठी)  
4. **रिटेन केलेले संदेश:** प्रकाशित करणारा रिटेन फ्लॅग सक्षम करू शकतो ज्याने शेवटचा संदेश ठेवला जातो  
5. **कनेक्शन वेळ:** संदेश प्रकाशित होण्यापूर्वी सबस्क्रायबर कनेक्ट झाला आहे याची खात्री करा

## सेन्सर आणि अॅक्च्युएटर समस्या

### ग्रोव्ह सेन्सर

#### समस्या: सेन्सर चुकीचे मूल्ये परत करतो  
**लक्षणे:** रीडिंग्ज 0, -1 किंवा अयोग्य असतात

**उपाय:**  
1. **कनेक्शन्स तपासा:** सेन्सर योग्यरित्या जोडलेला आहे याची खात्री करा  
2. **योग्य पोर्ट:** सेन्सर योग्य पोर्ट प्रकारात आहे का तपासा:  
   - अॅनालॉग सेन्सर → अॅनालॉग पोर्ट्स (A0, A2, A4)  
   - डिजिटल सेन्सर → डिजिटल पोर्ट्स (D5, D16, D18 इ.)  
   - I2C सेन्सर → I2C पोर्ट्स  
3. **कॅलिब्रेशन:** काही सेन्सरला कॅलिब्रेशनची गरज असते (मातीची ओलसरता, प्रकाश)  
4. **पॉवर सायकल:** सेन्सर डिसकनेक्ट करून पुन्हा कनेक्ट करा  
5. **सेन्सर डेटाशीट:** सेन्सरची वैशिष्ट्ये आणि आवश्यकता तपासा  

#### समस्या: कॅपॅसिटिव्ह मातीची ओलसरता सेन्सर नेहमी ओलसर वाचतो  
**लक्षणे:** सेन्सर कोरडे असताना देखील उच्च ओलसरता दाखवतो

**उपाय:**  
1. **कॅलिब्रेशन आवश्यक:** माती सेन्सरला कॅलिब्रेशन करावा लागतो:  
   - हवा (कोरड्या बेसलाइन) मध्ये मूल्य वाचा  
   - पाण्यात (ओलसर बेसलाइन) मध्ये मूल्य वाचा  
   - या मूल्यांदरम्यान मॅपिंग करा  
2. **सेन्सर कोटिंग तपासा:** ओलसरता सेन्सरचा कोटिंग खराब झाल्यास खराब होऊ शकतो  
3. **ठिकाण:** सेन्सर पूर्णपणे मातीमध्ये बसलेला आहे याची खात्री करा  

#### समस्या: तापमान/आर्द्रता सेन्सरचे वाचन चुकीचे  
**लक्षणे:** DHT11/DHT22 चुकीचे तापमान किंवा आर्द्रता दाखवते

**उपाय:**  
1. **सेन्सर ठिकाण:** थेट सूर्यप्रकाश, उष्णता स्रोत किंवा हवाबाह्य प्रवाह टाळा  
2. **गरम होण्याचा वेळ:** सेन्सरवर पॉवर दिल्यानंतर 2 सेकंद प्रतीक्षा करा  
3. **वाचन वारंवारता:** DHT सेन्सरला वाचन दरम्यान किमान 2 सेकंद वेळ द्या  
4. **संघनन तपासा:** संघननामुळे परिणाम होऊ शकतो  
5. **सेन्सर गुणवत्ता:** DHT11 DHT22 पेक्षा कमी अचूक आहे

## विकास पर्यावरण समस्या

### VS Code

#### समस्या: टर्मिनल स्वयंचलितपणे वर्चुअल एन्व्हायर्नमेंट सक्रिय करत नाही

**उपाय:**

2. इंटरप्रिटर निवडल्यानंतर VS Code रीस्टार्ट करा  
3. `settings.json` मध्ये खालील जोडा:  
   ```json
   "python.terminal.activateEnvironment": true
   ```
  

#### समस्या: डिव्हाइसवर कोड चालत नाही  
**लक्षणे:** कोड चालतो पण डिव्हाइसवर परिणाम नाही

**उपाय:**  
1. कोड सेव्ह झाला आहे का तपासा (फाईल टॅबवर डॉट)  
2. कोणता Python चालत आहे ते तपासा: `which python` किंवा `where python`  
3. Wio Terminal साठी: PlatformIO वापरून कोड अपलोड करा (अपलोड बटन क्लिक करा)  
4. Raspberry Pi साठी: Pi मध्ये SSH करा आणि कोड तिथे चालवा  
5. आउटपुट विंडोमध्ये त्रुटी तपासा  

#### समस्या: IntelliSense लायब्ररी फंक्शन्स दाखवत नाही  
**लक्षणे:** आयात केलेल्या मॉड्युलसाठी ऑटो-कंप्लिट नाही

**उपाय:**  
1. लायब्ररी वर्तमान पर्यावरणात इन्स्टॉल आहे याची खात्री करा  
2. VS Code विंडो रीलोड करा  
3. Python इंटरप्रिटर बरोबर सेट आहे का तपासा  
4. उपलब्ध असल्यास टाइप स्टब इन्स्टॉल करा: `pip install types-<library-name>`  

### Python वर्चुअल एन्व्हायर्नमेंट

#### समस्या: वर्चुअल एन्व्हायर्नमेंट तयार करता येत नाही  
**त्रुटी:** `The virtual environment was not created successfully`

**उपाय:**  
1. **venv मॉड्यूल इन्स्टॉल करा:**  
   - Ubuntu/Debian: `sudo apt install python3-venv`  
   - macOS: Python सोबत असतो  
   - विंडोज: पूर्ण Python पुन्हा इन्स्टॉल करा  
2. Python इन्स्टॉलेशन योग्य आहे का तपासा  
3. पूर्ण पाथ वापरून पाहा: `python3 -m venv .venv`  

#### समस्या: पॅकेज चुकीच्या ठिकाणी इन्स्टॉल झाले  
**लक्षणे:** पॅकेज इन्स्टॉल केल्यानंतर आयात त्रुटी

**उपाय:**  
1. **venv सक्रिय आहे का तपासा:** कमांड प्रॉम्प्टवर `(.venv)` दिसायला हवे  
2. **pip स्थान तपासा:** `which pip` `.venv/bin/pip` कडे निर्देशित करतो का तपासा  
3. venv सक्रिय करून पॅकेज पुन्हा इन्स्टॉल करा: `pip install <package>`  
4. वर्चुअल एन्व्हायर्नमेंटमध्ये `sudo` वापरू नका  

#### समस्या: वर्चुअल एन्व्हायर्नमेंट पोर्टेबल नाही  
**लक्षणे:** venv हलविल्यानंतर किंवा वेगळ्या संगणकावर काम करत नाही

**उपाय:**  
1. venv हलवू नका; नवीन ठिकाणी हटवून पुन्हा तयार करा  
2. **requirements.txt वापरा:**  
   ```bash
   pip freeze > requirements.txt
   pip install -r requirements.txt
   ```
3. venv पुन्हा तयार करा:  
   ```bash
   python3 -m venv .venv
   source .venv/bin/activate  # किंवा Windows वर activate.bat वापरा
   pip install -r requirements.txt
   ```
  

### अवलंबित्व

#### समस्या: पॅकेज इन्स्टॉलेशन अयशस्वी होतो  
**त्रुटी:** इन्स्टॉलेशन दरम्यान विविध pip त्रुटी

**उपाय:**  
1. **pip अपडेट करा:**  
   ```bash
   pip install --upgrade pip
   ```
2. **बिल्ड टूल्स इन्स्टॉल करा:**  
   - Ubuntu/Debian: `sudo apt install build-essential python3-dev`  
   - macOS: `xcode-select --install`  
   - विंडोज: Visual Studio Build Tools इन्स्टॉल करा  
3. इंटरनेट कनेक्शन तपासा  
4. वेगळा पॅकेज इंडेक्स वापरून पहा: `pip install --index-url https://pypi.org/simple/ <package>`  
5. विशिष्ट आवृत्ती इन्स्टॉल करा: `pip install <package>==<version>`  

#### समस्या: अवलंबित्व संघर्ष  
**त्रुटी:** `ERROR: pip's dependency resolver does not currently take into account all the packages that are installed`

**उपाय:**  
1. **प्रत्येक प्रोजेक्टसाठी नवीन वर्चुअल एन्व्हायर्नमेंट वापरा**  
2. पॅकेजेस अपडेट करा: `pip install --upgrade <package>`  
3. `pip check` वापरून संघर्ष शोधा  
4. संगत आवृत्त्या requirements.txt मध्ये निर्दिष्ट करा

## कार्यक्षमता समस्या

### समस्या: कोड हळूहळू चालतो  
**लक्षणे:** उशीर, टाइमआउट, प्रतिसाद नसणे

**उपाय:**  
1. सेन्सर वाचन वारंवारता कमी करा  
2. लूप ऑप्टिमाइझ करा: बिझी-वेटिंग टाळा, sleep() किंवा विलंब वापरा  
3. मेमरी समस्या:  
   - अनावश्यक अॅप्लिकेशन्स बंद करा  
   - साठवणुकीची जागा रिकामी करा  
   - Pi वर `top` किंवा `htop` वापरून मॉनिटर करा  
4. SD कार्ड स्पीड: वेगवान SD कार्ड किंवा SSD वापरा  
5. नेटवर्क विलंब: नेटवर्क कॉलसाठी असिंक्रोनस ऑपरेशन्स वापरा  

### समस्या: मेमरी संपली त्रुटी  
**त्रुटी:** `MemoryError` किंवा सिस्टीम फ्रीझ होणे

**उपाय:**  
1. Raspberry Pi साठी:  
   - अनावश्यक अॅप्लिकेशन्स बंद करा  
   - स्वॅप स्पेस वाढवा  
   - हलके OS (लाइट व्हर्जन) वापरा  
   - RAM अपग्रेड करा (Pi 4 मध्ये 2/4/8GB)  
2. Wio Terminal साठी:  
   - बफर आकार कमी करा  
   - लहान प्रतिमा वापरा  
   - स्ट्रिंगचा वापर ऑप्टिमाइझ करा  
   - मेमरी लीक तपासा (जी मेमरी रिलीज होत नाही)  

### समस्या: डेटा गमावणे किंवा भ्रष्ट होणे  
**लक्षणे:** संदेश हरवणे, फाईल्स corrupt होणे

**उपाय:**  
1. SD कार्ड समस्या:  
   - दर्जेदार SD कार्ड वापरा (स्वस्त/फसवणूक टाळा)  
   - नियमित बॅकअप घ्या  
   - शटडाऊन नीट करा (पॉवर अचानक बाहेर काढू नका)  
2. बफर ओव्हरफ्लो: कोडमध्ये बफर आकार वाढवा  
3. नेटवर्क विश्वासार्हता: रीट्राय लॉजिक आणि त्रुटी हाताळणी करा  
4. सेवा गुणवत्ता: महत्वाच्या संदेशांसाठी MQTT QoS 1 किंवा 2 वापरा

## सामान्य त्रुटी संदेश

### `ModuleNotFoundError: No module named 'X'`  
**कारण:** पॅकेज इन्स्टॉल नाही किंवा वर्चुअल एन्व्हायर्नमेंट सक्रिय नाही

**उपाय:**  
```bash
pip install X
```
सर्वप्रथम वर्चुअल एन्व्हायर्नमेंट सक्रिय करा.  

### Linux/macOS वर `Permission denied`  
**कारण:** उच्च अधिकार आवश्यक किंवा फाईल परवानग्यांची समस्या

**उपाय:**  
- सिस्टम ऑपरेशन्ससाठी: `sudo` वापरा  
- pip साठी: venv मध्ये प्रथम सक्रिय करा आणि नंतर sudo वापरू नका  
- सिरियल पोर्टसाठी: `sudo usermod -a -G dialout $USER`, मग लॉगआउट/लॉगिन करा  

### `OSError: [Errno 98] Address already in use`  
**कारण:** पोर्ट आधीच दुसऱ्या प्रक्रियेने वापरला जात आहे

**उपाय:**  
1. पोर्ट वापरलं जातं ते शोधा: `lsof -i :<port>` किंवा `netstat -ano | findstr :<port>`  
2. प्रक्रिया थांबवा किंवा कोडमध्ये वेगळा पोर्ट वापरा  

### `SSL: CERTIFICATE_VERIFY_FAILED`  
**कारण:** SSL प्रमाणपत्र प्रमाणीकरण अयशस्वी

**उपाय:**  
1. प्रमाणपत्रे अपडेट करा: `pip install --upgrade certifi`  
2. सिस्टीमचा वेळ बरोबर आहे का तपासा: `date`  
3. फक्त विकासासाठी (प्रॉडक्शनसाठी नाही): कोडमध्ये प्रमाणीकरण वगळा  

### `IndentationError: unexpected indent`  
**कारण:** Python मध्ये चुकीचा इंडेंटेशन (टॅब/स्पेस मिसळणे)

**उपाय:**  
1. सातत्यपूर्ण इंडेंटेशन वापरा (4 स्पेसेस हे Python चे मानक)  
2. एडिटर स्पेस वापर도록 सेट करा, टॅब्स नाही  
3. VS Code मध्ये `"editor.insertSpaces": true` आणि `"editor.tabSize": 4` सेट करा  

### `UnicodeDecodeError` किंवा `UnicodeEncodeError`  
**कारण:** कॅरेक्टर एनकोडिंग समस्या

**उपाय:**  
```python
# फाइल वाचताना
with open('file.txt', 'r', encoding='utf-8') as f:
    content = f.read()

# फाइल लिहिताना
with open('file.txt', 'w', encoding='utf-8') as f:
    f.write(content)
```
